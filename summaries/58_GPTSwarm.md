> **arXiv**: 2402.16823 (2024.02)

## Overview

| 항목 | 내용 |
|------|------|
| **Problem** | LLM 에이전트 시스템의 프롬프트와 토폴로지(에이전트 간 연결 구조)를 수동으로 설계해야 하며, 이들을 동시에 자동 최적화하는 방법이 없다. |
| **Motivation** | 에이전트 시스템을 계산 그래프로 모델링하면, 각 노드(에이전트/도구)의 프롬프트와 노드 간 엣지(정보 흐름)를 최적화 변수로 취급하여 자동으로 개선할 수 있다. 이는 신경망의 가중치 최적화와 유사한 패러다임. |
| **Limitation** | 저자 언급: 대규모 에이전트 그래프에서의 최적화 수렴 속도. 독자 관점: 그래프 구조 최적화가 이산적이라 미분 불가능, 메타 최적화에 LLM 호출이 필요하여 비용 발생. |

---

## Method

1. **에이전트 시스템을 계산 그래프로 모델링**: 노드 = LLM 에이전트 또는 도구, 엣지 = 정보 흐름/의존 관계
2. **노드 수준 최적화 (프롬프트)**: 각 에이전트의 프롬프트를 자동으로 개선
3. **엣지 수준 최적화 (토폴로지)**: 에이전트 간 연결 구조를 자동으로 탐색·수정
4. **자기 개선 에이전트 스웜**: 웹 검색, 메모리 등의 도구와 결합한 에이전트 그룹이 지속적으로 자기 개선

---

## Key Contribution

1. **에이전트 시스템의 그래프 추상화**: LLM 에이전트를 최적화 가능한 계산 그래프로 모델링하는 최초의 체계적 프레임워크
2. **프롬프트 + 토폴로지 동시 최적화**: 노드(프롬프트)와 엣지(구조)를 함께 자동 최적화

---

## Experiment & Results

- **학회**: ICML 2024 Oral (top 1.5%)
- 다양한 태스크에서 수동 설계 에이전트 시스템 대비 일관된 성능 향상
- 프롬프트 최적화만 또는 토폴로지 최적화만 수행하는 것보다 동시 최적화가 효과적
