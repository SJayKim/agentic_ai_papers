# Evaluating Long-Context Reasoning in LLM-Based WebAgents

> **논문 정보**: Andy Chung, Yichi Zhang, Kaixiang Lin, Aditya Rawal, Qiaozi Gao, Joyce Chai (University of Michigan, Amazon)
> **arXiv**: 2512.04307 (2025.12) | **학회**: NeurIPS 2025 Workshop (LAW 2025)
> **코드**: WebCanvas 기반 구현

---

## Problem

LLM 기반 WebAgent가 일상 디지털 환경에 통합됨에 따라, 장기 상호작용 이력(long interaction history)에 걸쳐 추론하는 능력이 필수적이다. 사용자는 며칠 전 에이전트가 수행한 작업의 결과를 참조하여 후속 지시를 내릴 수 있으며, 이를 위해 에이전트는 이전 컨텍스트를 정확히 회수하고 활용해야 한다. 그러나 현실적인 웹 환경에서 장문 컨텍스트(25K~150K 토큰) 상황에서 WebAgent의 성능을 체계적으로 평가한 연구는 거의 없다. 기존 벤치마크는 QA 형식에 한정되거나 비교적 짧은 컨텍스트만 분석하여 수백 스텝에 달하는 장기 상호작용에 대한 세밀한 분석이 부재하다. 실제 오픈 인터넷이라는 동적 환경에서 에이전트가 노이즈가 섞인 긴 이력 속에서 관련 정보를 검색하고 올바른 행동을 이어가는 능력에 대한 체계적 평가 프레임워크가 필요하다.

---

## Motivation

사용자가 에이전트에게 "앞서 저장한 그 영화를 구매해줘"처럼 이전 맥락을 암묵적으로 참조하는 지시를 내릴 때, 에이전트는 수많은 무관한 중간 작업 이력 속에서 올바른 정보를 찾아내야 한다. 이는 단순한 정보 검색이 아니라, 검색한 정보를 기반으로 실제 웹 환경에서 행동을 수행해야 하는 복합 추론 과제이다. 기존 long-context 평가는 대부분 텍스트 QA나 needle-in-a-haystack 패러다임에 머물러 있으며, 실제 행동 수행 에이전트의 맥락에서는 검증되지 않았다. 현실에서 사용자의 상호작용 이력은 시간이 지남에 따라 다양한 도메인의 작업이 누적되어 자연스럽게 노이즈가 증가한다. 따라서 노이즈가 포함된 장문 컨텍스트에서 WebAgent의 순차 의존 작업 수행 능력을 정량적으로 평가하는 벤치마크가 필요하다.

---

## Method

1. **순차 의존 서브태스크 설계**: 하나의 작업 A를 두 개의 순차 의존 서브태스크 A1, A2로 분할한다. A2는 A1에서 얻은 정보가 없으면 완수 불가능하도록 Under-specification 조건과, 주입된 무관한 작업과 혼동되지 않도록 Unambiguous 조건을 만족해야 한다.

2. **노이즈 주입을 통한 장문 컨텍스트 시뮬레이션**: A1 완료 후, 무관한 작업 궤적을 주입하여 25K~150K 토큰 범위의 컨텍스트를 생성. 사용자가 A1 이후 여러 다른 작업을 수행한 시간 경과를 시뮬레이션한다.

3. **데이터셋 구축**: WebCanvas의 543개 주석 작업에서 Claude-3.5/3.7로 실행하여 성공한 56개 작업을 선별, LLM으로 A1/A2로 분할 후 수동 검수.

4. **평가 기준**: WebCanvas의 규칙 기반 Key Step 평가를 채택. 태스크당 평균 4.5개의 Key Step을 모두 통과해야 성공.

5. **오류 분류 체계**: False End(미완료 종료), Loops(반복 루프), Inefficient Progress(비효율적 진행으로 스텝 한도 도달), Other Errors.

6. **검색 성능 분석**: A2 수행 시 올바른 도메인으로 복귀(retrieval success) 여부를 별도 측정, Task Success/Failure × Retrieval Success/Failure의 4분면 분석.

7. **Implicit RAG (iRAG)**: 각 스텝에서 에이전트에게 현재 서브태스크와 상호작용 이력에 대한 요약을 먼저 생성하게 한 뒤, 이 요약을 컨텍스트에 추가하여 행동 결정을 수행.

---

## Key Contribution

1. **WebAgent의 장문 컨텍스트 추론 능력을 평가하는 최초의 벤치마크**: 순차 의존 서브태스크와 노이즈 주입으로 25K~150K 토큰의 현실적 멀티세션 시나리오 시뮬레이션.
2. **컨텍스트 길이 증가에 따른 극적 성능 저하 실증**: 40-50% → 10% 이하로 하락.
3. **핵심 실패 모드 체계적 분류**: 루프 갇힘, 비효율적 진행, 원래 목표 상실 등을 정량화.
4. **Implicit RAG(iRAG) 접근법 제안**: 150K 컨텍스트에서 소폭 향상이나 근본적 한계 지속을 확인.

---

## Experiment & Results

- **기본 성능(No Noise)**: o4-mini 46.2% ± 3.8%, GPT-4.1 44.4% ± 3.8%로 절반 이하.
- **장문 컨텍스트 성능 저하**: 150K 토큰에서 GPT-4.1은 **2.7% ± 1.1%**, o4-mini는 **21.2% ± 3.1%**로 급락. Claude-3.7과 Llama-4도 10% 이하.
- **모델별 오류**: GPT-4.1 루프 비율 **44.29%**, Claude-3.7 Inefficient Progress **34.97%**, o4-mini 루프 **43.15%**.
- **검색 성능**: Task Success + Retrieval Failure 조합은 모든 모델에서 **0.0%** — 검색 실패 시 작업 성공 불가. GPT-4.1은 Task Failure + Retrieval Failure가 **72.0% ± 5.3%**로 검색 능력 가장 취약.
- **iRAG 효과**: 150K에서 Claude-3.7, GPT-4.1, o4-mini 모두 소폭 향상이나 전체 성공률은 여전히 낮음.
- **추론 모델의 우위**: o4-mini(추론 모델)가 비추론 모델 대비 장문 컨텍스트에서 상대적으로 우수.

---

## Limitation

- 실험 비용: API 속도 제한과 장문 컨텍스트로 한 번의 실행에 약 16일 소요.
- 서브태스크 분할의 비현실성: 원자적 작업은 자연스러운 순차 분할이 어렵다.
- 동적 인터넷 환경: 웹사이트 레이아웃 변경, 봇 차단 등으로 정확한 재현성 확보가 불가.
- LLM 비결정성: 온도 0에서도 완전 결정적이지 않으며, 장문 추론 체인에서 변동성이 증폭.
- 4개 모델만 평가하여 다른 아키텍처나 파인튜닝된 에이전트 전용 모델에 대한 검증 부재.
