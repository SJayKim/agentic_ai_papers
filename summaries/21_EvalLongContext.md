# Evaluating Long-Context Reasoning in LLM-Based WebAgents

> **논문 정보**: Andy Chung, Yichi Zhang, Kaixiang Lin, Aditya Rawal, Qiaozi Gao, Joyce Chai (University of Michigan, Amazon)
> **arXiv**: 2512.04307 (2025.12) | **학회**: NeurIPS 2025 Workshop (LAW 2025: Bridging Language, Agent, and World Models)
> **코드**: WebCanvas 기반 구현, Set-of-Marks 그라운딩과 접근성 트리를 활용한 WebAgent 아키텍처

---

## Problem

LLM 기반 WebAgent가 일상 디지털 환경(쇼핑, SNS, 뱅킹, 엔터테인먼트, 리서치 등)에 빠르게 통합되면서, 장기 상호작용 이력에 걸쳐 추론하는 능력이 개인화와 맥락 인식 보조의 핵심 요건이 되었다.

사용자는 "IMDb에서 최고 평점 영화를 찾아 watchlist에 추가해줘"라고 지시한 뒤 며칠 후 "앞서 저장한 그 영화를 구매해줘"처럼 이전 맥락을 암묵적으로 참조하는 후속 지시를 내리게 된다.

그러나 현실적인 웹 환경에서 장문 컨텍스트(25K~150K 토큰) 조건에서 행동 수행 에이전트(action-taking agent)의 성능을 체계적으로 평가한 연구는 거의 전무하다.

기존 벤치마크인 Prefeval은 QA 형식의 선호도 준수에 국한되어 있어 트라젝토리 기반 행동 수행 에이전트의 평가에 부적합하다.

RealWebAssist는 장기 웹 태스크를 다루지만 최대 20스텝 수준의 비교적 짧은 컨텍스트 길이만 분석하여, 수백 스텝에 달하는 장기 상호작용과 세밀한 오류 분석이 부재하다.

WebAgent는 지시문, 관측(accessibility tree + 스크린샷), 사고, 행동, 반영을 모두 메모리에 저장하므로 상호작용 이력이 시간이 지날수록 복잡하고 길어지는데, 이 이력 속에서 관련 정보를 추출해 올바른 행동을 이어가는 능력이 검증되지 않았다.

동적인 오픈 인터넷 환경(레이아웃 변경, 봇 차단, 광고 교체 등)에서 이러한 장기 추론 능력을 정량적으로 측정할 표준화된 벤치마크가 시급하다.

인간 사용자의 96%가 온라인에 있는 북미에서 WebAgent 활용이 보편화되는 상황에서, 실제 배포에 앞선 장기 신뢰성 검증은 필수적이다.

---

## Motivation

사용자가 에이전트에게 "앞서 저장한 그 영화를 구매해줘"처럼 이전 맥락을 암묵적으로 참조하는 지시를 내릴 때, 에이전트는 수많은 무관한 중간 작업 이력 속에서 올바른 정보를 찾아내야 한다.

이는 단순한 정보 검색이 아니라, 검색한 정보를 기반으로 실제 웹 환경에서 도메인 전환·클릭·폼 입력 등 구체적 행동을 수행해야 하는 복합 추론 과제이다.

기존 long-context 평가는 대부분 텍스트 QA나 needle-in-a-haystack 패러다임에 머물러 있으며, 실제 행동 수행 에이전트의 맥락에서는 전혀 검증되지 않았다.

현실에서 사용자의 상호작용 이력은 시간이 지남에 따라 여행·쇼핑·정보검색·서비스·엔터테인먼트 등 다양한 도메인의 작업이 누적되어 자연스럽게 노이즈가 증가한다.

한 스텝에서 WebAgent는 관측 해석, 이력 추론, 이전 피드백 활용, 사고 생성, 행동 선택, 올바른 웹 요소 식별, 구조화 출력이라는 다중 지시를 동시에 수행해야 하므로 장문 컨텍스트는 이 모든 과정을 저해할 수 있다.

따라서 노이즈가 포함된 장문 컨텍스트에서 WebAgent의 순차 의존 작업 수행 능력을 정량적으로 평가하는 벤치마크와, 실패 모드를 분해 분석하여 향후 메모리·계획 아키텍처 개선에 기여하는 연구가 필요하다.

특히 모델별 특성(추론 모델 vs 비추론 모델, 대형 폐쇄 모델 vs 오픈 모델)에 따른 장문 컨텍스트 행동 수행 차이를 이해해야 실제 배포 시 적합한 모델을 선택할 수 있다.

## Method

1. **순차 의존 서브태스크 설계**: 하나의 작업 A를 두 개의 순차 의존 서브태스크 A1, A2로 분할한다.

2. **Under-specification 조건**: A2는 A1에서 얻은 속성 정보가 누락되어야 하며(예: A1이 "공항 근처 호텔 찾기"면 A2는 "그 공항 근처 음식점 찾기"로 '공항' 속성이 미지정), 이를 통해 에이전트가 이전 트라젝토리 회수를 강제한다.

3. **Unambiguous 조건**: A1 외의 주입된 어떤 작업도 A2 수행에 사용될 수 없도록 LLM 분류기로 검증하여 의존 체인을 명확히 유지한다.

4. **데이터셋 소스 선정**: LLM으로 태스크를 직접 생성하면 실행 불가능한 경우(예: expedia에서 빈 폼으로 검색 버튼 클릭)가 많아, WebCanvas의 543개 주석 작업(dev 438 + test 105, 평균 8스텝, 총 4550스텝)을 기반으로 활용한다.

5. **실행 가능 작업 선별**: Claude-3.5와 Claude-3.7을 기반 모델로 WebAgent를 전체 데이터셋에 실행해 성공한 56개 작업만 남긴다.

6. **A1/A2 분할 및 수동 검수**: LLM으로 각 작업을 A1과 A2로 분할 후, A1이 필요한 모든 정보를 담는지, A2가 under-specification을 만족하는지 사람이 직접 확인한다.

7. **노이즈 주입을 통한 멀티세션 시뮬레이션**: A1 완료 후, 무관한 작업 궤적을 주입하여 25K, 50K, 75K, 100K, 150K 토큰 범위의 컨텍스트를 생성. 에이전트의 현재 URL을 마지막 주입 작업의 마지막 스텝 URL로 설정하여 시간 경과를 시뮬레이션한다.

8. **기준선(No Noise) 조건**: 노이즈 주입 없이 A1→A2를 바로 수행하는 baseline을 설정해 컨텍스트 길이만의 영향을 분리 측정한다.

9. **규칙 기반 Key Step 평가**: LLM 기반 평가는 편차가 크다는 관찰(WebVoyager 등 선행 연구 및 Anthropic 문서에서 T=0에서도 비결정성 확인)에 따라, WebCanvas의 평균 4.5개 Key Step이 모두 통과되어야 성공으로 판정.

10. **오류 분류 체계**: False End(미완료 종료), Loops(반복 루프로 스텝 한도 도달), Inefficient Progress(진행은 하지만 스텝 한도 도달), Other Errors(타임아웃·봇 차단·요청 거부).

11. **검색 성능 4분면 분석**: A2 수행 시 올바른 도메인 복귀 여부(Retrieval Success/Failure)와 태스크 성공 여부(Task Success/Failure)를 교차해 TS×RS, TS×RF, TF×RS, TF×RF 네 가지 조합으로 세분화.

12. **Efficiency Ratio**: #Reference Steps / #WebAgent Steps 공식으로 인간 annotator 대비 효율을 측정, 스텝 한도 도달 빈도를 간접 반영.

13. **Implicit RAG (iRAG)**: 각 스텝에 요약 생성 서브스텝 추가. 현재 서브태스크와 상호작용 이력에 대한 요약을 먼저 생성하고, 이 요약을 컨텍스트에 append한 뒤 행동 결정 수행. 복잡한 지시를 하위 지시로 분해하는 접근.

14. **WebAgent 아키텍처**: Planning(관측→사고→행동) + Evaluation(반영 생성) + Memory(이력 저장) 3모듈 구조, Set-of-Marks 시각 그라운딩, HTML selector 기반 행동 공간(goto, click, fill_form, select, search, go_back, scroll, cache_data, get_final_answer).

15. **평가 설정**: Claude-3.7(thinking token 미사용), GPT-4.1, Llama-4-17B Maverick, o4-mini 네 모델 평가. o4-mini 제외 모두 temperature=0. 이진 변수에는 표준오차 of proportion, 연속 변수에는 표준오차 of mean 사용.

## Key Contribution

1. **WebAgent의 장문 컨텍스트 추론을 평가하는 최초의 실증 벤치마크**: 순차 의존 서브태스크 + 노이즈 주입 기법으로 25K~150K 토큰의 현실적 멀티세션 시나리오 시뮬레이션, 기존 QA 기반 long-context 평가와 차별화.

2. **컨텍스트 길이 증가에 따른 극적 성능 저하의 실증적 정량화**: baseline 40~50%에서 150K 컨텍스트 10% 미만으로 하락, Claude-3.7·GPT-4.1·Llama-4·o4-mini 네 SOTA 모델에서 공통 관찰.

3. **장문 컨텍스트 실패 모드의 체계적 분류와 세분화**: Loops(루프 갇힘), Inefficient Progress(비효율적 진행), False End(미완료 종료), Other Errors 네 범주 + 모델별 편향 분석.

4. **검색-행동 4분면 평가 프레임워크 제안**: Task×Retrieval의 2×2 매트릭스로 도메인 복귀 능력과 태스크 수행 능력을 분리 분석, 검색 실패 시 태스크 성공 불가라는 결정적 관찰(TS+RF=0%).

5. **Implicit RAG(iRAG) 접근법 제안 및 효과 입증**: 각 스텝에 하위 지시 분해 요약 생성 단계를 추가하여 150K 컨텍스트에서 검색률과 성공률을 소폭 개선, 특히 GPT-4.1에서 가장 큰 혜택 관찰.

6. **규칙 기반 Key Step 평가의 필요성 재확인**: LLM 기반 평가의 비결정성 문제를 공식적으로 지적하고 WebCanvas의 rule-based 체크포인트 방식을 장문 컨텍스트 평가에 적용.

7. **추론 모델 우위 발견**: o4-mini가 비추론 모델 대비 장문 컨텍스트에서 상대적으로 우수한 성능(150K에서 21.2% vs GPT-4.1 2.7%)을 보여 추론 능력과 장문 견고성의 상관관계 시사.

## Experiment

- **기본 성능(No Noise)**: o4-mini 46.2% ± 3.8%, GPT-4.1 44.4% ± 3.8%로 절반 이하의 성공률. 이는 순차 의존 서브태스크 자체의 난이도를 반영.

- **150K 토큰 성능 붕괴**: GPT-4.1은 **2.7% ± 1.1%**로 붕괴, o4-mini는 **21.2% ± 3.1%**로 상대적 우위 유지. Claude-3.7과 Llama-4도 10% 이하로 하락.

- **노이즈 주입 효과**: 노이즈가 들어오는 순간 성공률이 약 30%p 감소, 검색이 요구되는 순간 급격한 성능 하락 확인.

- **오류 분포 (Claude-3.7)**: False End 16.37% ± 1.47%, Inefficient Progress **34.97% ± 2.65%**, Loops 16.67% ± 1.66%, Other Errors 1.19% ± 0.40%, Success 30.80% ± 3.48%.

- **오류 분포 (GPT-4.1)**: False End 6.90% ± 1.09%, Inefficient Progress 32.74% ± 5.63%, Loops **44.29% ± 8.03%**, Other Errors 1.67% ± 0.51%, Success 14.40% ± 4.22%.

- **오류 분포 (Llama-4, 최대 75K까지만 평가)**: False End 11.61% ± 3.24%, Inefficient Progress **39.96% ± 3.80%**, Loops 21.21% ± 1.37%, Success 27.01% ± 5.10%.

- **오류 분포 (o4-mini)**: False End 2.68% ± 0.61%, Inefficient Progress 19.35% ± 5.16%, Loops **43.15% ± 2.86%**, Success 33.93% ± 6.17%.

- **검색 성능 4분면 (Claude-3.7)**: TS+RF 0.0%, TS+RS 25.1% ± 2.0%, TF+RS 26.9% ± 1.5%, **TF+RF 48.0% ± 2.3%**.

- **검색 성능 4분면 (GPT-4.1)**: TS+RF 0.0%, TS+RS 7.0% ± 1.4%, TF+RS 21.0% ± 4.4%, **TF+RF 72.0% ± 5.3%** — 검색 능력이 가장 취약.

- **검색 성능 4분면 (Llama-4)**: TS+RF 0.0%, TS+RS 17.1% ± 2.8%, **TF+RS 42.9% ± 1.9%**, TF+RF 40.0% ± 4.1% — 검색은 되지만 태스크 실패가 지배적.

- **검색 성능 4분면 (o4-mini)**: TS+RF 0.0%, TS+RS 21.7% ± 1.2%, TF+RS 37.4% ± 2.3%, TF+RF 40.9% ± 3.2%.

- **핵심 발견**: 모든 모델에서 TS+RF = **0.0%** — 검색 실패 시 태스크 성공은 원천적으로 불가능함을 입증.

- **iRAG 효과**: 150K 컨텍스트에서 Claude-3.7, GPT-4.1, o4-mini 모두 baseline 대비 개선. Claude-3.7과 Llama-4에서 검색 카운트 modest 개선, **GPT-4.1이 iRAG의 최대 수혜**.

- **Efficiency Ratio**: No Noise 조건에서는 인간 annotator에 근접하거나 초과하는 효율을 보이나, 컨텍스트 길이 증가에 따라 급격히 하락(대부분 스텝 한도 도달).

- **실험 비용**: API rate limit과 장문 컨텍스트 특성상 한 번의 실험 실행에 약 16일 소요.

- **데이터셋 규모**: WebCanvas 543개 작업에서 Claude-3.5/3.7 실행으로 성공 56개 선별, 평균 4.5개 Key Step/task.

## Limitation

- 실험 비용: API rate limit과 장문 컨텍스트로 단일 실행에 약 16일 소요되어 대규모 ablation이나 다중 seed 반복이 현실적으로 어렵다.

- 서브태스크 분할의 비현실성: WebCanvas의 많은 작업이 원자적(atomic)이어서 자연스러운 순차 분할이 어렵다. 예컨대 "Apple.com에서 iPod mini 가격 찾기"는 "iPod mini 페이지 이동"과 "가격 확인"이라는 어색한 분할만 가능.

- 동적 인터넷 환경: 페이지 레이아웃 변경, 봇 차단 조치, 광고 교체, 사이트 구조 업데이트 등으로 이전에 실행 가능하던 태스크가 갑자기 실행 불가능해지거나 그 반대가 가능해 정확한 재현성 확보가 불가.

- LLM 비결정성: temperature=0에서도 완전한 결정성이 보장되지 않으며(Anthropic·OpenAI 문서 공식 인정), 장문 추론 체인에서 문자열 단위 변동성이 증폭된다.

- 모델 커버리지 제한: Claude-3.7·GPT-4.1·Llama-4·o4-mini 4개 모델만 평가했으며, Llama-4는 75K 토큰까지만 평가해 150K 영역에서 직접 비교 불가. 다른 아키텍처나 에이전트 전용 파인튜닝 모델의 검증 부재.

- iRAG의 근본적 한계: 요약이 적절히 생성되어 검색이 성공해도, 에이전트가 그 정보를 활용해 올바른 행동을 수행하지 못하고 마지막 주입 트라젝토리를 따라가는 사례가 여전히 존재 — 요약만으로는 행동 판단 오류를 해결하지 못함.
