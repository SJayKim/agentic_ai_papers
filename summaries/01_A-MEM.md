# A-MEM: Agentic Memory for LLM Agents

> **논문 정보**: Wujiang Xu, Zujie Liang, Kai Mei, Hang Gao, Juntao Tan, Yongfeng Zhang (Rutgers University, AIOS Foundation)
> **arXiv**: 2502.12110v11 (2025.10.08) | **학회**: Preprint
> **코드**: https://github.com/WujiangXu/AgenticMemory / https://github.com/WujiangXu/A-mem-sys

---

## Problem

기존 LLM 에이전트 메모리 시스템(MemGPT, MemoryBank, SCM 등)은 에이전트 개발자가 메모리 저장 구조, 저장 시점, 검색 타이밍을 사전에 정의해야 하며, 이 고정된 패턴이 다양한 태스크 환경에서의 적응력을 근본적으로 제한한다. Mem0 등이 그래프 데이터베이스를 도입하여 구조화된 메모리 조직을 시도했으나, 사전 정의된 스키마와 관계에 의존하기 때문에 새로운 지식이 들어올 때 혁신적 연결을 만들거나 조직 패턴을 발전시키는 것이 불가능하다. 예컨대 에이전트가 새로운 수학적 풀이법을 학습하더라도 기존 프레임워크 내의 카테고리와 링크로만 저장할 수 있어, 지식 진화가 차단된다. 이러한 경직된 구조와 고정된 워크플로우는 에이전트가 복잡하고 개방적인 태스크를 장기적으로 처리하는 데 심각한 병목이 된다. 결국 유연하고 범용적인 메모리 시스템의 부재가 LLM 에이전트의 장기 상호작용 능력을 가로막는 핵심 과제이다.

---

## Motivation

저자들은 Zettelkasten(제텔카스텐) 지식관리 방법론에서 핵심 직관을 가져온다. Zettelkasten은 원자적 노트(atomic note)와 유연한 링킹으로 상호 연결된 지식 네트워크를 구축하는 체계로, 사전 정의된 분류 없이도 지식이 자연스럽게 조직화되고 진화하는 특성을 갖는다. 이 원리를 LLM 에이전트 메모리에 적용하면, 고정된 스키마 대신 메모리 자체가 자율적으로 맥락 설명을 생성하고, 의미적 유사성에 기반해 다른 메모리와 링크를 형성하며, 새 경험이 축적될 때 기존 메모리의 표현까지 갱신하는 동적 진화가 가능해진다. 기존 Agentic RAG가 검색(retrieval) 단계에서만 에이전시를 발휘하는 것과 달리, 저장과 진화 수준에서 에이전시를 부여함으로써 정적 지식 베이스의 근본적 한계를 넘어설 수 있다는 관점이 핵심적으로 차별화된다.

## Method

1. **Note Construction (노트 생성)**: 에이전트가 환경과 상호작용할 때, 각 메모리 m_i를 {원본 콘텐츠 c_i, 타임스탬프 t_i, LLM 생성 키워드 K_i, 태그 G_i, 맥락 설명 X_i, 임베딩 e_i, 링크 집합 L_i}의 다속성 노트로 구성한다. LLM에 설계된 프롬프트 템플릿을 입력하여 키워드·태그·맥락 설명을 자동 생성하고, 텍스트 인코더(all-minilm-l6-v2)로 모든 텍스트 요소를 결합한 밀집 벡터를 생성한다.

2. **Link Generation (링크 생성)**: 새 노트 m_n이 추가되면, 코사인 유사도 기반으로 기존 메모리에서 top-k(기본 k=10)개의 가장 관련 높은 메모리 M_near를 검색한다. 이후 LLM이 새 노트와 후보 메모리들의 공통 속성을 분석하여 의미 있는 연결이 존재하는 쌍에 대해 양방향 링크를 생성한다. 관련 메모리들이 "박스(box)"로 묶여 Zettelkasten의 유연한 링킹을 구현하되, 하나의 메모리가 동시에 여러 박스에 속할 수 있다.

3. **Memory Evolution (메모리 진화)**: 링크 생성 후, 최근접 이웃 M_near의 각 메모리 m_j에 대해 LLM이 새 메모리와의 관계를 분석하여 기존 메모리의 맥락 설명, 키워드, 태그를 갱신할지 결정한다. 진화된 메모리 m*_j가 원본을 대체하며, 시간이 지남에 따라 고차 패턴과 개념이 자연 발생적으로 형성된다.

4. **Memory Retrieval (메모리 검색)**: 질의 텍스트 q를 동일 텍스트 인코더로 임베딩한 뒤 코사인 유사도로 top-k 메모리를 검색한다. 검색된 메모리에 링크된 같은 박스 내의 관련 메모리도 함께 접근되어, 연결된 맥락 정보가 에이전트의 추론을 풍부하게 한다.

## Key Contribution

1. **자율적 메모리 조직화 시스템**: 사전 정의된 메모리 연산 없이 LLM 에이전트가 맥락 설명 생성, 동적 연결 수립, 기존 메모리 진화를 자율적으로 수행하는 최초의 에이전틱 메모리 아키텍처를 제안하여, 고정 스키마 기반 시스템의 적응성 한계를 해결한다.
2. **링크 생성 + 메모리 진화 메커니즘**: 새 메모리 유입 시 임베딩 유사도 필터링과 LLM 기반 관계 분석을 결합하여 유기적 연결을 형성하고, 기존 메모리의 표현까지 갱신하는 이중 메커니즘을 설계하여 정적 지식 베이스 한계를 극복한다.
3. **토큰 효율성**: 선택적 top-k 검색으로 메모리 연산당 약 1,200 토큰만 사용하여, 전체 대화를 투입하는 baseline(~16,900 토큰) 대비 85~93% 토큰 절감을 달성한다.
4. **범용 확장성**: 6개 파운데이션 모델(GPT-4o, GPT-4o-mini, Qwen2.5 1.5B/3B, Llama 3.2 1B/3B)에서 일관된 성능 향상을 검증하여 모델 비의존적 범용성을 입증한다.

## Experiment & Results

**데이터셋**: (1) LoCoMo — 평균 9K 토큰, 최대 35 세션의 장기 대화 데이터셋, 7,512개 QA 쌍(Multi-hop, Temporal, Open Domain, Single Hop, Adversarial 5개 카테고리). (2) DialSim — TV 시리즈 기반 장기 다자 대화 QA(1,300 세션, ~350K 토큰, 1,000+ 질문).

**Baseline**: LoCoMo, ReadAgent, MemoryBank, MemGPT.

**주요 수치**:
- GPT-4o-mini 기준 Temporal F1에서 A-MEM 45.85 vs MemGPT 25.52 vs LoCoMo 18.41 — 기존 최고 대비 **80% 향상**.
- Qwen2.5-3B에서 평균 F1 랭킹 1.0으로 모든 baseline을 상회; Multi-hop F1 12.57(MemGPT 5.07 대비 148% 향상).
- Llama 3.2-1B에서 Single Hop F1 28.51(MemGPT 10.16 대비 181% 향상), Adversarial F1 58.81.
- DialSim에서 A-MEM F1 3.45, SBERT Similarity 19.51 — LoCoMo(2.55, 15.76) 대비 각각 35%, 24% 향상; MemGPT(1.18, 8.54) 대비 192%, 128% 향상.
- 토큰 사용량: A-MEM ~1,200 토큰 vs LoCoMo/MemGPT ~16,900 토큰 (85~93% 절감). 연산당 비용 $0.0003 미만.

**Ablation (GPT-4o-mini)**: Link Generation(LG)과 Memory Evolution(ME) 모두 제거 시 Multi-hop F1이 27.02→9.65로 64% 하락. ME만 제거 시 21.35로 중간 성능. 두 모듈이 상호보완적으로 작동함을 확인.

**Scaling**: 메모리 100만 건까지 확장 시 검색 시간 0.31μs→3.70μs로 선형 이하 증가. 메모리 사용량은 MemoryBank와 동일(O(N)), 추가 저장 오버헤드 없음. ReadAgent는 동일 규모에서 120,069μs로 A-MEM 대비 32,000배 느림.

## Limitation

저자 명시: (1) 메모리 조직화 품질이 기반 LLM의 능력에 의존하며, 모델마다 생성하는 맥락 설명과 연결 패턴이 상이할 수 있다. (2) 현재 텍스트 기반 상호작용에만 대응하며, 이미지·오디오 등 멀티모달 확장이 미비하다.

독자 관점: (3) 메모리 진화 시 매번 LLM을 호출하므로 쓰기 지연(latency)이 발생하며, GPT-4o-mini 기준 메모리 연산당 5.4초로 실시간 대화에서 병목이 될 수 있다. (4) 평가가 LoCoMo·DialSim 두 벤치마크에 한정되어, 코드 생성·웹 탐색 등 다양한 에이전트 태스크에서의 범용성은 미검증이다. (5) 진화 과정에서 잘못된 연결이 누적될 경우의 오류 전파(error propagation) 위험이나 이에 대한 자가 교정 메커니즘이 부재하다.
