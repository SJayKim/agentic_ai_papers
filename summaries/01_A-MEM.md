# A-MEM: Agentic Memory for LLM Agents

> **논문 정보**: Wujiang Xu, Zujie Liang, Kai Mei, Hang Gao, Juntao Tan, Yongfeng Zhang (Rutgers University, AIOS Foundation)
> **arXiv**: 2502.12110v11 (2025.10.08) | **학회**: Preprint
> **코드**: https://github.com/WujiangXu/AgenticMemory / https://github.com/WujiangXu/A-mem-sys

---

## Problem

기존 LLM 에이전트 메모리 시스템(MemGPT, MemoryBank, SCM, Ret-LLM 등)은 에이전트 개발자가 메모리 저장 구조, 워크플로우 내 저장 시점, 검색 타이밍을 사전에 수동으로 정의해야 하며, 이 고정된 패턴이 새로운 태스크 환경으로의 일반화를 근본적으로 가로막는다.
Mem0와 같이 그래프 데이터베이스를 도입하여 구조화된 메모리 조직을 시도한 접근 역시 사전 정의된 스키마와 관계 유형에 의존하기 때문에, 새 지식이 유입될 때 혁신적 연결을 형성하거나 조직 패턴 자체를 발전시키는 것이 불가능하다.
예컨대 에이전트가 새로운 수학적 풀이법을 학습하더라도 기존 프레임워크 내의 카테고리와 링크 템플릿 범위에서만 저장 가능하여, 지식이 축적될수록 표현력이 오히려 병목이 된다.
MemGPT와 LoCoMo 기반 baseline은 질의마다 전체 대화(약 16,900 토큰)를 프롬프트로 투입해야 하므로 장기 상호작용에서 토큰 비용이 폭증하고, ReadAgent는 100만 건 규모에서 검색 시간이 120,069μs에 달해 확장성이 심각히 제약된다.
결국 LoCoMo 장기 대화 벤치마크에서 Temporal F1 기준 GPT-4o-mini MemGPT 25.52, LoCoMo 18.41에 머무르는 등, 경직된 구조 자체가 성능 한계로 직결된다.
이러한 정적 스키마와 수동 워크플로우의 조합은 복잡하고 개방적인 태스크를 장기적으로 수행해야 하는 LLM 에이전트의 근본적 병목으로 남아 있다.
유연하고 범용적인 메모리 시스템의 부재가 LLM 에이전트 장기 상호작용 능력의 핵심 공백이다.

## Motivation

저자들은 Zettelkasten(제텔카스텐) 지식관리 방법론을 핵심 영감으로 삼는다. 이 방법론은 원자적 노트(atomic note)와 유연한 링킹(flexible linking)으로 상호 연결된 지식 네트워크를 구축하며, 사전 정의된 분류 체계 없이도 지식이 자연스럽게 조직화되고 진화하는 특성을 갖는다.
이 원리를 LLM 에이전트 메모리에 이식하면, 고정된 스키마 대신 메모리 자체가 자율적으로 맥락 설명(contextual description)을 생성하고, 의미적 유사성과 공유 속성에 기반해 다른 메모리와 링크를 형성하며, 새 경험 유입 시 기존 메모리의 표현까지 갱신하는 동적 진화가 가능해진다.
특히 저자들은 기존 Agentic RAG(예: Self-RAG, IRCoT)가 "검색(retrieval) 단계"에서만 에이전시를 발휘하는 것과 달리, "저장(storage)과 진화(evolution) 수준"에서 에이전시를 부여해야 정적 지식 베이스의 한계를 근본적으로 넘어설 수 있다는 관점을 제시한다.
또한 LLM이 생성하는 키워드·태그·맥락 설명과 밀집 임베딩을 결합하면, 임베딩 유사도로 저렴하게 후보를 좁힌 뒤 LLM으로 정교한 관계를 분석하는 2단계 구조가 가능해져 확장성과 의미적 풍부함을 동시에 달성할 수 있다.
하나의 메모리가 여러 "박스(box)"에 동시에 속할 수 있다는 설계는 사람의 연상 학습과 유사하게 다면적 지식 조직을 가능케 한다.

## Method

1. **Note Construction (노트 구성)**: 에이전트와 환경 간 상호작용이 발생할 때마다 메모리 m_i = {c_i, t_i, K_i, G_i, X_i, e_i, L_i}를 생성한다. c_i는 원본 콘텐츠, t_i는 타임스탬프이며, 프롬프트 템플릿 P_s1을 사용해 LLM이 키워드 K_i, 태그 G_i, 맥락 설명 X_i를 자동 생성한다(식 2). Zettelkasten의 원자성 원칙에 따라 각 노트는 자기완결적인 단일 지식 단위를 담는다.

2. **Dense Embedding (밀집 임베딩)**: 텍스트 인코더 f_enc(논문에서는 all-MiniLM-L6-v2)에 c_i, K_i, G_i, X_i를 이어붙인 문자열을 입력하여 e_i = f_enc(concat(c_i, K_i, G_i, X_i))를 얻는다(식 3). 이 임베딩이 이후 유사도 기반 검색과 링크 후보 선별의 공통 기반이 된다.

3. **Link Candidate Retrieval (링크 후보 검색)**: 새 노트 m_n이 유입되면 기존 메모리 전체에 대해 코사인 유사도 s_{n,j} = e_n·e_j / (|e_n||e_j|)를 계산하고(식 4), top-k(기본 k=10) 이웃 집합 M^n_near = {m_j | rank(s_{n,j}) ≤ k}를 추출한다(식 5). 임베딩 필터링으로 LLM 호출 비용을 선형 이하로 억제한다.

4. **LLM-based Link Generation (링크 생성)**: 후보 M^n_near와 새 노트 m_n을 프롬프트 P_s2와 함께 LLM에 제시하여, 단순 유사도로는 포착되지 않는 인과·개념적 관계까지 분석해 양방향 링크 집합 L_i ← LLM(m_n ‖ M^n_near ‖ P_s2)를 갱신한다(식 6). 연결된 메모리들이 하나의 "박스"를 형성하되, 한 메모리가 동시에 여러 박스에 속할 수 있다.

5. **Memory Evolution (메모리 진화)**: 각 이웃 m_j에 대해 m*_j ← LLM(m_n ‖ M^n_near\m_j ‖ m_j ‖ P_s3)를 수행하여(식 7), 기존 메모리의 맥락 설명·키워드·태그를 새 경험에 맞춰 갱신한다. 갱신된 m*_j가 원본 m_j를 대체하여, 시간이 흐르면 고차 패턴과 개념이 자연 발생적으로 형성된다.

6. **Context-Aware Retrieval (질의 시 검색)**: 질의 q가 들어오면 동일 인코더로 e_q를 얻고, 모든 e_i와의 코사인 유사도 s_{q,i}를 계산해 top-k를 선택(식 8–10)한다. 선택된 메모리와 같은 박스에 속하는 링크 이웃 메모리도 함께 에이전트 프롬프트로 전달되어 연결된 맥락이 추론을 풍부하게 만든다.

7. **구현 및 하이퍼파라미터**: 모든 baseline과 동일한 시스템 프롬프트(Appendix B)를 공유하며, Qwen-1.5B/3B 및 Llama 3.2 1B/3B는 Ollama 로컬 배포에 LiteLLM 구조화 출력으로 실행하고, GPT 계열은 공식 structured output API를 사용한다. 검색 기본 k=10이며 카테고리별로 미세 조정한다.

## Key Contribution

1. **자율적 에이전틱 메모리 아키텍처**: 사전 정의된 메모리 연산 없이 LLM이 맥락 설명 생성, 동적 연결 수립, 기존 메모리 진화를 스스로 수행하는 최초의 에이전틱 메모리 시스템을 제안하여, MemGPT·MemoryBank 등 고정 워크플로우 기반 시스템의 적응성 한계를 해결한다.
2. **링크 생성 + 메모리 진화 이중 메커니즘**: 임베딩 기반 top-k 필터링과 LLM 기반 관계 분석을 결합해 유기적 연결을 형성하고, 신규 메모리가 기존 메모리의 표현까지 갱신하도록 설계하여 Mem0류의 정적 그래프 스키마가 가진 표현 동결 문제를 극복한다.
3. **저비용·고효율 검색**: 선택적 top-k 검색으로 메모리 연산당 약 1,200 토큰만 사용하여 LoCoMo/MemGPT baseline(~16,900 토큰) 대비 85–93% 토큰 절감, 연산당 $0.0003 미만의 비용을 달성해 대규모 배포의 경제성을 확보한다.
4. **범용 확장성 검증**: GPT-4o, GPT-4o-mini, Qwen2.5 1.5B/3B, Llama 3.2 1B/3B 6개 모델에서 일관된 평균 F1 랭킹 1.0~1.6을 기록하여 모델 규모·계열에 비의존적인 범용성을 입증한다.
5. **대규모 확장성**: 100만 건 메모리 규모에서도 검색 시간 3.70μs로 ReadAgent(120,069μs) 대비 약 32,000배 빠르며, 메모리 사용은 MemoryBank와 동일 O(N)로 추가 저장 오버헤드가 없다.

## Experiment

**데이터셋**: (1) LoCoMo — 평균 9K 토큰·최대 35 세션의 장기 대화(Multi-hop, Temporal, Open Domain, Single Hop, Adversarial 5개 카테고리, 총 7,512 QA 쌍). (2) DialSim — Friends·The Big Bang Theory·The Office 기반 TV 시리즈 장기 다자 대화(1,300 세션, 약 350,000 토큰, 세션당 1,000+ 질문).

**Baselines**: LoCoMo(전체 대화 프롬프트 직접 주입), ReadAgent(페이지 단위 gist 메모리), MemoryBank(에빙하우스 망각곡선 기반 갱신), MemGPT(OS 유사 2단 메모리).

**지표**: F1, BLEU-1, ROUGE-L, ROUGE-2, METEOR, SBERT Similarity, 평균 토큰 길이.

**주요 수치**:
- GPT-4o-mini 기준 Temporal F1에서 A-MEM 45.85 vs MemGPT 25.52 vs LoCoMo 18.41 — 기존 최고 대비 약 80% 향상, 평균 F1 랭킹 1.2.
- Qwen2.5-3B에서 평균 F1 랭킹 1.0, Multi-hop F1 12.57(MemGPT 5.07 대비 +148%), Temporal F1 27.59.
- Llama 3.2-1B에서 Single Hop F1 28.51(MemGPT 10.16 대비 +181%), Adversarial F1 58.81.
- DialSim에서 A-MEM F1 3.45, SBERT 19.51 — LoCoMo(2.55 / 15.76) 대비 +35%/+24%, MemGPT(1.18 / 8.54) 대비 +192%/+128%.
- 토큰 사용량: A-MEM ~1,200 vs LoCoMo/MemGPT ~16,900(85–93% 절감), 연산당 $0.0003 미만, 처리 시간 GPT-4o-mini 5.4초/Llama 3.2 1B 1.1초(단일 GPU).

**Ablation (GPT-4o-mini)**: Link Generation(LG)과 Memory Evolution(ME)을 모두 제거 시 Multi-hop F1이 27.02 → 9.65로 64% 하락, Single Hop 44.65 → 13.28로 급락. ME만 제거 시 Multi-hop 21.35로 중간 성능 — LG가 기반이고 ME가 정교화 역할임을 확인.

**Hyperparameter (k)**: k=10–40 범위에서 Multi-hop F1 19.91 → 27.02, Single Hop F1 31.15 → 44.55로 증가하되 임계점 이후 포화. 과도한 k는 노이즈 증가를 유발.

**Scaling**: 1K~1M 규모 4단계에서 A-MEM 검색 시간 0.31μs → 3.70μs(선형 이하), MemoryBank 0.24 → 1.91μs와 유사, ReadAgent 43.62 → 120,069μs로 폭증. 공간은 세 시스템 모두 O(N)(1M에서 1,464.84MB 동일).

**T-SNE 시각화**: LoCoMo 두 대화에서 A-MEM 임베딩이 baseline 대비 뚜렷한 군집 구조를 형성, 메모리 진화가 표현 구조화에 기여함을 시각적으로 입증.

## Limitation

저자 명시: (1) 메모리 조직화 품질이 기반 LLM의 능력에 의존하여, 모델마다 생성되는 맥락 설명과 링크 패턴이 상이할 수 있다. (2) 현재 구현은 텍스트 상호작용에 한정되며 이미지·오디오 등 멀티모달 확장이 미비하다.
독자 관점: (3) 메모리 진화 시 매 이웃마다 LLM을 호출하므로 쓰기 지연이 커져, GPT-4o-mini 기준 메모리 연산당 5.4초는 실시간 대화 루프에서 병목이 될 수 있다.
(4) 평가가 LoCoMo·DialSim 두 벤치마크(모두 장기 대화 QA)에 한정되어, 코드 생성·웹 탐색·도구 사용 등 다양한 에이전트 태스크로의 범용성은 미검증이다.
(5) LLM이 잘못된 링크나 과잉 일반화된 맥락 설명을 진화 과정에서 덮어쓸 경우 오류가 누적될 수 있으나, 이를 교정할 자가 검증·롤백 메커니즘이 부재하다.
(6) Top-k 유사도만으로 후보를 거르므로, 표면 유사도는 낮지만 개념적으로 중요한 원거리 메모리가 링크 후보에서 조기에 배제될 가능성이 있다.
(7) 메모리 갱신으로 원본 m_j가 m*_j로 대체되어 감사·재현성 관점에서 원본 궤적 추적이 어려워질 수 있다.
