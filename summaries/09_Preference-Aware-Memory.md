# Preference-Aware Memory Update for Long-Term LLM Agents

## 필수 요소

| 항목 | 내용 |
|------|------|
| **Problem** | LLM 기반 에이전트의 장기 메모리 시스템은 저장(storage)과 검색(retrieval)에는 발전이 있었지만, 시간이 지남에 따라 변화하는 사용자 선호(preference)를 동적으로 추적하고 메모리를 갱신하는 메커니즘이 결여되어 있다. 기존 시스템(MemoryBank, MemGPT, A-MEM 등)은 사용자의 선호가 시간에 걸쳐 안정적이라는 정적(static) 가정에 기반하고 있어, 사용자 행동이 변화하면 시대에 뒤떨어진 정보에 기반한 응답을 생성하게 된다. |
| **Motivation** | 실제 배포 환경에서 사용자는 비정상적(non-stationary)으로, 대화 맥락·개인 감정·업무 단계 등에 따라 어조 스타일, 감정 상태, 원하는 정보 밀도, 형식성 정도가 점진적으로 또는 갑작스럽게 변한다. 기존 메모리 시스템은 이러한 변화를 감지하지 못해 대화 일관성과 사용자 만족도를 저하시킨다. 예를 들어, 사용자가 처음에는 유머러스하고 짧은 답변을 요구하다가 갑자기 진지하고 상세한 설명을 요구하면, PAMU 없는 모델은 계속 가벼운 유머 톤으로 응답하여 사용자 의도와 어긋난다. |
| **Method** | **PAMU (Preference-Aware Memory Update Mechanism)**는 세 단계로 구성된다. (1) **Preference Extractor**: 매 대화 턴마다 사용자 선호 벡터 P = {p1, ..., pD}를 5개 차원으로 추출한다. 어조 스타일(RoBERTa 기반 다중 분류), 응답 길이(최근 K턴 평균 정규화), 감정 톤(SKEP 감정 분류 모델), 정보 밀도(OpenNRE로 추출한 (subject, predicate, object) 트리플 수 / 응답 단어 수 비율), 형식성(정규화된 0~1 점수)으로 구성된다. (2) **Preference Change Perception Module**: 슬라이딩 윈도우 평균(SW)과 지수 이동 평균(EMA)을 융합한다. SW는 단기 변화에 민감하게 반응하고, EMA(감쇠 계수 β)는 장기 추세를 안정적으로 추적한다. 두 추정치의 가중 합 wt = λ·SW + (1-λ)·EMA로 융합 선호 벡터를 생성한다. 이 두 추정치의 괴리 |SW - EMA|를 정규화한 변화 감지 점수 C_t가 임계값 δ를 초과하면 선호 변화로 판단한다. (3) **Preference-Guided Prompting**: 융합된 선호 벡터를 자연어 텍스트 지시문으로 변환하여("Please answer in style: [Tone: humorous], [Emotion: relaxed], [Density: moderate], [Length: brief]") 원래 입력 프롬프트 앞에 주입한다. 연속형 차원은 [0,0.33) → Sparse, [0.33,0.66) → Moderate, [0.66,1] → Dense로 이산화하여 의미 태그로 표현한다. 이 방법은 모델 아키텍처 수정이나 파인튜닝 없이 순수 프롬프트 엔지니어링으로 동작한다. |
| **Key Contribution** | 기존 메모리 시스템이 저장·검색에 집중한 반면, PAMU는 메모리 **업데이트** 문제를 최초로 선호 인식 관점에서 명시적으로 다룬다. SW와 EMA의 융합으로 단기 요동과 장기 추세를 동시에 모델링하는 이중 추정 구조, 두 추정치 괴리 기반 변화 감지 신호(칼만 필터 근사 구조), 그리고 어떤 메모리 프레임워크에도 플러그인 방식으로 결합할 수 있는 모듈형 설계가 핵심 novelty다. 또한 RoBERTa·SKEP·OpenNRE 등 전문 모델로 5차원 선호 벡터를 구조적으로 추출하는 명시적 표현 방식도 기존 방법과 차별화된다. |
| **Experiment/Results** | **데이터셋**: LoCoMo (50개 대화, 평균 300턴, 최대 35세션, 대화당 약 9,000 토큰). **태스크**: Single-Hop (2,705쌍), Multi-Hop (1,104쌍), Temporal Reasoning (1,547쌍). **평가 지표**: F1 Score, BLEU-1 Score. **모델**: Qwen 2.5-1.5B/3B, LLaMA-7B/30B, LLaMA 3.2-1.5B/3B. 주요 결과 (PAMU 적용 전/후, p<0.05 유의미한 개선 표시): Single-Hop(Qwen 2.5-1.5B) — A-MEM BLEU-1: 11.35 → 15.73; Multi-Hop(LLaMA 3.2-3B) — A-MEM BLEU-1: 13.27 → 23.14; Temporal Reasoning(LLaMA-7B) — A-MEM F1: 17.55 → 23.23, BLEU-1: 14.67 → 21.46; Temporal Reasoning(LLaMA-30B) — A-MEM F1: 12.54 → 19.87, BLEU-1: 11.85 → 18.23. 모든 5개 베이스라인에서 BLEU-1 기준 일관된 유의미한 향상이 관찰됐으며, 특히 Temporal Reasoning 태스크에서 F1과 BLEU-1 모두 큰 폭으로 개선됐다. Case study(GPT-4 자동 평가 + 10명 인간 평가자): PAMU 적용 시 선호 정렬 점수 2.1→4.8(GPT), 2.2→4.5(인간); 스타일 일관성 37%→92%(GPT), 35%→94%(인간); 선호 감지율 48%→97%(GPT), 45%→95%(인간). |
| **Limitation** | (1) 저자가 명시적으로 밝히지는 않았으나, F1 절대값 자체가 10~20대로 낮아 태스크 자체의 난이도가 높거나 메모리 프레임워크 전반의 성능 한계가 있음을 시사한다. (2) 선호 추출에 RoBERTa, SKEP, OpenNRE 등 여러 외부 모델을 병렬로 실행해야 하므로 추론 비용이 증가한다. (3) λ, β, W, δ 등의 하이퍼파라미터 선택이 성능에 영향을 미칠 수 있으나 그 민감도 분석이 제시되지 않는다. (4) 평가가 LoCoMo 단일 데이터셋에 한정되어 다른 도메인 및 언어로의 일반화 가능성이 불분명하다. (5) 5차원 선호 벡터가 실제 사용자 선호의 전체 공간을 충분히 포괄하는지에 대한 근거가 부족하다. |

## 선택 요소 (해당되는 것만)

| 항목 | 내용 |
|------|------|
| **Baseline 비교** | ReadAgent(RA.), MemoryBank(MB.), MemGPT(MG.), A-MEM(AM.) 4개 프레임워크에 PAMU를 플러그인 방식으로 추가하여 Before/After 비교. Single-Hop에서 Qwen 2.5-1.5B 기준 RA. BLEU-1: 4.87→8.97, MB. BLEU-1: 8.24→10.57, MG. BLEU-1: 7.54→11.46, AM. BLEU-1: 11.35→15.73. Temporal Reasoning에서 LLaMA-30B 기준 모든 베이스라인 F1 최소 3.21p 이상(MG. 5.64→9.95), 최대 7.33p(AM. 12.54→19.87) 향상. F1보다 BLEU-1에서 더 일관적이고 큰 향상이 관찰됨. |
| **Ablation Study** | Temporal Reasoning(LLaMA 7B) F1+BLEU-1 평균 기준 Full 모델 vs. 각 컴포넌트 제거 시 A-MEM 결과: Full=22.35, w/o SW=15.36(-6.99), w/o EMA=14.05(-8.30), Equal Fusion(λ=0.5 고정)=20.34(-2.01), w/o Detection=16.24(-6.11), w/o Prompt(선호 프롬프트 제거)=15.45(-6.90), Single Pref(5차원→1차원)=18.95(-3.40), Static Pref(고정 선호)=19.47(-2.88). EMA 제거가 가장 큰 성능 하락을 야기(장기 추세 모델링의 중요성)하며, SW 제거와 프롬프트 주입 제거도 각각 약 6~7점 하락으로 치명적임. 변화 감지 모듈 제거도 6.11점 하락을 유발해 적응적 응답 전략 전환의 중요성을 확인. |
