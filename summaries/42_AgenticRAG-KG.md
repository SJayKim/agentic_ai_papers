# Agentic RAG with Knowledge Graphs for Complex Multi-Hop Reasoning

> **논문 정보**: Jean Lelong, Adnane Errrazine, Annabelle Blangero (Ekimetrics, France)
> **arXiv**: 2507.16507 (2025.07)
> **코드**: N/A

---

## Overview

| 항목 | 내용 |
|------|------|
| **Problem** | 기존 RAG는 top-k 벡터 검색으로 제한적인 텍스트 스니펫만 반환하여, 완전한 목록(예: 특정 저자의 모든 논문), 다중 데이터 포인트 통합, 복잡한 관계 경로 탐색(저자→논문→펀딩 프로젝트)이 필요한 쿼리에 부적합하다. |
| **Motivation** | 지식 집약적 도메인(과학 데이터)에서는 단순 스니펫 추출이 아닌, 인간 연구자처럼 증거를 수집·평가하고 후속 검색을 계획하는 반복적 탐구가 필요하다. 지식 그래프의 구조화된 관계와 에이전트의 동적 추론을 결합하면 이를 달성할 수 있다. |
| **Limitation** | 저자 언급: 특정 기관(INRAE) 데이터에 특화된 구현. 독자 관점: 에이전트가 Cypher 쿼리를 직접 생성해야 하므로 LLM의 Cypher 문법 이해에 의존. 정량적 벤치마크 비교 없이 정성적 시연에 그침. |

## Method

1. **하이브리드 지식 베이스**
   - **Knowledge Graph (Neo4j)**: INRAE 출판물에서 구축한 417K+ 노드(저자 56%, 키워드 23%, 출판물 9.3%, 소프트웨어, 프로젝트 등) + 100만+ 관계
   - **벡터 DB**: 출판물 텍스트(제목+초록+결론)의 dense(Jina v3) + sparse(BM25) 벡터

2. **에이전트 기반 멀티툴 오케스트레이션** (DeepSeek-R1 기반)
   - **SearchGraph**: 핵심 도구. Neo4j KG에 Cypher 쿼리를 전송하여 엔티티 검색, 관계 경로 탐색, 완전한 목록 검색 수행
   - **SearchPublications**: 벡터 DB에서 하이브리드 검색 + Cohere 리랭커로 관련 출판물 텍스트 검색. 그래프 탐색의 진입점 역할
   - 에이전트가 쿼리를 이해 → 분해 → 도구 선택·호출을 반복 → 증거 수집·평가 → 종합 응답 생성

3. **다중 홉 추론**: 에이전트가 KG를 따라 여러 단계의 관계를 탐색 (저자 → 논문 → 키워드 → 관련 프로젝트)하며 증거 체인을 구축

---

## Key Contribution

1. **에이전틱 KG 쿼리의 실세계 적용**: 에이전트가 KG를 동적으로 탐색하며 반복적 다중 홉 추론을 수행하는 실제 과학 데이터 플랫폼(INRAExplorer) 구현
2. **Map-reduce 요약과의 차별화**: GraphRAG 스타일의 사전 요약 대신, 에이전트가 실시간으로 증거를 수집·평가·계획하는 "인간 연구자" 패러다임

## Experiment & Results

- **시스템**: INRAExplorer — INRAE 과학 데이터 (2019-2024 오픈액세스 출판물)
- **KG 규모**: 417,030 노드, 100만+ 관계

정성적 시연으로 시스템의 능력을 보여줌:
- "특정 저자의 모든 논문 목록" — SearchGraph로 Cypher 쿼리 생성하여 완전한 목록 반환
- "특정 연구 주제와 관련된 프로젝트 및 소프트웨어" — 다중 홉 관계 탐색
- "두 연구 분야 간 교차 연구자 식별" — KG의 관계 경로를 따라 복잡한 조인 수행

기존 RAG 대비 완전성(exhaustiveness)과 구조화된 응답에서 질적 우위를 보임
