# How Memory Management Impacts LLM Agents: An Empirical Study of Experience-Following Behavior

> **논문 정보**: Zidi Xiong, Yuping Lin, Wenya Xie, Pengfei He, Zirui Liu, Jiliang Tang, Himabindu Lakkaraju, Zhen Xiang (Harvard University, University of Georgia, Michigan State University, University of Minnesota)

> **코드**: https://github.com/yuplin2333/agent_memory_manage.git

## Problem

LLM 에이전트의 에피소딕 메모리 모듈은 과거 태스크 쿼리와 실행 궤적을 저장하고 유사 태스크에서 재검색하여 장기적 성능 향상을 가능하게 하는 핵심 구성요소이다.

그러나 기존 메모리 관리 전략들은 챗봇, 코드 생성, 운전 등 특정 에이전트 유형이나 태스크에 맞춤 설계되어 있어, 서로 다른 에이전트 아키텍처 전반에 걸쳐 일반화되는 원리를 제공하지 못한다.

특히 메모리 뱅크에 잘못된 실행 결과나 노이즈가 포함된 궤적이 축적되면, 이후 태스크에서 검색된 시연을 통해 오류가 전파(error propagation)되어 장기 성능이 점진적으로 악화된다.

또한 초기 평가자의 품질 필터를 통과한 레코드라도 현재 태스크 분포와 정렬되지 않아 오히려 해로운 시연으로 작용하는 misaligned experience replay 현상이 존재한다.

무분별한 메모리 추가(add-all)는 고정 메모리 baseline보다도 성능을 저하시키는 결과를 초래하며, 이는 단순히 경험을 더 많이 축적한다고 에이전트가 자기 개선하지 않음을 시사한다.

메모리 추가(addition)와 삭제(deletion)라는 가장 기본적 두 연산이 장기 성능에 미치는 영향에 대한 체계적이고 통합적인 분석이 부재한 상황이다.

선행 연구들은 구조 변환, 병합, 요약, 반성 등 고급 메커니즘에 치중해 왔으며, 기본 연산의 동역학을 규명한 사례는 거의 없다.

결과적으로 장기 운영되는 에이전트에서 메모리 품질 관리의 부재가 핵심 병목으로 작용하여, 신뢰 가능한 자기 개선을 가로막고 있다.

## Motivation

기존 메모리 관리 기법들은 특정 태스크에 맞춤화되어 있어 범용적 설계 원리를 제공하지 못하며, 서로 다른 에이전트에서 일관된 효과를 보장하지 못한다.

저자들은 "메모리 뱅크의 지속적 진화가 에이전트의 장기 실행에 어떤 영향을 미치는가"라는 근본적 질문을 제기하고, 가장 기본적인 두 연산인 추가와 삭제에 집중하여 답하고자 한다.

정적 외부 지식 베이스를 사용하는 in-context learning 설정과 달리, 에이전트의 메모리 뱅크는 시간에 따라 진화하고 에이전트 자신이 생성한 노이즈 궤적을 축적하는 동적 검색 풀이다.

이러한 동적 노이즈 검색 풀의 거동을 이해하면, 다양한 에이전트에 일반화되는 견고한 장기 메모리 관리 설계 원칙을 도출할 수 있다는 통찰이 본 연구의 출발점이다.

저자들은 입력 유사도와 출력 유사도 간 경험 추종(experience-following) 현상을 가설로 제시하고, 이 성질이 오류 전파와 misaligned replay라는 두 가지 실패 모드로 이어질 수 있음을 검증하고자 한다.

특히 궤적 평가자(trajectory evaluator)의 신뢰도가 메모리 추가와 삭제 모두에서 결정적이라고 보고, 소량의 300개 궤적으로 파인튜닝된 평가자가 대형 LLM 평가자를 능가할 수 있다는 실용적 가설을 탐구한다.

미래 태스크의 실행 품질을 저장된 메모리 레코드의 "무료 품질 라벨"로 재활용할 수 있다는 아이디어를 통해, 인간 감독 없이도 misaligned 레코드를 식별·제거할 수 있는 저비용 메커니즘을 제안하고자 한다.

또한 태스크 분포 변화나 메모리 용량 제약 같은 현실적 운영 조건에서 단순한 추가/삭제만으로도 안정적 장기 성능을 유지할 수 있는지 검증함으로써, 실전 에이전트 설계에 직접 활용 가능한 가이드라인을 제시하고자 한다.

## Method

1. **합성 에이전트 RegAgent 설계**: 선형 함수 w⊤x 회귀 태스크를 수행하는 통제 가능한 에이전트로, 입력 벡터 x와 인접 데모를 받아 출력을 예측하며 메모리 노이즈 수준과 오차를 직접 측정할 수 있다.

2. **실제 에이전트 3종 선정**: EHRAgent(의료 기록 질의응답), AgentDriver(자율 주행), CIC-IoT Agent(IoT 침입 탐지)를 사용하여 입출력 형식과 검색 메커니즘의 다양성을 확보한다.

3. **백본 LLM 구성**: 주 실험에는 GPT-4o-mini를 사용하고, 일반화 검증을 위해 GPT-4o와 DeepSeek-V3를 추가 백본으로 활용한다.

4. **네 가지 메모리 추가 전략 비교**: (a) 고정 메모리 πfixed(추가 없음), (b) 전부 추가 πall, (c) 자동 평가자 기반 선택적 추가 Coarse C1/C2/C3, (d) 인간/오라클 평가 기반 Strict 추가.

5. **Coarse 평가자 설계**: RegAgent에서는 절대 오차 임계값 1.6/1.4/1.2를 C1/C2/C3에 대응시키고, 실제 에이전트에서는 GPT-4o-mini(C1), GPT-4.1-mini(C2), 300개 정답 판정으로 파인튜닝한 GPT-4.1-mini(C3)를 사용한다.

6. **Strict 평가자 시뮬레이션**: 실제 인간 피드백이 불가능하므로 생성 출력과 ground truth를 비교하여 오라클 judge를 대체하며, π_human(q,e)를 구현한다.

7. **경험 추종 측정 파이프라인**: 각 쿼리 q와 검색된 메모리 레코드 사이의 입력 유사도·출력 유사도를 임베딩 코사인 유사도로 계산하고, 테스트 스트림 전체에 대한 누적 평균을 추적하여 장기 추세를 본다.

8. **오류 전파 실험 설계**: 동일한 검색 레코드를 사용하되 LLM 출력을 ground truth로 대체한 error-free(EF) 변형을 만들어, 실제 실행과 EF 간 성능 격차를 시간축에서 비교한다.

9. **주기적 삭제 ϕ_per**: 기간 [t', t] 내 검색 횟수 증가량이 임계 α 이하인 레코드를 삭제하여, 메모리 크기를 M ≤ α(t−t')K 경계로 제한한다.

10. **이력 기반 삭제 ϕ_hist**: 최소 n회 이상 검색된 레코드 중, 다운스트림 실행 유틸리티 Φ의 평균이 임계 β 이하인 경우 삭제하며, Φ는 addition에 사용된 동일 평가자를 재활용한다.

11. **결합 삭제 ϕ_comb**: 주기적 삭제와 이력 기반 삭제의 OR 결합으로 ϕ_comb = ϕ_per ∨ ϕ_hist를 적용하여 메모리 크기와 품질을 동시에 최적화한다.

12. **태스크 분포 변화 시나리오**: EHRAgent와 AgentDriver의 테스트셋 입력 벡터를 재집계·재정렬하여 중간 시점에 태스크 분포가 급변하는 상황을 합성한다.

13. **메모리 자원 제약 시나리오**: EHRAgent는 100개, AgentDriver는 180개로 용량을 고정하고, 초과 시 최저 평균 유틸리티 레코드만 제거하는 수정 결합 정책을 적용한다.

14. **Misalignment 분석 기법**: RegAgent에서 삭제된 레코드와 유지된 레코드의 절대 오차 분포를 KDE로 비교하여, 이력 기반 삭제가 노이즈·misaligned 레코드를 체계적으로 걸러내는지 검증한다.

15. **평가 메트릭**: RegAgent는 예측 정확도(SR), EHRAgent/CIC-IoT는 ACC, AgentDriver는 성공률(SR)을 사용하며, 메모리 크기는 최종 레코드 수로 보고한다.

## Key Contribution

1. **Experience-Following Property의 정량적 규명**: LLM 에이전트가 메모리에서 검색한 레코드의 입력 유사도가 높을수록 출력 유사도도 비례하여 증가함을 RegAgent에서 Pearson r ≈ 1로 입증하고, 4개 에이전트·3종 백본에서 일관되게 확인하였다.

2. **두 가지 실패 모드의 식별과 분리**: 경험 추종으로부터 파생되는 (i) 오류 전파(노이즈 실행이 향후 태스크로 누적 전파)와 (ii) misaligned experience replay(품질 필터 통과 레코드가 현재 태스크에 부적합해 성능 저하 유발)을 독립 현상으로 규정하였다.

3. **궤적 평가자의 결정적 역할 실증**: 메모리 추가와 삭제 모두에서 평가자의 신뢰도가 장기 성능을 좌우함을 보이고, vanilla LLM judge보다 300개 데이터로 파인튜닝한 소형 C3 평가자가 더 효과적일 수 있음을 실험으로 증명하였다.

4. **이력 기반 삭제 전략의 제안**: 미래 태스크의 실행 유틸리티를 "무료 품질 라벨"로 재활용하여 misaligned 레코드를 제거하는 ϕ_hist 기법을 제시하고, strict 평가자와 결합 시 추가만 수행한 baseline보다 성능과 메모리 효율을 동시에 개선함을 보였다.

5. **결합 삭제 기반 메모리 압축 프로토콜**: 주기적 + 이력 기반 삭제의 OR 결합으로 EHRAgent 메모리를 1012 → 248(약 75% 감소)까지 압축하면서도 성능을 오히려 향상시키는 실전형 장기 운영 전략을 확립하였다.

6. **도전적 시나리오에서의 견고성 입증**: 태스크 분포 변화 환경에서도 strict 추가만으로 AgentDriver 성능이 no-shift 변형을 능가할 수 있으며, 자원 제약 환경에서도 고정 용량 대비 선택적 유지가 성능을 개선함을 실증하였다.

7. **범용 에이전트 메모리 설계 원리 도출**: 특정 에이전트에 맞춤화된 기존 기법들과 달리 추가·삭제라는 기본 연산 수준에서 분석하여 RegAgent/EHRAgent/AgentDriver/CIC-IoT로 검증된 일반화 가능한 설계 원리를 제공하였다.

## Experiment

- **Add-all vs Fixed**: 무분별 추가는 4개 에이전트 모두에서 고정 메모리보다 성능이 하락하여, RegAgent 55.48% vs 67.53%, EHRAgent 13.05% vs 16.75%, AgentDriver 32.32% vs 40.11%, CIC-IoT 59.90% vs 71.50%의 역전 현상이 관찰되었다.

- **Strict 평가자의 우월성**: 엄격한 선택적 추가는 전 에이전트에서 최고 성능을 달성하여 RegAgent 70.95%, EHRAgent 38.50%, AgentDriver 51.00%, CIC-IoT 85.40%를 기록하고, 메모리 크기도 add-all 대비 약 30% 수준으로 유지되었다.

- **Coarse C3(파인튜닝)의 효과**: 300개 판정 데이터로 파인튜닝한 C3가 다른 coarse 평가자를 일관되게 능가하여 EHRAgent에서 C1 26.19% → C3 34.66%, AgentDriver에서 C1 36.92% → C3 47.37%, CIC-IoT에서 C1 74.00% → C3 79.50%로 향상되었다.

- **Experience-Following 상관계수**: RegAgent에서 메모리 확장 시 입출력 유사도 간 Pearson r ≈ 1의 근완전 상관이 관찰되었고, 고정 메모리 baseline에서는 입출력 유사도가 모두 낮게 유지되어 에이전트가 모방 대신 추론에 의존함이 확인되었다.

- **오류 전파 가시화**: RegAgent와 AgentDriver에서 add-all 및 coarse 전략은 error-free 변형과의 성능 격차가 시간이 갈수록 벌어졌고, AgentDriver의 strict 전략은 약 2000회 실행 이후 ground-truth baseline을 오히려 초과하였다.

- **이력 기반 삭제 + Strict**: EHRAgent에서 no-deletion 38.67% → history 42.06%, combined 42.34%로 향상되었고, CIC-IoT에서는 85.40% → 89.60%(history)로 증가하였다.

- **결합 삭제의 압축 효과**: Strict + combined 삭제 시 EHRAgent 메모리가 1012 → 248(약 75.5% 감소), AgentDriver 1178 → 323, CIC-IoT 904 → 188으로 대폭 축소되면서 성능 저하 없이 유지되었다.

- **주기적 삭제 단독 효과**: Coarse C1 + periodic에서 EHRAgent 메모리 1447 → 338로 약 77% 축소되면서 성능은 25.91% → 26.65%로 소폭 개선되어, 추가만 한 설정에 중복 레코드가 다수 누적됨을 시사하였다.

- **KDE 품질 분석**: RegAgent에서 이력 기반 삭제로 제거된 레코드는 유지된 레코드보다 절대 오차 분포가 명확히 더 넓고 큰 값에 치우쳤으며, error > 0.5인 레코드가 subsequent 태스크에 오차를 전파함이 검증되었다.

- **태스크 분포 변화**: AgentDriver에서 strict 추가만으로도 no-shift 변형 성능을 초과하였고, EHRAgent에서는 combined 삭제가 history 단독보다 안정적 성능을 보였다.

- **자원 제약**: 초기 용량 고정(EHRAgent 100, AgentDriver 180) 하에서도 선택적 유지 전략이 fixed-memory 변형을 유의미하게 상회하였고, 용량이 커질수록 strict 평가자 하에서 성능이 수렴함이 관찰되었다.

- **백본 일반화**: GPT-4o-mini 외에도 GPT-4o, DeepSeek-V3 백본에서 strict 추가 시 동일한 자기 개선 추세와 experience-following 패턴이 재현되었다.

## Limitation

본 연구는 메모리 관리의 범위를 추가와 삭제라는 두 가지 기본 연산에 한정하였으며, 구조적 변환, 병합, 요약, 반성 같은 고급 메커니즘은 의도적으로 제외하였다.

따라서 이러한 고급 업데이트 기법을 통합하거나 본 논문이 다루지 않은 에이전트 아키텍처에 결론을 확장하려면, 추가적인 세부 분석과 재검증이 필요하다.

실험 결과는 광범위한 경험적 분석에 기반하지만 형식적 이론 보장(theoretical guarantee)을 제공하지 못하며, 에이전트 시스템의 본질적 복잡성으로 인해 이러한 증명은 현재로선 실현 난이도가 높다.

인간 평가자(strict evaluator)는 실제 인간 피드백 대신 ground truth 비교로 시뮬레이션되었기 때문에, 실제 운영 환경에서 인간 감독을 수집할 때의 비용, 지연, 확장성 문제는 본 논문에서 다뤄지지 않았다.

실험에 사용된 에이전트가 합성 RegAgent를 포함하여 총 4종으로 한정되어 있어, 멀티모달 에이전트, 장기 대화 에이전트, 로보틱스 에이전트 등 더 다양한 도메인에서의 검증이 후속 연구로 남아 있다.

미래 태스크 유틸리티를 품질 라벨로 재활용하는 아이디어는 유틸리티 평가자 자체의 편향에 취약하며, 본 논문에서도 vanilla LLM judge를 사용할 때 AgentDriver에서 성능 저하가 관찰되어 평가자 신뢰도 확보가 선결 과제로 남아 있다.

태스크 분포 변화 시나리오가 테스트셋을 재정렬하는 합성 방식으로 구성되어 실제 분포 변화의 모든 양상을 포괄하지 못하며, 분포 변화 속도나 카테고리 전환이 급격한 상황에서의 견고성은 추가 검증이 필요하다.

코드는 공개되었으나 memory 뱅크 초기화, 평가자 파인튜닝 데이터 구성 등 일부 하이퍼파라미터는 에이전트별로 수작업 조정되어 있어, 새로운 에이전트에 적용할 때 튜닝 부담이 존재한다.
