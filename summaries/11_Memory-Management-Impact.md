# How Memory Management Impacts LLM Agents: An Empirical Study of Experience-Following Behavior

> **논문 정보**: Zidi Xiong, Yuping Lin, Wenya Xie, Pengfei He, Zirui Liu, Jiliang Tang, Himabindu Lakkaraju, Zhen Xiang (Harvard University, University of Georgia, Michigan State University, University of Minnesota)
> **arXiv**: 2505.16067v2 (2025.10) | **학회**: -
> **코드**: https://github.com/yuplin2333/agent_memory_manage.git

## Problem

LLM 에이전트의 에피소딕 메모리 모듈은 과거 실행 경험을 저장하고 검색하여 미래 태스크 성능을 향상시키는 핵심 구성요소이다. 그러나 기존 메모리 관리 전략들은 특정 에이전트나 태스크에 맞춤 설계되어 있어, 메모리 뱅크의 동적 변화가 장기적 에이전트 성능에 미치는 근본적 영향에 대한 통합적 이해가 부족하다. 특히, 메모리에 잘못된 실행 결과가 추가되면 이후 태스크에서 오류가 전파(error propagation)되어 성능이 점진적으로 악화될 수 있다. 또한, 초기 평가를 통과한 메모리 레코드라도 현재 태스크와 정렬되지 않아 오히려 해로운 시연(misaligned experience replay)으로 작용할 수 있다. 무분별한 메모리 추가(add-all) 방식은 고정 메모리 대비 오히려 성능을 저하시키는 결과를 초래하며, 메모리 품질 관리의 부재가 장기적 에이전트 운영의 핵심 병목이 되고 있다.

---

## Motivation

기존 연구들은 구조적 변환, 병합, 요약, 반성 등 다양한 메모리 관리 전략을 제안했지만, 이들은 특정 에이전트 유형(예: 챗봇)에 한정되어 범용적 원리를 제공하지 못한다. 메모리 추가와 삭제라는 가장 기본적인 두 연산이 장기 성능에 미치는 영향을 체계적으로 분석한 연구가 부재하다. 저자들은 메모리 뱅크가 시간에 따라 진화하면서 에이전트 자체가 생성한 노이즈가 포함된 궤적을 축적한다는 점에 주목한다. 이러한 동적이고 노이즈가 있는 검색 풀의 특성을 이해하면, 보다 견고한 장기 메모리 관리 설계 원칙을 도출할 수 있다. 특히, 궤적 평가자(trajectory evaluator)의 역할이 메모리 추가와 삭제 모두에서 결정적이라는 가설을 검증하고자 하며, 미래 태스크 평가 결과를 메모리 품질 라벨로 활용할 수 있다는 실용적 통찰을 제공하고자 한다.

## Method

### 1. 메모리 추가(Addition) 실험 설계
- 4개의 에이전트 설정: 합성 에이전트(RegAgent, 선형 함수 회귀), 실제 에이전트 3종(EHRAgent, AgentDriver, CIC-IoT Agent)
- 백본 LLM: GPT-4o-mini (추가 실험으로 GPT-4o, DeepSeek-V3)
- 4가지 추가 전략: (a) 고정 메모리(πfixed), (b) 전부 추가(πall), (c) 자동 평가 기반 선택적 추가(Coarse C1/C2/C3), (d) 인간 평가 기반 엄격한 추가(Strict)

### 2. Experience-Following Property 분석
- 각 태스크 쿼리와 검색된 메모리 레코드 간 입력/출력 유사도를 측정
- 입출력 유사도 간 강한 상관관계(Pearson r ≈ 1)를 확인

### 3. 오류 전파(Error Propagation) 실험
- 각 추가 전략에 대해 LLM 출력을 정답으로 대체한 error-free 변형과 비교
- 실제 실행과 error-free 변형 간 성능 격차를 시간에 따라 추적

### 4. 메모리 삭제(Deletion) 전략
- (a) 주기적 삭제: 일정 기간 내 검색 빈도가 임계값 α 이하인 레코드 삭제
- (b) 이력 기반 삭제: n회 이상 검색된 레코드 중, 다운스트림 실행 평균 유틸리티가 임계값 β 이하인 레코드 삭제
- (c) 결합 삭제: 주기적 + 이력 기반 삭제 병행

### 5. 도전적 시나리오
- 태스크 분포 변화(Task Distribution Shift): 테스트셋 입력 재정렬
- 메모리 자원 제약(Resource Constraint): 메모리 용량 고정, 초과 시 최저 유틸리티 레코드 제거

## Key Contribution

1. **Experience-Following Property 발견**: LLM 에이전트가 메모리에서 검색한 경험의 입력 유사도가 높을수록 출력도 유사하게 따라하는 현상을 정량적으로 규명.
2. **두 가지 핵심 과제 식별**: (1) 오류 전파 — 잘못된 메모리가 미래 태스크에 오류를 누적·전파, (2) 정렬되지 않은 경험 재생 — 품질 필터를 통과했으나 현재 태스크에 부적합한 시연이 성능 저하 유발.
3. **궤적 평가자의 결정적 역할 입증**: 메모리 추가와 삭제 모두에서 평가자의 신뢰도가 장기 성능을 좌우함을 실험적으로 증명.
4. **이력 기반 삭제 전략 제안**: 미래 태스크의 실행 품질을 무료 품질 라벨로 활용하여 misaligned 레코드를 제거하는 실용적 방법 제시.
5. **분포 변화 및 자원 제약 하에서도 효과적인 메모리 관리**가 장기 성능을 유지·향상시킬 수 있음을 입증.

## Experiment & Results

- **Add-all vs Fixed Memory**: 무분별 추가는 4개 에이전트 모두에서 고정 메모리 대비 성능 하락. RegAgent 55.48% vs 67.53%, EHRAgent 13.05% vs 16.75%, AgentDriver 32.32% vs 40.11%, CIC-IoT 59.90% vs 71.50%.
- **Strict 평가자의 우월성**: 엄격한 선택적 추가는 모든 에이전트에서 최고 성능. RegAgent 70.95%, EHRAgent 38.50%, AgentDriver 51.00%, CIC-IoT 85.40%.
- **파인튜닝 평가자(C3)의 효과**: 300개 데이터로 파인튜닝한 C3가 다른 coarse 평가자를 일관되게 능가. EHRAgent에서 C1(26.19%) → C3(34.66%), AgentDriver에서 C1(36.92%) → C3(47.37%).
- **이력 기반 삭제 + Strict 평가자**: EHRAgent에서 삭제 없이 38.67% → 이력 삭제 42.06%, CIC-IoT에서 85.40% → 89.60%로 향상.
- **결합 삭제의 메모리 효율**: Strict + 결합 삭제 시 EHRAgent 메모리 1012 → 248개로 75.5% 감소하면서 성능 42.34% 유지(삭제 없이 38.67% 대비 오히려 향상).
- **Experience-Following**: RegAgent에서 메모리 확장 시 입출력 유사도 간 Pearson r ≈ 1의 근완전 상관 확인.
- **분포 변화 시나리오**: AgentDriver에서 strict 추가만으로도 분포 변화 없는 변형의 성능을 초과.
- **자원 제약**: 고정 용량에서도 선택적 유지 전략으로 고정 메모리 대비 높은 성능 달성.

## Limitation

- 메모리 관리의 범위가 추가와 삭제라는 두 가지 기본 연산에 한정되어 있으며, 구조적 변환, 병합, 요약, 반성 등 고급 메커니즘은 다루지 않았다.
- 따라서 고급 업데이트 메커니즘이나 다른 에이전트 아키텍처를 사용하는 시스템에 일반화할 때 추가적인 세밀한 분석이 필요하다.
- 실험 결과는 광범위한 경험적 분석에 기반하지만, 형식적인 이론적 보장(theoretical guarantee)을 제공하지 못한다.
- 인간 평가(strict evaluator)를 시뮬레이션을 통해 대체했으며, 실제 운영 환경에서의 인간 피드백 수집 비용과 확장성 문제는 논의되지 않았다.
- 실험에 사용된 에이전트가 4종으로 한정되어 있어, 더 다양한 도메인과 에이전트 구조에서의 검증이 필요하다.
