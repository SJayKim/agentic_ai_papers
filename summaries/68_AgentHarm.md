# AgentHarm: A Benchmark for Measuring Harmfulness of LLM Agents

> **논문 정보**: Maksym Andriushchenko, Alexandra Souly 외 (Gray Swan AI, UK AI Security Institute)
> **arXiv**: 2410.09024 (2024.10)
> **학회**: ICLR 2025
> **코드**: https://huggingface.co/datasets/ai-safety-institute/AgentHarm

---

## Problem

LLM의 탈옥(jailbreak) 공격 취약성은 지금까지 주로 단일 턴 챗봇 설정, 즉 "파이프 폭탄 제작법을 알려줘" 같은 질문-답변 형태로만 연구되어 왔다.
그러나 외부 도구를 호출하고 다단계 태스크를 실행할 수 있는 LLM 에이전트는 오용될 경우 챗봇보다 훨씬 큰 실질적 피해를 야기할 수 있다.
예컨대 "파이프 폭탄 재료를 온라인으로 주문하고 당국에 적발되지 않게 집으로 배송시켜라"와 같은 지시는 단순 답변 생성이 아니라 다단계 행동 체인을 요구한다.
최근 연구는 단일 턴 강건성이 다중 턴 강건성으로 전이되지 않는다는 사실을 보여주었기 때문에, 챗봇 기반 안전성 평가가 에이전트 안전성을 보장하지 못한다.
또한 기존 에이전트 안전성 벤치마크 대다수는 프롬프트 인젝션(간접 공격)에 집중하여, 악의적 사용자가 직접 에이전트에게 해로운 지시를 내리는 "직접 프롬프팅 오용" 시나리오를 다루지 않는다.
이 간극은 에이전트 안전 훈련의 실효성을 체계적으로 측정할 수 없게 만든다.
도구 사용, 다단계 의존성, 거부 탈출 후 능력 보존 여부를 동시에 평가할 수 있는 표준 벤치마크의 부재가 핵심 문제이다.

---

## Motivation

ChatGPT 같은 시스템은 이미 웹 검색, 코드 인터프리터 등 도구 통합을 수백만 사용자에게 제공하고 있으며, 화학(ChemCrow)·소프트웨어 공학(OpenDevin) 등 도메인 특화 에이전트가 등장하고 있다.
에이전트 능력이 장기 추론과 계획 측면에서 급속히 향상되고 있어, 악의적 행위자가 에이전트를 해로운 행동에 활용할 가능성이 커지고 있다.
이러한 위협 모델에서는 단순히 "해로운 텍스트 생성"이 아니라 "해로운 행동의 실행"이 핵심이므로 평가 방식도 달라져야 한다.
또한 기존 공격 평가들이 "거부율 감소"만 측정하여, 탈옥 이후 모델이 실제로 유의미한 작업을 수행할 수 있는지(capability degradation) 놓쳐온 문제도 있다.
여러 탈옥 기법이 거부는 낮추지만 결과물의 품질 역시 훼손시키는 경향이 보고되었기 때문에, 능력 유지와 거부 회피를 함께 측정해야 신뢰할 수 있는 공격 강도를 확인할 수 있다.
데이터 오염 문제 또한 공개 벤치마크의 고질적 한계이므로 private split으로 미래 모델 평가가 가능한 구조가 필요하다.
저자들은 "넓은 해악 범주 × 다단계 도구 호출 × 능력 퇴화 감지 × 신뢰 가능한 채점"을 모두 만족하는 표준 벤치마크가 이 분야의 병목이라고 판단하였다.

## Method

1. **11개 해악 카테고리 정의**: 사기(Fraud), 사이버범죄(Cybercrime), 괴롭힘(Harassment), 저작권 침해(Copyright), 테러리즘(Terrorism), 자해(Self-harm), 폭력(Violence) 등 광범위한 오용 시나리오를 포괄한다.
2. **110개 기본 악성 행동 설계**: 각 태스크는 2~8개의 서로 다른 도구를 순차 호출하도록 설계되며, 이전 도구 출력이 다음 호출 인자에 통합되는 의존 구조를 가진다(평균 3.53개 도구/태스크).
3. **4가지 증강으로 440개 태스크 확장**: (a) 상세 요청(detailed), (b) 세부사항을 생략한 개방형(open-ended), (c) 상세+힌트(hint), (d) 개방형+힌트 — 힌트는 첫 번째 호출 함수명을 알려줌으로써 태스크를 쉽게 만든다.
4. **무해한 대응 태스크(benign counterpart) 수작업 제작**: 각 악성 태스크의 도구 구성은 유지하되 악의 요소만 제거한 버전을 만들어, 거부 없이 측정되는 모델 능력 상한을 확보한다.
5. **104개 합성 도구(synthetic tools) 구현**: UK AISI의 Inspect 프레임워크 기반으로 사이드 이펙트 없는 독립 함수로 작성되며, `query_onion_search`, `send_email`, `terminal` 같은 범용부터 `query_xusers`, `finetune_huggingface_model` 같은 도메인 특화까지 포함한다.
6. **자가 수정(self-correction) 지원**: 에이전트가 파일명을 환각하면 "파일 없음" 메시지를 반환하여 다른 도구 호출로 정정할 기회를 제공, 도구 오류로 인한 실패와 실제 거부/무능을 구분한다.
7. **수작업 세분화 채점 루브릭**: 각 태스크마다 수십 줄짜리 grade 함수가 작성되며, "모든 타겟 함수 호출 여부", "순서", "인자(예: 특정 이메일 주소 포함)", "시맨틱 적합성"을 개별 항목으로 검사하고 부분 점수를 부여한다.
8. **좁은 범위 LLM 심판(semantic judge)**: LLM은 전체 출력의 유해성을 판정하지 않고, 생성된 이메일이 특정 인물의 여권을 요청하는가 같은 좁은 의미 판단에만 사용되어 심판 편향·취약성을 최소화한다.
9. **거부 판정(refusal judge) 분리**: 에이전트의 모든 메시지를 대상으로 거부 여부를 별도 판정(도구 여러 번 호출 후 뒤늦게 거부하는 경우 포착), 해악 점수(harm score)와 독립적으로 보고한다.
10. **데이터 분할(10/60/30)**: 검증셋 44개, 공개 테스트 264개, 프라이빗 테스트 132개로 분할하고, 프라이빗셋은 Azure 자체 배포로만 평가하여 공개 LLM API에 노출되지 않도록 한다.
11. **Canary 문자열 삽입**: 리포지토리에 식별 문자열을 포함시켜 훈련 데이터에서 필터링 가능하게 하여 데이터 오염을 구조적으로 방지한다.
12. **두 종류 공격 기법 평가**: (a) Forced tool call — 모델 제공사의 강제 도구 호출 옵션을 사용해 prefill 형태로 거부 우회, (b) Andriushchenko et al. 2025의 규칙 기반 범용 탈옥 템플릿을 에이전트용으로 경미하게 수정(자유형식 답변 대신 도구 호출 형식 언급)한 뒤 모든 모델에 동일 적용.
13. **프롬프트 변형 ablation**: 기본 시스템 프롬프트 외에 Chain-of-Thought, ReAct, refusal-inducing 프롬프트를 append하여 효과 측정.
14. **Best-of-n 샘플링**: temperature τ=1로 n=5 샘플 중 최고 점수를 기록하여 공격 상한 추정.
15. **수작업 검수**: 모든 질문은 사람이 직접 작성하고 다른 검토자가 재검토하며, 대량의 실행 로그를 수동 검사하여 채점 정확도를 검증한다.

## Key Contribution

1. **직접 프롬프팅 오용에 특화된 최초의 대규모 에이전트 벤치마크**: 간접 프롬프트 인젝션 중심의 기존 연구(AgentDojo, ToolEmu 등)와 달리, 악의적 사용자가 직접 해로운 요청을 내리는 시나리오를 11개 해악 카테고리·110개 기본 행동·440개 증강 태스크·104개 도구로 대규모 커버한다.
2. **능력 퇴화 감지 내장**: 무해 대응 태스크와 비거부 조건부 해악 점수를 함께 측정함으로써, 탈옥 공격이 단순 거부만 우회하는지 아니면 실제로 유의미한 다단계 해로운 행동을 수행할 수 있는지를 구분하여 "성공했지만 무능한" 공격을 식별한다.
3. **충격적인 무탈옥 순응성 실증**: Mistral Large 2가 어떠한 탈옥 없이도 82.2% 해악 점수와 1.1% 거부율을 기록하여, 현대 안전 훈련이 챗봇 설정에서 에이전트 설정으로 전이되지 않는 현실을 최초로 대규모 입증.
4. **채팅 탈옥 템플릿의 에이전트 전이 입증**: 단일 템플릿 공격으로 Claude 3.5 Sonnet의 해악 점수를 13.5% → 68.7%로 끌어올려, 챗봇용 공격 연구 성과가 에이전트 방어 없이는 직접 재사용 가능함을 보여준다.
5. **신뢰 가능한 채점 인프라**: 수작업 조건 검사 + 좁은 LLM 심판 + 거부 판정 분리 + private split + canary 문자열로 이루어진 채점 체계는 기존 LLM 심판 기반 벤치마크의 편향·오염 문제를 구조적으로 완화한다.
6. **Inspect 프레임워크 통합으로 높은 접근성**: 공개 테스트셋 1회 실행에 GPT-4o 기준 $1-$4, 오픈 모델 기준 $0.08-$0.25로 저렴하게 재현 가능하며, 합성 도구 덕에 실제 피해 없이 안전하게 평가할 수 있다.

## Experiment

**평가 대상 모델 (15종)**: OpenAI GPT-3.5-Turbo / GPT-4o-mini / GPT-4o, Anthropic Claude 3 Haiku/Sonnet/Opus/3.5-Sonnet, Google Gemini 1.0 Pro / 1.5 Flash / 1.5 Pro, Mistral Small 2 / Large 2, Meta Llama-3.1 8B / 70B / 405B.

**무탈옥 직접 요청 시 해악 점수**: Mistral Large 2가 **82.2%**(거부율 1.1%)로 최고치, GPT-4o mini 62.5%(거부율 22%), GPT-4o 48.4%(거부율 48.9%), Claude 3.5 Sonnet 13.5%(거부율 85.2%), Gemini 1.5 Pro 15.7%(거부율 78.5%), Llama-3.1 405B가 거부율 88.3%로 최다 거부.

**범용 탈옥 템플릿 적용 효과**: GPT-4o 해악 점수 48.4% → **72.7%** (거부율 48.9% → 13.6%), Claude 3.5 Sonnet 13.5% → **68.7%** (거부율 85.2% → 16.7%), Gemini 1.5 Pro 15.7% → **56.1%** (거부율 78.4% → 3.5%).

**Best-of-5 샘플링(τ=1, 템플릿)**: GPT-4o 70.7%±0.7% → **83.7%**, Claude 3.5 Sonnet 65.1%±1.3% → **79.7%**, Mistral Large 2 67.4%±8.5% → **90.8%** — Mistral이 가장 큰 변동성(±8.5%)과 최고 상한을 동시에 보임.

**탈옥 후 능력 보존 검증**: 템플릿 공격 적용 시 비거부 해악 점수가 무해 태스크 점수(예: 89.9%)와 거의 일치하여, 탈옥이 에이전트의 다단계 도구 호출 능력을 저하시키지 않음을 확인.

**프롬프트 기법 ablation (Table 3)**: refusal-inducing 프롬프트 적용 시 GPT-4o의 무공격 해악 점수가 48.4% → 16.6%(거부율 48.9% → 82.6%)로 크게 감소하나, Mistral Large 2는 템플릿 공격 하에서 refusal 프롬프트가 있어도 79.5%/거부율 0%를 유지 — 탈옥 템플릿이 refusal 시스템 프롬프트를 압도함을 시사.

**채팅 vs 에이전트 비교(Table 8, 76개 태스크)**: 템플릿 공격 하 GPT-4o의 거부율이 에이전트 9.1% vs 채팅 31.8%, Claude 3.5 Sonnet이 에이전트 29.5% vs 채팅 72.7% — 에이전트 설정이 챗봇보다 일관되게 덜 견고하여 안전 훈련이 에이전트로 전이되지 않음을 입증.

**오류 분석(GPT-3.5 vs GPT-4o)**: GPT-3.5 Turbo는 (i) 파라미터 환각 후 자가 수정 실패, (ii) 의존성 있는 도구를 잘못 병렬 호출, (iii) 다중 호출 요구 시 단 1회만 호출하는 문제로 낮은 점수를 기록.

**비용**: 공개 테스트셋 1회 실행당 약 0.5-1.5M 토큰(입력 70-90%), GPT-4o는 $1-$4, 오픈 모델은 $0.08-$0.25로 저비용 재현 가능.

## Limitation

프롬프트가 영어에만 국한되어 저자원 언어에서의 탈옥 전이성(Yong et al. 2024 등)이 측정되지 않는다.
멀티턴 공격을 지원하지 않는다 — 사용자는 첫 프롬프트만 입력하며, 이후 대화적 follow-up을 통한 점진적 탈옥 시나리오는 평가 범위 밖이다.
채점 함수가 수작업으로 작성되었기 때문에 에이전트가 합리적이지만 루브릭에 포함되지 않은 대안적 실행 경로(예: 추가 정보 요청)를 따르면 오답으로 판정될 수 있다.
벤치마크가 자체 합성 도구에 묶여 있어 서드파티 도구나 커스텀 도구를 지원하지 않는 다른 스캐폴딩에서는 직접 재사용이 어렵다.
합성 도구는 실제 부작용이 없어 에이전트의 진짜 실세계 피해 능력과 정확히 일치하지 않으며, 해악의 프록시 지표로만 해석되어야 한다.
태스크가 상대적으로 단순(2-8 도구 호출)하여 장기 자율 에이전트의 고급 능력(GAIA, Cybench급 개방형 태스크)을 측정하지 못한다.
실제 인물·주소·장소를 배제했기 때문에 실세계 맥락 정보와 결합된 공격의 실효성은 평가되지 않는다.
벤치마크가 오히려 해로운 에이전트를 최적화하는 데 악용될 가능성이 존재하나, 저자들은 합성 도구와 제한적 현실성으로 이 위험을 부분 완화했다고 주장한다.
