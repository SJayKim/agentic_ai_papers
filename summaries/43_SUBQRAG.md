# SUBQRAG: Sub-Question Driven Dynamic Graph RAG

> **논문 정보**: Jiaoyang Li, Junhao Ruan, Shengwei Tang, Saihan Chen, Kaiyan Chang, Yuan Ge, Tong Xiao, Jingbo Zhu (Northeastern University, NiuTrans Research)
> **arXiv**: 2510.07718 (2025.10)
> **코드**: https://github.com/ljy1228/SubQRAG

---

## Problem

기존 Graph RAG는 넓은 시야의 문서 간 사실 연결에는 강점을 보이지만, 복잡한 멀티홉 QA에 요구되는 깊이 있는 구조적 추론에는 취약하다.
특히 대부분의 Graph RAG는 여전히 **단일 단계(single-step) 검색**에 의존하여, 복잡한 질문 전체를 한 번에 그래프 진입점(entry point)으로 매핑한다.
이러한 단일 단계 검색은 추론 과정을 논리적 홉 단위로 분해하지 못하며, 각 홉마다 검색 초점을 동적으로 조정할 수 없다.
두 번째 문제는 사전 구축된 KG의 **정적이고 불완전한 특성**이다 — 구조가 고정되어 있고 커버리지가 부분적이며 세분성이 거칠어 추론 근거가 약화된다.
세 번째 문제는 **오류 누적(error accumulation)**이다 — 초기 진입 노드가 단일 holistic 질문에 의해 결정되므로, 초기 부정확성이 이후 경로 탐색과 노드 확장 전체로 전파되어 추론 체인을 붕괴시킨다.
또한 document-level 지식은 한정된 검색 윈도우를 단일 홉의 증거로 가득 채워, 전체 추론 체인을 완성하는 데 필요한 서로 다른 정보들을 밀어내는 문제를 유발한다.
LLM 자체의 파라미터 지식은 재학습 비용 때문에 실시간 갱신이 불가능하고 커버리지가 제한되어, 최신 정보 반영 실패와 환각 위험이 QA 태스크에서 커진다.

## Motivation

멀티홉 QA에서 단일 단계 검색은 하나의 지배적 토픽에 집중하여 다른 홉에 필요한 증거를 누락시키는 반면, 질문을 **검증 가능한 서브 질문 체인**으로 분해하면 각 홉을 독립적으로 검색·검증할 수 있다.
서브 질문 단위로 검색하면 검색 초점을 홉마다 동적으로 재조정할 수 있어, 단일 단계 검색의 구조적 한계를 우회할 수 있다.
사전 구축된 KG가 불완전하더라도, 해당 서브 질문이 답해지지 않는 순간 원본 문서에서 새 트리플을 추출하여 **쿼리 시점에 KG를 실시간으로 갱신**하면 정적 KG의 구조적 결함을 보완할 수 있다.
이전 서브 질문의 답변을 후속 서브 질문에 주입하는 **rewriting** 단계는 지시 대명사·맥락 공백을 제거하여 독립적으로 자기완결적인(self-contained) 질문을 만들고, 검색 정밀도와 추론 신뢰도를 높인다.
최종적으로 추론 과정에서 실제로 사용된 트리플만을 **graph memory**로 집계하면, 전체 문서를 컨텍스트에 넣는 방식보다 윈도우를 훨씬 효율적으로 사용하면서도 추적 가능한 증거 경로를 제공할 수 있다.
이러한 구조화된 증거 경로는 해석 가능성을 높이는 동시에 무방향 그래프 탐색(undirected graph traversal)에 내재된 오류 누적을 완화한다.
즉, **분해 + 동적 갱신 + 그래프 메모리**의 세 요소를 결합함으로써 "추론 깊이(depth)"와 "지식 베이스 폭(breadth)" 사이의 균형을 달성하는 것이 본 연구의 핵심 동기다.

## Method

1. **전체 파이프라인 4단계 구조**: (I) Offline Indexing → (II) Question Decomposition + Rewriting → (III) Retrieval & Dynamic Graph Updating → (IV) Answer Generation.

2. **Offline Indexing (오프라인 인덱싱)**: LLM(gpt-4o-mini)이 원본 문서를 파싱하여 엔티티와 관계를 추출하고 ⟨h, r, t⟩ 트리플로 조직화.
코퍼스 수준 KG `G = {(h, r, t) | h, t ∈ E, r ∈ R}`을 구성하며, 노드 `h, t`는 엔티티, 엣지 `r`은 두 엔티티 간의 의미적 관계를 나타낸다.
새 트리플이 이미 존재하는지 확인하여 중복을 제거하고 `G`의 간결성을 유지한다.

3. **Question Decomposition (질문 분해)**: 복잡한 멀티홉 질문 `q`를 순서화된 서브 질문 집합 `{q_1, q_2, ..., q_n}`으로 논리적으로 분해.
각 `q_i`는 이전 서브 질문들 `{q_1, ..., q_{i-1}}`의 결과에 의존할 수 있으며, 이 순차 의존성이 논리적 해소 순서를 보장한다.

4. **Question Rewriting (질문 재작성)**: `i`번째 서브 질문 `q_i`에 대해, 이미 해소된 답변 집합 `A_{<i} = {a_1, ..., a_{i-1}}`을 주입하는 변환 `q'_i = Rewrite(q_i, A_{<i})`을 적용.
이는 지시 대명사 해소 및 맥락 공백 제거를 통해 원래 의도를 보존하면서 자기완결적이고 중의성 없는 질문으로 변환하여, 검색 정밀도와 추론 신뢰도를 동시에 개선한다.
예시: 원 질문 "해리포터 저자의 남편의 직업은?" → 서브 1 "해리포터의 저자는 누구인가?" → 서브 2 "#1의 남편은 누구인가?" → 서브 3 "#2의 직업은 무엇인가?".

5. **Retrieval — 임베딩 기반 트리플 검색**: 각 서브 질문 `q_i`에 대해 임베딩 모델 `f(·)` (all-MiniLM-L6-v2)로 의미적 유사도를 계산하여 KG에서 관련 트리플을 검색.
각 검색 단계에서 상위 **top-5 트리플**을 선택하여 답변 생성의 근거로 활용한다.

6. **Dynamic Graph Updating (동적 그래프 갱신)**: 기존 KG가 현재 서브 질문에 답하기에 불충분하다고 판단되면, 원본 문서(corpus)로 폴백하여 새 트리플을 추출하고 KG에 통합.
즉, 서브 질문이 KG 갱신의 트리거 역할을 하여 "정적 KG + 동적 보완"의 하이브리드 구조를 구현한다.
갱신된 그래프는 다음 서브 질문에도 계속 활용되어, 질의 세션 동안 KG가 점진적으로 풍부해진다.

7. **Graph Memory 구성**: 모든 서브 질문에서 실제로 사용된 트리플을 누적 합집합으로 집계 — `T_used = ∪_{i=1}^{m} T_used_i`.
이 `T_used`가 "graph memory"이며, 추적 가능한 증거 경로(evidence path)로서의 역할을 한다.

8. **Answer Generation (답변 생성)**: 생성기(gpt-4o-mini)에 원래 질문 `q`와 그래프 메모리 `T_used`를 함께 입력.
document-level 입력 대신 트리플 기반 구조화된 증거를 사용하므로 컨텍스트 윈도우를 효율적으로 활용하고 더 풍부한 근거를 제공한다.

9. **오류 누적 완화 메커니즘**: 무방향 그래프 탐색에서 발생하는 오류 누적을, 서브 질문별로 독립 검증된 트리플만 집계하는 구조로 경감.
각 홉이 명시적으로 분리되어 있으므로 한 홉의 실패가 전체 추론 경로로 확산되는 것을 막는다.

10. **구현 세부사항**: 백본 LLM은 모든 단계(엔티티 인식, 지식 추출, 서브 질문 분해, 트리플 구성, 질문 재작성, 질문 응답)에서 gpt-4o-mini 사용.
검색기는 all-MiniLM-L6-v2, 실험은 단일 NVIDIA RTX 3090 GPU에서 수행.

## Key Contribution

1. **서브 질문 주도 동적 검색 프레임워크 제안**: 원래 질문을 검증 가능한 서브 질문 체인으로 분해하고 각 서브 질문마다 독립적으로 트리플을 검색함으로써, 단일 단계 검색의 한계를 극복하고 홉별로 검색 초점을 동적으로 조정할 수 있게 한다.

2. **쿼리 시점 동적 그래프 갱신(Dynamic Graph Updating)**: 사전 구축된 KG가 불충분할 때 서브 질문이 원본 문서 검색을 트리거하여 새 트리플을 추출·통합하는 메커니즘. 정적 KG의 불완전성과 coverage 한계를 실시간으로 보완한다.

3. **Graph Memory as Structured Guidance**: 각 서브 질문을 지원한 트리플들을 서브그래프로 통합하여 생성기에 제공하는 구조. 이는 추론 궤적을 구조화하여 컨텍스트 윈도우 제약을 완화하고 QA 성능을 향상시킨다.

4. **추론 깊이와 지식 베이스 폭의 균형 달성**: 멀티 스텝 추론(깊이)과 실시간 그래프 갱신(폭)을 결합하여 정적 Graph RAG의 불완전성과 오류 누적 문제를 동시에 해결하는 새로운 설계 원칙을 제시한다.

5. **3개 멀티홉 QA 벤치마크에서의 SOTA 수준 성능 입증**: HotpotQA, 2WikiMultiHopQA, MuSiQue에서 기존 HippoRAG2를 포함한 강력한 Graph RAG 베이스라인들을 일관되게 능가하며, 특히 EM 지표에서 큰 개선을 달성.

6. **추적 가능한 증거 경로(Traceable Evidence Path) 생성**: Graph Memory가 답변의 근거가 되는 트리플 경로를 명시적으로 제공하여 해석 가능성을 높이고, 멀티홉 추론의 디버깅과 검증을 용이하게 한다.

## Experiment

- **벤치마크**: HotpotQA, MuSiQue, 2WikiMultiHopQA — 세 가지 멀티홉 QA 데이터셋으로 cross-document reasoning, complex contextual understanding 등 다양한 추론 시나리오를 커버.
- **실험 프로토콜**: HippoRAG2의 프로토콜을 엄격히 따름 — 동일 검색 코퍼스 사용, 각 validation set에서 **1,000개 질문** 샘플링.
- **백본/검색기**: 모든 Graph RAG 방법은 **gpt-4o-mini** 생성기 + **all-MiniLM-L6-v2** 검색기 + **top-5 트리플** 검색 통일.
- **하드웨어**: 단일 NVIDIA RTX 3090 GPU.
- **베이스라인**: Zero-shot, 검색 기반(GTR, all-MiniLM-L6-v2), Graph RAG(RAPTOR, GraphRAG, LightRAG, HippoRAG2).
- **평가 지표**: Exact Match (EM), F1 score.

**주요 결과 (Table 1, EM / F1)**:
- **MuSiQue**: SubQRAG **29.70 / 38.14** vs HippoRAG2 28.20 / 40.70 — EM +1.5pt, Zero-shot(11.20) 대비 EM **+18.5pt**.
- **2Wiki**: SubQRAG **61.90 / 64.30** vs HippoRAG2 50.60 / 59.79 — EM **+11.3pt**, F1 +4.5pt.
- **HotpotQA**: SubQRAG **56.00 / 64.30** vs HippoRAG2 51.90 / 66.18 — EM **+4.1pt**.
- HippoRAG2 대비 상대 EM 향상: MuSiQue **+5.3%**, 2Wiki **+22.3%**, HotpotQA **+7.9%**.
- Zero-shot(gpt-4o-mini 단독)은 세 벤치마크에서 11.20/30.20/28.60 EM에 머물러, RAG의 필요성을 재확인.
- RAPTOR/GraphRAG/LightRAG은 일부 벤치마크에서 오히려 Zero-shot보다 낮은 성능(예: MuSiQue에서 RAPTOR 4.80, GraphRAG 4.69)을 보여, 단순 KG 기반 방법의 한계를 드러냄.

**Ablation (HotpotQA, Table 2)**:
- Full SubQRAG: **56.0 / 64.3**.
- w/o Decomposition: **50.5 / 59.6** (EM -5.5, F1 -4.7) — 분해 제거 시 가장 큰 타격.
- w/o Rewriting: **49.5 / 50.2** (EM -6.5, F1 **-14.1**) — 재작성 제거가 F1에서 가장 큰 하락 유발.
- w/o Update: **54.5 / 63.7** (EM -1.5, F1 -0.6) — 동적 갱신의 기여는 상대적으로 작지만 여전히 양의 기여.
- 해석: Decomposition과 Rewriting이 핵심 요인이며, graph updating은 KG가 자주 갱신되지는 않아 상대적으로 modest한 효과이나 여전히 게인을 제공.

## Limitation

저자들은 본문에서 명시적인 한계 섹션을 두지 않았으나, 암묵적으로 dynamic graph updating의 효과가 KG 자체의 초기 커버리지와 질문 분포에 따라 modest할 수 있음을 ablation에서 드러낸다.
독자 관점 한계 1 — **서브 질문 분해 품질의 LLM 의존성**: 분해·재작성 모두 gpt-4o-mini에 의존하므로, 약한 LLM에서는 잘못된 분해가 전체 파이프라인을 오염시킬 수 있다.
독자 관점 한계 2 — **쿼리 시점 그래프 갱신의 지연 비용**: 각 질의마다 원본 문서에서 새 트리플을 추출하여 KG를 갱신하는 작업은 응답 지연을 크게 증가시킬 수 있으며, production 환경에서의 latency 프로파일이 공개되지 않았다.
독자 관점 한계 3 — **순차 의존성의 병목**: 서브 질문이 선행 답변에 의존하는 순차 구조는 병렬화를 막고, 초기 서브 질문의 실패가 후속 체인 전체로 전파되는 잠재적 오류 전파 경로를 남긴다.
독자 관점 한계 4 — **평가 범위의 제약**: 세 벤치마크 모두 QA에 한정되어 있고, 각각 1,000 샘플로 제한되어 있어 장기 대화, 도메인 특화 QA, 비영어 설정 등으로의 일반화는 검증되지 않았다.
독자 관점 한계 5 — **트리플 추출의 정확성 의존성**: LLM 기반 엔티티/관계 추출이 노이즈 트리플을 생성할 경우 그래프 품질이 손상되며, 추출 품질과 KG 노이즈 상호작용은 분석되지 않았다.
독자 관점 한계 6 — **고정된 top-5 검색**: 모든 서브 질문에 동일한 top-5 트리플 검색을 적용하는데, 홉 복잡도에 따라 적응적으로 k를 조정하는 전략은 탐색되지 않았다.
독자 관점 한계 7 — **Graph Memory의 확장성**: 서브 질문 수가 많아질수록 `T_used`가 선형적으로 증가하여 장문 추론 체인에서는 다시 컨텍스트 윈도우 압박이 발생할 수 있다.
