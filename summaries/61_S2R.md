> **논문 정보**: Ruotian Ma¹, Peisong Wang², Cheng Liu¹, Xingyan Liu¹, Jiaqi Chen³, Bang Zhang¹, Xin Zhou⁴, Nan Du¹†, Jia Li⁵† (¹Tencent, ²Tsinghua, ³HKU, ⁴Fudan, ⁵HKUST-GZ)

## Problem

최근 OpenAI o1을 시작으로 LLM의 "딥 씽킹(deep thinking)" 능력을 test-time compute scaling으로 유도하려는 시도가 주류가 되었다.

그러나 기존 접근은 막대한 자원을 요구한다 — MCTS 기반 long-CoT 데이터 대량 구축, 수만~수십만 건의 고품질 human-annotated 궤적, 대규모 RL 학습이 전제되어야만 o1류 추론 능력이 발현되었다.

DeepSeek-R1이 RL만으로도 deep thinking이 가능함을 보였지만, R1 역시 수백 B급 모델과 방대한 학습 데이터를 전제로 한다.

따라서 **"작거나 약한(smaller, less powerful) base LLM에서 어떻게 효율적으로 자가 사고 능력을 키울 것인가"** 는 여전히 미해결 문제로 남아 있다.

특히 Qwen2.5-Math-7B 같은 7B급 모델은 MATH500에서 단 51.0% 정확도에 머물며, long-CoT distillation으로 끌어올리려 해도 32B급 teacher 모델(QwQ-32B-Preview)로부터 증류해야 해 teacher 의존성과 비용 문제가 발생한다.

또한 Kumar et al. (2024) 등은 SFT만으로는 LLM이 유효한 self-correction을 학습하지 못한다는 점을 보고했고, Huang et al. (2023)은 LLM이 자기 실수를 스스로 식별하는 데 실패한다는 점을 지적했다 — 즉 self-verification / self-correction 자체가 LLM의 약점이다.

이에 저자들은 작고 약한 모델에서도 최소한의 자원으로 dynamic trial-and-error를 학습시키는 효율적 프레임워크를 요구한다.

## Motivation

o1의 deep thinking을 해부하면 핵심은 **self-checking, self-verifying, self-correcting, self-exploring** 네 가지 스킬의 반복적 적용이다.

저자들은 이 중 가장 본질적인 두 스킬인 **self-verification**과 **self-correction**만 명시적으로 학습시키면 작은 모델도 문제 난이도에 맞춰 test-time compute를 유연하게 배분할 수 있다는 가설을 세운다.

즉 "larger teacher 모델의 long-CoT를 흉내 내기"가 아니라, "trial-and-error → verify → correct → trial-and-error ..." 의 반복 구조 자체를 모델에 내재화한다는 철학이다.

이 구조는 세 가지 장점을 가진다 — (1) 쉬운 문제는 1 trial로 종료, 어려운 문제는 여러 trial로 확장되는 **dynamic compute allocation**, (2) rule-based reward만으로도 학습 가능해 **PRM/ORM 학습 비용 제거**, (3) 수학 도메인에서 학습해도 logical/code reasoning에 일반화.

저자들은 또한 "verification을 어떻게 구성하는가"가 결정적이라는 점을 관찰했다 — 기존 LLM들은 "verify without re-solving"하라고 prompt해도 문제를 다시 푸는 방식(**Problem-Solving verification**)으로 회귀하는 reversal curse (Berglund et al., 2023)를 보인다.

이에 "답을 원식에 대입해 확인"하는 **Confirmative verification**을 설계해 정답 편향을 줄이는 방향으로 나아간다.

## Method

S²R은 3단계 파이프라인으로 작동한다: **Stage 0 (데이터 구축) → Stage 1 (Behavior Initialization via SFT) → Stage 2 (RL 강화)**.

1. **Action Space 정의**: 정책 π는 문제 x에 대해 세 종류 action만 생성한다 — `solve` (해답 시도), `verify` (직전 solve 평가), `<end>` (종료).
2. **Deterministic action transition**: `solve` 다음엔 반드시 `verify`가 오고, `verify` 결과가 INCORRECT면 다음 action은 `solve`, CORRECT면 `<end>`로 전이된다. 이는 regex 기반 Parser가 verify 텍스트를 binary 판단으로 변환해 결정된다.
3. **Trajectory 표기**: y = (s₁, v₁, s₂, v₂, ..., s_k, v_k, <end>) — solve와 verify가 k번 반복된 trial-and-error 궤적이다.
4. **Verification 구성 (Stage 0, 핵심)**: 두 방식을 비교 — "Problem-Solving" (문제를 재차 풀어 답 확인) vs "Confirmative" (답을 원식에 대입해 검증). Confirmative는 LLM-as-a-judge로 invalid를 필터링하고, 모든 실험에서 Confirmative를 채택한다.
5. **Self-correction 데이터 구성**: 여러 개의 실패한 solution (각각 오류를 인지한 verification이 뒤따름) + 최종 성공 solution을 concatenate해 correction sequence를 만든다. 이는 Kumar et al. (2024), Snell et al. (2024b) 패턴을 따른다.
6. **Dynamic trial-and-error 궤적**: 세 원칙에 따라 구성한다 — (a) k∈{1,2,3,4}로 길이 다양화, (b) 실패 trial은 정책 자신의 응답에서 샘플링 (자기 오류 학습), (c) 문제 난이도별로 trial 길이 조정 (어려운 문제 → 긴 궤적).
7. **Stage 1 SFT masking**: loss에서 `verify` action 전부 + 마지막 `solve` (정답) + `<end>`만 supervise, 중간의 틀린 `solve`는 학습 신호에서 mask out. 이는 실패한 trial을 흉내 내지 않으면서 "실패를 인지하고 고치는" 메타 행동만 학습하게 한다.
8. **Stage 2 Outcome-level RLOO**: 최종 solution s_T의 정답 여부로 trajectory 전체에 ±1 reward. REINFORCE Leave-One-Out baseline (M개 샘플 중 자신 제외 평균)으로 advantage 추정, KL 정규화와 PPO 스타일 clip을 결합해 최적화한다 — **"모델이 자유롭게 trial-and-error 경로 탐색"** 을 유도.
9. **Stage 2 Process-level Group-based RL (PRL)**: 각 action을 sub-process로 간주. `solve` action은 해당 중간 정답성으로, `verify` action은 Parser(v_j)가 V_golden(s_j)와 일치하는지로 ±1 reward. Baseline은 **reward context를 공유하는 action 그룹의 평균 reward**로 추정한다 — 예컨대 "1회 실패 + 1회 verify 성공 (-1, 1) context"를 공유하는 action들만 묶어 baseline 계산.
10. **PRL의 철학**: 같은 amount of information을 받은 action들끼리만 비교해야 공정한 advantage라는 원칙. GRPO (Shao et al., 2024) 아이디어를 process-level로 확장.
11. **Rule-based reward만 사용**: 별도 PRM/ORM 학습 불필요 — V_golden은 정답 매칭, Parser는 regex로 구현.
12. **Offline RL 확장 (§2.4)**: online 샘플링 비용을 없애기 위해 사전 샘플된 trajectory로 offline RL 수행. 대규모 샘플링이 가능해 baseline 추정 정확도가 올라가며, 특히 PRL에서 이점이 크다.
13. **데이터 규모**: Qwen2.5-Math-7B는 Stage 1에 **3.1k** trial-and-error 궤적, Llama3.1-8B는 4614개, Qwen2-7B는 4366개 사용. Stage 2 RL은 약 10k instance로 수행 (너무 쉽거나 어려운 문제는 filter out).
14. **Difficulty-aware sampling**: 문제당 5 response를 샘플링, 모델의 pass rate로 난이도 추정 후 궤적 길이를 할당한다.

## Key Contribution

1. **최소 데이터로 거대 성능 증폭**: Qwen2.5-Math-7B에서 단 **3.1k** 초기화 샘플 + 10k RL 데이터로 MATH500 정확도를 **51.0% → 81.6%** (BI) → **84.4%** (ORL) 까지 끌어올렸다. 같은 규모의 long-CoT distill (QwQ-32B-Preview 기반, 80.2%)보다 우수하다.
2. **Self-verify/Self-correct as first-class skill**: deep thinking을 모호한 "long CoT" 덩어리가 아닌, `solve/verify/<end>` 3-action FSM으로 분해해 해석 가능하고 학습 가능한 skill로 재정의했다.
3. **Confirmative vs Problem-Solving verification 규명**: LLM이 verify prompt에도 문제를 다시 푸는 경향(reversal curse)을 보이며, confirmative 방식이 정답 편향이 적고 balanced한 판단을 낸다는 분석을 최초로 체계화했다.
4. **Outcome-level RL vs Process-level RL의 역할 분리**: ORL은 강한 base model(Qwen2.5-Math-7B)에서 유리 — 자유로운 trajectory exploration이 가능. PRL은 약한 base model(Qwen2-7B-Instruct)에서 중간 단계 accuracy를 끌어올려 효과적. 즉 base 모델 capability에 따라 RL 전략을 골라야 함을 데이터로 입증했다.
5. **Rule-based reward만으로 충분**: PRM/ORM 학습 없이도 강력한 결과 — Eurus-2-7B-PRIME (암묵적 PRM 학습), rStar-Math-7B (대규모 MCTS) 대비 훨씬 적은 자원으로 우수한 성능 달성.
6. **Cross-domain 일반화**: 수학만 학습했는데 FOLIO (논리), CRUXEval (code), StrategyQA (multi-hop), MMLUPro-STEM에서 일관된 개선 — self-verify/self-correct가 도메인 독립적 meta-skill임을 시사.
7. **Offline RL의 가능성 제시**: online RL의 샘플링 비용 없이 comparable한 성능. 특히 PRL + offline 조합이 accurate baseline 덕분에 online PRL을 능가하는 유효한 대안임을 보였다.

## Experiment

**평가 셋업**: 3개 base 모델 × 7개 math benchmark × Pass@1 greedy decoding, vLLM 기반 inference.

- **Base models**: Llama-3.1-8B-Instruct (general), Qwen2-7B-Instruct (general), Qwen2.5-Math-7B (math-specialized).
- **Benchmarks**: MATH500, AIME 2024, AMC 2023, College Math, Olympiad Bench, GSM8K, GaokaoEn 2023.

**핵심 수치 (Qwen2.5-Math-7B base, Table 2)**:
- Base: MATH500 **51.0**, AMC23 45.0, GSM8K 58.3, Average 35.6.
- S²R-BI (SFT only): MATH500 **81.6** (+30.6), GSM8K **91.9** (+33.6), Average 59.3.
- S²R-PRL: MATH500 83.4, AMC23 70.0, OlympiadBench 46.4, Average 62.0.
- S²R-ORL: MATH500 **84.4**, AMC23 **77.5**, Average **62.4** — 3.1k+10k 데이터로 frontier baseline 대부분 상회.

**경쟁 방법 비교 (모두 Qwen2.5-Math-7B base)**:
- Eurus-2-7B-PRIME: MATH500 79.2, AMC23 57.8 — 암묵적 PRM 학습에도 불구하고 S²R-ORL에 뒤짐.
- rStar-Math-7B: MATH500 78.4, AMC23 47.5 — 대규모 MCTS 기반인데도 AMC23에서 S²R-ORL이 +30점 우위.
- Qwen2.5-7B-SimpleRL: MATH500 82.4, OlympiadBench 43.3 — S²R-ORL이 MATH500 +2.0, OlympiadBench +1.6.
- Qwen2.5-Math-7B + Long-CoT SFT (32B teacher 증류): MATH500 80.2 — 같은 SFT scale에서 S²R-BI가 +1.4.

**Frontier 비교**:
- GPT-4o: MATH500 76.6 — S²R-ORL(84.4)이 +7.8 우위.
- GPT-o1-mini: MATH500 90.0 — 여전히 S²R-ORL보다 높으나 격차 좁혀짐.

**일반 모델에서도 효과 일관**:
- Llama-3.1-8B: MATH500 48.0 → S²R-ORL 55.0 (+7.0), Average 36.6 → 40.3.
- Qwen2-7B: MATH500 51.2 → S²R-PRL 65.4 (+14.2), Average 35.3 → 44.2 — 약한 base에서는 PRL(+8.9)이 ORL(+8.8)과 비슷하거나 더 나음.

**Cross-domain 일반화 (Table 3, Qwen2.5-Math-7B base)**:
- FOLIO: 37.9 (base) → 58.1 (BI) → 61.6 (ORL).
- CRUXEval: 40.8 → 48.0 → 50.9.
- StrategyQA: 61.1 → 88.7 → **90.8** (Qwen2.5-Math-72B-Instruct의 94.3에 근접).
- MMLUPro-STEM: 46.0 → 49.8 → 50.0.

**Verification 품질 (Figure 3, Qwen2.5-Math-7B)**:
- Verification Accuracy: SFT 61.58 → PRL 74.61 → ORL 66.49.
- Error Recall: 66.83 → **90.28** (PRL) → 87.85 (ORL) — 틀린 답을 탐지하는 recall이 크게 향상.
- Correct Precision: 84.94 → 64.75 → 70.11.

**Self-correction Rate**:
- Incorrect → Correct rate: SFT 6.52% → PRL 12.22% → ORL **13.64%** (성공적 교정 2배↑).
- Correct → Incorrect rate: SFT 1.96% → PRL 1.46% → ORL **0.97%** (잘못 수정하는 빈도는 오히려 감소).

**난이도별 분석 (Figure 4, MATH500, Llama3.1-8B)**:
- Level 1: 0.814 (SFT) → 0.930 (SFT+RL), trials 3.279 → 2.209 (쉬운 문제는 trial 줄임).
- Level 5: 0.239 → 0.276, trials 4.104 → 5.254 (어려운 문제는 trial 증가) — dynamic compute allocation 확인.
- Qwen2.5-Math-7B Level 5: 0.619 → 0.649, trials 2.254 → 2.149.

**Online vs Offline RL (Table 5)**:
- Qwen2.5-Math-7B-S2R-ORL-offline: MATH500 82.0, Average 61.2 (online 62.4 대비 comparable).
- Qwen2.5-Math-7B-S2R-PRL-offline: MATH500 83.4, College Math **50.0**, GSM8K **72.2** — 일부 벤치마크에서 online을 능가.
- 관찰: ORL은 online이 유리, PRL은 offline이 더 유리 (baseline 추정 정확도 이점).

**Verification 방식 비교 (Table 4)**:
- Qwen2.5-Math-7B Problem-solving: overall 77.25 / correct 91.21 / **incorrect 56.67** (틀린 답 탐지가 약함).
- Qwen2.5-Math-7B Confirmative: overall 61.58 / correct 82.80 / **incorrect 68.04** (더 balanced) — 그래서 Confirmative 채택.

## Limitation

1. **Math 도메인에 강하게 집중된 평가**: cross-domain 실험(FOLIO, CRUXEval, StrategyQA, MMLUPro-STEM)이 있으나 모두 rule-verifiable 성격 → open-ended generation, creative writing, 장문 QA 같은 soft-reward 영역에 대한 확장 가능성은 미검증이다.
2. **Rule-based reward의 적용 범위 제약**: V_golden이 ground-truth 매칭에 의존 → 정답이 명시적으로 존재하는 과제에 한정. 정답이 모호하거나 여러 valid answer가 있는 task에서는 reward modeling을 다시 설계해야 한다.
3. **Confirmative verification의 한계 내재**: 답을 대입해 검증하는 방식이라 "대입 가능한 수식/조건" 이 명확한 수학에서 강점이지만, 증명(proof)이나 open reasoning에서는 substitute 기반 확인이 어렵다.
4. **작은 모델(~7B)과 MATH-level 난이도에 특화**: AIME 2024 등 최상위 난이도에서는 여전히 GPT-o1-mini(56.7) 대비 Qwen2.5-Math-7B-S2R-ORL (23.3)의 격차가 크다 — olympiad-급 high-reasoning에는 scale이 더 필요함을 시사.
5. **Verify-Correct가 FSM으로 고정됨**: `solve → verify → (solve 또는 <end>)` 구조라 더 복잡한 메타-추론(verify 실패 시 plan 재설계, tool 호출, 외부 지식 참조 등)은 action space 밖이다.
6. **SFT mask 설계의 수작업성**: 어느 action을 mask할지(verify 전부 + 마지막 solve)는 휴리스틱 결정 — 더 긴 궤적이나 다른 task 구조에서는 최적 masking 전략이 달라질 수 있다.
7. **PRL의 group-based baseline은 context 공유 action이 충분히 많아야 신뢰**: reward context가 희소한 상황(긴 trajectory, 다양한 실패 패턴)에서는 baseline 분산이 커져 학습 불안정 위험이 있다.
8. **Difficulty-aware trajectory length 할당은 base model 샘플링에 의존**: base model이 특정 난이도 문제에 모두 실패하면 correct trajectory 자체가 수집 불가 — cold-start 문제는 여전히 teacher(예: 다른 strong model)에서 correct solution을 가져와야 해결된다.
