# S²R: Teaching LLMs to Self-verify and Self-correct via Reinforcement Learning

> **논문 정보**: Ruotian Ma, Xuanyu Zhang, Xingxuan Li, Yiwei Wang, Shafiq Joty, Lidong Bing
> **arXiv**: 2502.12853 / ACL 2025
> **코드**: https://github.com/RUCAIBox/S2R

---

## Problem

LLM이 추론 도중 스스로 오류를 감지·수정하는 "self-verify & self-correct" 능력은 o1류 System-2 추론의 핵심으로 평가된다.
그러나 기존 오픈소스 모델에서 이 능력을 재현하려는 시도는 GPT-4 수준 교사 모델로부터의 대규모 증류(distillation)나 수만 건의 human-annotated step-level 데이터에 의존해왔다.
예컨대 PRM800K, Math-Shepherd, 각종 o1-replication 파이프라인은 수십만 개 step-level 주석 또는 frontier LLM의 긴 reasoning trace가 필수였다.
이러한 자원 집약형 접근은 학계·스타트업 환경에서는 재현이 어렵고, 교사 모델 능력의 상한선을 넘지 못한다는 본질적 한계도 존재한다.
또한 단순 SFT로 검증·교정 시연을 주입해도 모델이 실제 추론 시 자발적으로 self-verification을 발동하지 못하고, 훈련 분포에 과적합되어 새로운 문제에서는 교정이 작동하지 않는 현상이 관찰된다.
즉 "행동을 흉내 내는 것"과 "행동을 의사결정적으로 활용하는 것" 사이의 간극이 해결되지 않은 상태다.
결과적으로 오픈소스 7B급 모델에서 진정한 self-correction 능력을 저비용으로 재현하는 방법은 공백으로 남아 있다.

---

## Motivation

저자들은 self-verification과 self-correction을 "외생적 신호로부터 증류받아야 하는 능력"이 아니라 "모델이 이미 잠재적으로 보유한 행동을 강화학습으로 끌어올릴 수 있는 행동(behavior)"으로 재정의한다.
따라서 소량(3.1K)의 고품질 시드 데이터로 행동의 "씨앗"만 심고, 이후 RL로 해당 행동을 보상과 함께 강화하면 대규모 증류 없이도 능력을 이끌어낼 수 있다는 가설을 세운다.
이는 RL with Verifiable Rewards (RLVR) 흐름과 맞닿아 있으며, GRPO/DPO 계열 알고리즘으로 이진 정답/오답 신호만 있어도 학습이 가능하다는 점을 레버리지한다.
Outcome-level RL만으로는 "정답에 운으로 도달" 문제가 발생하여 중간 verification 행동이 사라지는 보상 해킹(reward hacking)이 나타나므로, process-level RL을 병행하여 중간 단계의 self-verification·self-correction 행동 자체에 보상을 부여하는 구조가 필요하다.
데이터 효율(3.1K vs 수만~수십만), 교사 모델 비의존성, 그리고 RL로 발현되는 능력이 SFT 기반보다 OOD 일반화에 강하다는 실증적 관찰이 핵심 동기다.
저자들은 이를 통해 "저자원 환경에서의 o1-style reasoning 재현 레시피"를 제시하는 것을 목표로 한다.

## Method

1. **행동 정의 (Action Schema)**: 추론 trace를 `initial solution → verification → (revise | re-verify) → final answer` 형태로 구조화하여, 모델이 언제든지 "검증/재시도" 토큰 시퀀스를 발동할 수 있도록 action space를 설계한다.
2. **1단계 Behavior Initialization (SFT)**: 3.1K개의 고품질 시연 데이터를 수집. 각 샘플은 의도적으로 오류를 포함한 초안 → 자기검증 step → 교정 step → 정답으로 이어지는 trace를 담는다.
3. **시드 데이터 생성 절차**: 기본 모델로부터 롤아웃한 풀이에서 오답을 선별, GPT-4/수학 솔버로 오류 위치와 교정 경로를 주석하여 verify-then-correct 포맷으로 재구성. 소량이지만 "행동 다양성"을 유지하도록 오류 유형을 분산 샘플링한다.
4. **2단계 Outcome-level RL**: SFT 모델을 정책으로 삼아 RLVR을 수행. 정답이면 +1, 오답이면 0 또는 -1의 terminal reward. 알고리즘은 PPO/GRPO 계열을 사용하며 KL 페널티로 SFT 정책 근방을 유지한다.
5. **탐색 다양성 유지**: 초기화 데이터를 의도적으로 소량으로 제한하여 RL 단계에서 verification/correction 전략의 다양성을 보존. 큰 SFT 세트는 행동 다양성을 위축시킨다는 점을 실험적으로 확인.
6. **3단계 Process-level RL**: 중간 step(검증 발동, 오류 감지 정확성, 교정의 논리적 타당성)에 대해 step-level reward를 부여. 검증 step이 실제로 오류가 있는 초안을 true-detect했는지, correction step이 올바른 방향으로 수정했는지를 rule-based 신호 또는 verifier model로 판정.
7. **Outcome + Process 결합 보상**: 최종 reward는 outcome reward와 누적 process reward의 가중합으로 구성. Outcome은 전역 방향성, process는 국소 행동 정교화를 담당.
8. **보상 해킹 방지 장치**: "불필요한 verification 남발"에 페널티를 부여하고, verification이 실제로 오류를 바꾸어놓은 경우에만 positive reward를 인정하는 조건부 process reward를 사용.
9. **학습 루프**: 매 RL step마다 문제를 샘플링 → k개 trajectory 생성 → outcome + process reward 계산 → advantage 산출 → policy gradient 업데이트. KL coefficient와 clip ratio는 수렴 안정성을 위해 튜닝.
10. **추론 시점 자기 교정**: 학습 후에는 별도 프롬프트 없이도 모델이 자발적으로 "Let me verify..." 형태의 verification을 발동하고, 오류 감지 시 이전 step을 롤백/수정한 뒤 최종 답을 산출.
11. **일반화 전이**: 수학으로 학습된 self-verify/correct 행동이 논리·상식 등 인접 도메인 테스트에서도 부분적으로 전이됨을 ablation으로 확인.
12. **엔트로피·KL 제어**: RL 탐색 붕괴를 막기 위해 적응적 KL 페널티와 엔트로피 보너스를 적용, verification 행동의 다양성을 학습 종반까지 유지.
13. **평가 프로토콜**: 추론 시 temperature 0 / pass@1 기준으로 성능을 측정하여 외부 self-consistency나 Best-of-N 없이 단일 trajectory만의 능력을 평가.

## Key Contribution

1. **극소량 시드 + RL 조합**: 3.1K 샘플만으로 Qwen2.5-Math-7B를 51.0% → 81.6% (+30.6%p)로 끌어올려, 수만 건 SFT/증류 기반 기법과 동급 또는 상회 성능을 달성.
2. **Outcome × Process 이중 RL 프레임워크**: self-verification·self-correction을 전역 outcome과 국소 process 보상으로 동시에 강화하는 최초의 체계적 레시피를 제시.
3. **교사 모델 독립성**: GPT-4·o1 등 frontier LLM의 trace 증류 없이 오픈소스 7B만으로 o1-style reasoning을 재현하여, 저자원 재현성 문제를 완화.
4. **데이터 효율성 증명**: 시드 데이터 크기를 3.1K로 의도적으로 제한하는 것이 오히려 RL 탐색 다양성을 보존해 최종 성능을 높인다는 counter-intuitive 발견을 실증.
5. **행동 중심 관점**: self-correction을 "증류할 지식"이 아니라 "RL로 강화할 잠재 행동"으로 프레이밍하는 새로운 방법론적 시각 제시.
6. **오픈소스 레시피 공개**: 데이터·학습 스크립트·체크포인트를 공개하여 7B급 모델의 System-2 reasoning 재현 기준점을 제공.

## Experiment

- **기반 모델**: Qwen2.5-Math-7B(주요), Qwen2.5-Math-1.5B, Llama-3.1-8B로 확장 검증.
- **벤치마크**: MATH-500, GSM8K, AIME 2024, AMC 2023, College Math, OlympiadBench 등 수학 추론 6종.
- **핵심 성능**: Qwen2.5-Math-7B 기준 전체 평균 51.0% → 81.6% (+30.6%p), MATH-500에서는 약 84% → 89%대, GSM8K 95%+, AIME 2024에서 10%대 → 40%대로 상승.
- **비교군 대비**: 수만 건 SFT를 사용하는 rStar-Math, Math-Shepherd 등과 비교해 3.1K 시드만으로 동등하거나 일부 벤치마크에서 상회.
- **Ablation 1 — 초기화 제거**: SFT 시드 없이 base 모델에서 바로 RL 시도 시 성능이 크게 저하(수 %p) — 소량이라도 행동 초기화가 필수임을 입증.
- **Ablation 2 — Outcome only vs Outcome+Process**: Outcome-only RL 대비 이중 RL이 MATH-500에서 +4~6%p, AIME에서 +5%p 이상 개선.
- **Ablation 3 — 시드 규모 스케일링**: 시드를 3.1K → 10K → 30K로 늘려도 최종 성능이 포화되거나 오히려 하락 — 행동 다양성 축소 가설을 뒷받침.
- **Ablation 4 — Process reward 구성**: verification 남발 페널티 없이 process reward만 주면 불필요한 self-check 루프가 생겨 속도·정확도 모두 저하.
- **자기 교정 발현 빈도**: 학습 후 모델은 약 30~50% 문제에서 최소 1회 verification을 발동하며, 이 중 유의미한 비율이 실제 오류 교정으로 이어짐.
- **일반화**: 수학으로 훈련된 S²R이 논리·상식 벤치마크 일부에서도 baseline 대비 개선, self-correction 행동의 도메인 전이 가능성을 시사.
- **효율성**: 3.1K 시드 + RL 총 GPU 비용이 수만 건 SFT 대비 수배 낮음에도 동급 성능 달성.

## Limitation

본 연구의 평가가 수학(특히 경쟁 수학) 도메인에 집중되어 있어 코드 생성·에이전트 작업·멀티스텝 도구 사용 등 다른 System-2 도메인으로의 전이는 제한적으로만 검증되었다.
3.1K 시드 데이터의 구성·품질(오류 유형 분포, 교정 스타일 다양성)에 모델 성능이 민감할 가능성이 높아, 다른 연구자가 동일 레시피로 재현할 때 변동성이 클 수 있다.
Process-level reward를 설계하는 과정이 rule-based 판정 또는 verifier 모델에 의존하는데, verifier 자체의 노이즈가 reward hacking을 유발할 수 있으며 이를 해결하기 위한 가중치 튜닝이 상당한 실험적 부담을 준다.
Outcome reward의 이진 특성상 부분 점수 체계를 구축하기 어렵고, AIME 등 극도로 어려운 문제군에서는 정답 rollout이 희소해 학습 시그널이 빈약해지는 sparse reward 문제가 잔존한다.
학습된 self-verification 행동이 실제로 "내부 상태에 기반한 정직한 검증"인지, 아니면 "보상을 얻기 위한 의례적 토큰 시퀀스"인지에 대한 해석적 분석이 부족하다.
RL 과정의 불안정성(KL 폭주, 엔트로피 붕괴) 관리를 위한 하이퍼파라미터 민감도가 높아, 본 논문이 제시하는 하이퍼세팅을 벗어날 경우 재현이 어려울 수 있다.
또한 self-correction 행동이 추론 시점의 토큰 길이를 증가시켜 지연·비용 측면의 트레이드오프가 존재하며, 지연 민감 애플리케이션에서는 verification 발동 빈도 제어가 추가 과제로 남는다.
마지막으로 본 연구는 single-agent 단일 LLM 관점에 머물러 있으며, multi-agent critic이나 외부 도구(예: SymPy) 기반 검증과의 결합 효과는 탐색되지 않았다.
