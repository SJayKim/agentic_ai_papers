# Automated Design of Agentic Systems (ADAS)

> **논문 정보**: Shengran Hu, Cong Lu, Jeff Clune (University of British Columbia, Vector Institute, Canada CIFAR AI Chair)
> **arXiv**: 2408.xxxxx (2024) | **학회**: ICLR 2025
> **코드**: https://github.com/ShengranHu/ADAS

---

## Problem

현대의 에이전틱 시스템(Chain-of-Thought, Self-Reflection, Toolformer 등)은 연구자와 엔지니어가 수작업으로 빌딩 블록을 설계·조합하여 구축된다.
이러한 수작업 접근은 도메인별 튜닝이 필수적이며, 막대한 인적 노력과 시간 비용이 소요된다.
머신러닝의 역사를 살펴보면 HOG 특징 같은 수작업 결과물이 CNN의 학습된 특징으로 대체되었고, 수동 아키텍처 설계도 NAS(Neural Architecture Search)에 밀려난 바 있다.
아직 발견되지 않은 에이전트 빌딩 블록이 매우 많으며, 이를 수작업으로 조합해 효과적인 시스템을 만드는 일은 인간 직관의 한계에 묶여 있다.
기존 자동화 시도들(PromptBreeder, OPRO 등)은 오직 텍스트 프롬프트 문구만 최적화할 뿐, 워크플로우나 도구 사용 같은 다른 구성 요소를 변경하지 못하는 구조적 한계를 가진다.
일부 연구(AgentOptimizer, AutoFlow)는 도구나 워크플로우를 학습하려 시도했으나, 탐색 공간이 너무 좁거나 최적화가 어려운 표현을 채택하여 범용성이 떨어진다.
결과적으로 모든 구성 요소(프롬프트, 도구 사용, 워크플로우)를 동시에 탐색할 수 있는 통합적 프레임워크가 부재한 상태이다.

---

## Motivation

핵심 통찰은 에이전트 전체를 코드로 정의하면, 튜링 완전성을 가진 프로그래밍 언어(Python) 덕분에 이론적으로 모든 가능한 에이전틱 시스템을 탐색 공간에 담을 수 있다는 점이다.
이는 프롬프트만 최적화하거나 그래프 구조만 탐색하는 기존 방식과 근본적으로 차별화된 접근이다.
최신 Foundation Model들은 코드 작성 능력이 매우 뛰어나므로, FM 자체를 "메타 에이전트"로 사용해 새로운 에이전트를 코드 공간에서 자동 프로그래밍하는 것이 가능하다.
코드 기반 표현은 해석 가능성(readability)이 높아 디버깅과 안전성 확보에도 유리하며, 설계 패턴을 사람이 명시적으로 이해할 수 있다.
LangChain 같은 오픈소스 프레임워크 위에서 기존 빌딩 블록(RAG, search tool 등)을 재활용하며 탐색을 수행할 수 있는 실용적 이점도 크다.
기존 open-endedness 알고리즘이 보여주듯, 아카이브에 축적된 이전 발견이 디딤돌(stepping stone) 역할을 하여 점진적으로 더 복잡하고 강력한 시스템이 출현하는 메커니즘을 에이전트 설계에 적용할 수 있다.

## Method

1. **ADAS 프레임워크 형식화**: Automated Design of Agentic Systems를 (1) 탐색 공간(Search Space), (2) 탐색 알고리즘(Search Algorithm), (3) 평가 함수(Evaluation Function)의 세 축으로 정의한다.

2. **코드 기반 탐색 공간**: 에이전트를 Python 코드로 정의하여 프롬프트, 도구 사용, 워크플로우 및 이들의 조합까지 이론적으로 모든 가능한 구성을 표현한다.

3. **최소 프레임워크 제공**: 메타 에이전트에게 FM 쿼리 API, 프롬프트 포맷팅 함수 등 핵심 유틸리티를 포함한 약 100줄 분량의 간결한 프레임워크를 기본으로 제공한다.

4. **forward 함수 프로그래밍**: 메타 에이전트는 이 프레임워크 위에서 `forward(Task)` 함수 하나만 구현하여 새로운 에이전트를 정의한다.

5. **Meta Agent Search 루프**: 메타 에이전트(GPT-4)가 아카이브를 참조해 "흥미롭고 새로운(novel or worthwhile)" 에이전트를 코드로 작성한다.

6. **검증 및 오류 수정**: 생성된 코드를 검증셋에서 실행하고, 오류가 발생하면 최대 5회까지 자동 수정 반복을 수행한다.

7. **아카이브 갱신**: 평가 결과(성공률 또는 F1 점수)와 함께 에이전트를 아카이브에 추가하고, 이 확장된 아카이브를 바탕으로 다음 반복에서 더 나은 에이전트를 생성한다.

8. **시드 에이전트 초기화**: CoT, COT-SC, Self-Refine, LLM-Debate, Quality-Diversity 등 기존 SOTA 수작업 에이전트들을 아카이브의 초기 시드로 활용한다.

9. **개방형 탐색 원리**: open-endedness 알고리즘에서 영감을 받아 "흥미로움" 개념을 목표로 삼고, 이전 발견이 다음 발견을 위한 디딤돌 역할을 하도록 설계한다.

10. **점진적 혁신 관찰**: 예컨대 반복 3에서 출현한 "다중 COT → 정제 → 앙상블" 패턴은 이후 설계의 핵심 빌딩 블록으로 재사용된다.

11. **다중 평가 프로토콜**: 각 에이전트를 검증셋과 홀드아웃 테스트셋 모두에서 5회 반복 실행하여 FM의 확률적 샘플링에 따른 분산을 감소시킨다.

12. **신뢰 구간 보고**: 중앙값과 95% 부트스트랩 신뢰 구간을 함께 보고하여 통계적 유의성을 엄격히 검증한다.

13. **비대칭 모델 설정**: 탐색 비용 절감을 위해 메타 에이전트는 GPT-4, 평가 대상 에이전트와 베이스라인은 GPT-3.5로 구성한다.

14. **안전 실행 환경**: 생성된 코드의 잠재적 위험을 방지하기 위해 컨테이너화된 격리 환경에서 실행하고 수동 검사를 병행한다.

15. **25회 반복 탐색**: 각 도메인별로 Meta Agent Search를 25 이터레이션 동안 실행하여 아카이브를 점진적으로 확장한다.

## Key Contribution

1. **ADAS라는 새로운 연구 영역을 공식화**: 에이전틱 시스템의 자동 설계 문제를 탐색 공간·탐색 알고리즘·평가 함수의 세 축으로 체계화하여 기존 산발적 시도들을 통합하는 개념 체계를 제공했다.

2. **코드 공간 탐색의 보편성 입증**: 프롬프트만 최적화하던 기존 접근과 달리 에이전트 전체를 코드로 표현함으로써 이론적으로 모든 에이전틱 시스템을 포괄하는 탐색 공간을 최초로 제안했다.

3. **Meta Agent Search 알고리즘 제안**: 아카이브 기반 반복 탐색으로 FM 메타 에이전트가 점진적으로 더 나은 에이전트를 프로그래밍하는 간결하면서도 강력한 알고리즘을 제시했다.

4. **수작업 SOTA 대비 대폭적 성능 향상**: DROP에서 F1 +13.6, MGSM에서 정확도 +14.4%, GSM8K 전이 +25.9% 등 다수 도메인에서 기존 최고 수작업 에이전트를 큰 격차로 능가했다.

5. **도메인 간 전이 가능성 확인**: 수학 도메인에서 발견된 에이전트가 독해 도메인(DROP)에서도 수작업 베이스라인을 능가하는 강한 범용성을 보였다.

6. **모델 간 전이 가능성 확인**: GPT-3.5로 발견된 에이전트가 Claude-Sonnet, GPT-4, Claude-Haiku 등 상이한 FM에서도 우수한 성능을 유지함을 실증했다.

7. **오픈소스 공개**: 모든 코드와 발견된 에이전트 아카이브를 공개해 후속 연구의 재현성과 확장성을 보장했다.

## Experiment

**데이터셋 및 설정**: ARC Challenge, DROP(독해, F1), MGSM(수학, 정확도), MMLU(다중과제), ARC-AGI Science, 전이 평가용 GSM8K·GSM-Hard를 사용한다.
메타 에이전트는 GPT-4, 발견 에이전트와 베이스라인은 GPT-3.5로 평가하며 25회 반복 탐색을 수행한다.

**주요 수치 (멀티 도메인 표 1)**:
- **DROP (독해)**: Meta Agent Search F1 **79.4** vs. Role Assignment 65.8 → **+13.6 F1** 향상
- **MGSM (수학)**: **53.4%** vs. LLM Debate 39.0% → **+14.4%** 향상
- **MMLU (다중과제)**: **69.6%** vs. COT-SC 65.9% → **+3.7%** 향상
- **Science**: 34.6% vs. Self-Refine 31.6% → +3.0% 향상

**전이 성능 (표 2, MGSM → held-out)**:
- **GSM8K**: **69.5%** vs. LLM Debate 43.6% → **+25.9%** 향상
- **GSM-Hard**: **31.2%** vs. Role Assignment 18.0% → **+13.2%** 향상
- **MMLU 전이**: 67.0% F1 (Structured Multimodal Feedback Loop) vs. COT-SC 65.9%

**모델 전이 (표 3, ARC)**: GPT-3.5에서 발견된 "Dynamic Memory and Refinement Agent"가 Claude-Sonnet에서 **48.3%**, GPT-4에서 37.0%, Claude-Haiku에서 9.7% 달성 (Self-Refine 베이스라인 Claude-Sonnet 39.3% 대비 +9.0%).

**프롬프트 최적화 비교**: OPRO 대비 DROP F1 79.4 vs. 69.1, MGSM 53.4% vs. 30.6%로 전 도메인 우위.

**분석**: Science·MMLU에서 개선 폭이 작은 이유는 FM의 지식 부족이 병목이기 때문이며, 독해·수학처럼 FM이 지식을 충분히 보유한 도메인에서 에이전트 설계 최적화의 효과가 극대화된다.
ARC 탐색 과정 분석 결과 반복 3에서 출현한 "다중 COT + 정제 + 앙상블" 패턴이 이후 설계들의 공통 디딤돌이 되는 점진적 혁신 패턴이 관찰되었다.

## Limitation

저자들이 명시한 한계로는 현재 평가가 단일 단계 QA 과제에만 국한되어 있으며, 다단계 상호작용이 필요한 복잡한 실세계 응용으로의 확장 검증이 부족하다는 점이다.
생성 코드의 안전성 문제 역시 존재하여, 현재는 컨테이너화된 격리 환경과 수동 검사에 의존하고 있어 완전 자동화된 안전 보장 메커니즘이 부재하다.
독자 관점에서, 메타 에이전트로 GPT-4를 사용하고 평가 대상은 GPT-3.5를 사용하는 비대칭 설정이므로 탐색 비용이 상당하며 대규모 배포 시 비용 효율성이 명확하지 않다.
탐색된 에이전트의 설계 패턴이 왜 효과적인지에 대한 이론적·인과적 분석이 부족하여 결과가 다소 관찰적 수준에 머무른다.
탐색 알고리즘 자체가 "흥미로움" 기반 단순 탐색에 의존해 탐색-활용(exploration-exploitation) 균형이 명시적으로 관리되지 않으며, Quality-Diversity나 MAP-Elites 같은 정교한 전략 도입 여지가 남아 있다.
고차 ADAS(메타 에이전트 자체를 ADAS로 개선) 및 온라인 지속 학습은 제안만 되었을 뿐 본 논문에서는 검증되지 않았다.
마지막으로, 벤치마크 의존적 평가 프로토콜은 실제 활용 시나리오에서의 일반화 능력을 완전히 담보하지 못하며, 단일 도메인 탐색 결과가 다중 도메인 범용 에이전트 설계로 자연스럽게 확장되는지 추가 검증이 필요하다.
