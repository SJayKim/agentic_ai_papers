# Automated Design of Agentic Systems (ADAS)

> **논문 정보**: Shengran Hu, Cong Lu, Jeff Clune (University of British Columbia, Vector Institute, Canada CIFAR AI Chair)
> **arXiv**: 2408.xxxxx (2024) | **학회**: ICLR 2025
> **코드**: https://github.com/ShengranHu/ADAS

---

## Problem

기존의 에이전틱 시스템(Chain-of-Thought, Self-Reflection, Toolformer 등)은 연구자와 엔지니어가 수동으로 빌딩 블록을 설계하고 조합하여 구축한다. 이러한 수작업 기반 접근은 도메인별 튜닝이 필요하고 상당한 인적 노력이 소요된다. 머신러닝의 역사가 보여주듯, 수작업 설계(HOG 특징, 수동 아키텍처 등)는 결국 학습 기반 솔루션(CNN, Neural Architecture Search)으로 대체되어 왔다. 아직 발견되지 않은 에이전틱 시스템의 빌딩 블록이 매우 많고, 이를 모두 발견하더라도 서로 다른 방식으로 결합하여 효과적인 시스템을 구성하는 것은 시간 소모적이며 인간의 직관에 제한된다. 기존 자동화 시도(PromptBreeder 등)는 텍스트 프롬프트만 최적화하여 워크플로우나 도구 사용 같은 다른 구성 요소를 변경할 수 없다는 근본적 한계가 있다.

---

## Motivation

핵심 관찰은 에이전틱 시스템 전체를 코드로 정의하면, 프로그래밍 언어(Python)의 튜링 완전성 덕분에 이론적으로 모든 가능한 에이전틱 시스템(프롬프트, 도구 사용, 워크플로우 및 이들의 조합)을 탐색 공간에 포함시킬 수 있다는 점이다. 이는 프롬프트만 최적화하거나 그래프 구조만 탐색하는 기존 방식과 근본적으로 다르다. 또한 최신 Foundation Model이 코딩에 능숙하므로, FM 자체를 "메타 에이전트"로 활용하여 코드 공간에서 새로운 에이전트를 자동으로 프로그래밍할 수 있다. 코드 기반 표현은 해석 가능성이 높아 디버깅과 안전성 확보에도 유리하며, LangChain 같은 기존 오픈소스 프레임워크 위에서 기존 빌딩 블록을 재활용하며 탐색할 수 있다는 실용적 장점도 있다.

---

## Method

1. **ADAS 프레임워크 정의**: Automated Design of Agentic Systems를 (1) 탐색 공간(Search Space), (2) 탐색 알고리즘(Search Algorithm), (3) 평가 함수(Evaluation Function)의 세 구성 요소로 형식화한다. 탐색 공간은 에이전트를 코드(Python)로 정의하여 이론적으로 모든 가능한 에이전틱 시스템을 포괄한다.

2. **기본 프레임워크 제공**: 메타 에이전트에게 FM 쿼리 API, 프롬프트 포맷팅 등 기본 함수를 포함한 약 100줄의 간결한 프레임워크를 제공한다. 메타 에이전트는 이 프레임워크 위에서 `forward` 함수만 프로그래밍하여 새로운 에이전트를 정의한다.

3. **Meta Agent Search 알고리즘**: (1) 메타 에이전트(GPT-4)가 기존 에이전트 아카이브를 참조하여 "흥미롭고 새로운(novel or worthwhile)" 에이전트를 코드로 프로그래밍한다. (2) 생성된 에이전트를 검증 데이터셋에서 평가한다. (3) 코드에 오류가 있으면 최대 5회까지 수정을 반복한다. (4) 평가 결과와 함께 아카이브에 추가한다. (5) 업데이트된 아카이브를 기반으로 다음 반복에서 더 나은 에이전트를 생성한다.

4. **아카이브 기반 개방형 탐색**: 기존 open-endedness 알고리즘에서 영감을 받아, 아카이브에 축적된 이전 발견들이 디딤돌(stepping stone) 역할을 하여 점진적으로 더 복잡하고 성능 좋은 에이전트가 발견된다. 예를 들어, 초기 반복에서 등장한 "다중 COT → 정제 → 앙상블" 패턴이 이후 에이전트 설계의 핵심 빌딩 블록이 된다.

5. **시드 초기화**: CoT, COT-SC, Self-Refine, LLM-Debate, Quality-Diversity 등 기존 최신 수작업 에이전트를 아카이브의 초기 시드로 사용한다.

6. **평가**: 발견된 에이전트를 검증셋과 홀드아웃 테스트셋 모두에서 5회 반복 평가하여 분산을 줄이고, 95% 부트스트랩 신뢰 구간을 보고한다.

---

## Key Contribution

1. **ADAS라는 새로운 연구 영역을 정의**: 에이전틱 시스템의 자동 설계를 탐색 공간·탐색 알고리즘·평가 함수의 세 축으로 형식화하여, 기존 산발적 시도들을 통합하는 체계를 제시했다.

2. **코드 공간 기반 탐색의 가능성 입증**: 프롬프트만 최적화하는 기존 방식과 달리, 에이전트 전체를 코드로 정의하고 FM이 코드를 프로그래밍하는 접근이 이론적으로 모든 가능한 에이전트를 탐색할 수 있음을 보였다.

3. **Meta Agent Search 알고리즘 제안**: 아카이브 기반 반복 탐색을 통해 점진적으로 더 나은 에이전트를 자동 발견하는 간결하면서도 효과적인 알고리즘을 제시했다.

4. **수작업 설계 대비 대폭적 성능 향상**: DROP에서 F1 +13.6, MGSM에서 정확도 +14.4%, 전이 시 GSM8K에서 +25.9% 등 다양한 도메인에서 SOTA 수작업 에이전트를 크게 능가했다.

5. **도메인 및 모델 간 전이 가능성 입증**: 수학 도메인에서 발견된 에이전트가 독해 과제에서도 수작업 베이스라인을 능가하고, GPT-3.5에서 발견된 에이전트가 Claude-Sonnet에서도 우수한 성능을 보여 범용성을 확인했다.

## Experiment & Results

**데이터셋 및 설정**: (1) ARC Challenge, (2) DROP (독해, F1 스코어), (3) MGSM (수학, 정확도), (4) MMLU (다중과제), (5) ARC-AGI Science, (6) 전이 평가로 GSM8K 및 GSM-Hard. 메타 에이전트는 GPT-4, 발견된 에이전트와 베이스라인은 GPT-3.5로 평가. 25회 반복 탐색.

**주요 수치**:
- **DROP (독해)**: Meta Agent Search F1 79.4 vs. 최고 베이스라인(Role Assignment) 65.8 → **+13.6 F1** 향상
- **MGSM (수학)**: Meta Agent Search 53.4% vs. 최고 베이스라인(LLM Debate) 39.0% → **+14.4%** 향상
- **GSM8K 전이**: 69.5% vs. 최고 베이스라인(LLM Debate) 43.6% → **+25.9%** 향상
- **GSM-Hard 전이**: 31.2% vs. 최고 베이스라인(Role Assignment) 18.0% → **+13.2%** 향상
- **ARC 모델 전이**: GPT-3.5에서 발견된 에이전트가 Claude-Sonnet에서 **48.3%** 정확도 달성 (Self-Refine 베이스라인 39.3%)
- **MMLU (다중과제)**: 69.6% vs. 최고 베이스라인 65.9% → +3.7% 향상

**프롬프트 최적화 비교**: OPRO와 비교 시, 모든 도메인에서 Meta Agent Search가 우위 (예: DROP F1 79.4 vs. 69.1, MGSM 53.4% vs. 30.6%).

**Ablation/분석**: Science·MMLU에서 개선 폭이 상대적으로 작은 이유는 FM의 지식 부족이 병목이기 때문이며, 독해·수학처럼 FM이 충분한 지식을 보유한 도메인에서는 에이전틱 시스템 최적화의 효과가 극대화된다. ARC에서 탐색 과정 분석 결과, 반복 3에서 등장한 "다중 COT + 정제 + 앙상블" 패턴이 이후 설계의 핵심 디딤돌이 되는 점진적 혁신이 관찰되었다.

---

## Limitation

저자들은 현재 단일 단계 QA 과제에서만 평가했으며, 복잡한 환경과 다단계 상호작용이 필요한 실제 응용으로의 확장이 필요하다고 명시한다. 또한 생성된 코드의 안전성 문제가 있어, 현재는 컨테이너화된 격리 환경에서 실행하고 수동 검사를 병행하고 있다. 독자 관점에서, 메타 에이전트로 GPT-4를 사용하고 평가 대상은 GPT-3.5를 사용하는 비대칭 설정이므로 탐색 비용이 상당하며, 실제 대규모 배포 시 비용 효율성이 불분명하다. 탐색된 에이전트의 설계 패턴이 왜 효과적인지에 대한 이론적 분석이 부족하고, 탐색 알고리즘 자체가 상대적으로 단순하여(탐색-활용 균형 미비) 더 정교한 탐색 전략의 여지가 있다. 마지막으로, 고차 ADAS(메타 에이전트 자체를 ADAS로 개선)는 제안만 되었을 뿐 검증되지 않았다.
