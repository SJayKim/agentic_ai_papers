# SeCom: On Memory Construction and Retrieval for Personalized Conversational Agents

> **논문 정보**: Zhuoshi Pan, Qianhui Wu, Huiqiang Jiang, Xufang Luo, Hao Cheng, Dongsheng Li, Yuqing Yang, Chin-Yew Lin, H. Vicky Zhao, Lili Qiu, Jianfeng Gao (Tsinghua University, Microsoft)
> **arXiv**: 2502.05589 | **학회**: ICLR 2025
> **프로젝트**: https://aka.ms/secom

---

## Problem

장기 개방 도메인 대화(long-term open-domain conversation)에서 LLM 기반 에이전트는 여러 세션에 걸쳐 누적된 수백 턴의 이력을 활용하여 일관되고 개인화된 응답을 생성해야 한다.
기존 메모리 관리 방법은 대화 이력을 턴(turn) 단위, 세션(session) 단위, 또는 요약(summarization) 기반으로 메모리 뱅크에 저장하지만, 각 방식은 검색 정확도와 의미 품질 양쪽에서 한계를 보인다.
턴 단위 메모리는 세분도가 지나치게 작아서 관련 정보가 여러 턴에 분산되면서 키워드가 명시적으로 포함되지 않은 턴이 검색에서 누락되고, 검색된 컨텍스트는 단편적(fragmentary)이어서 완전한 정보 흐름을 포착하지 못한다.
세션 단위 메모리는 하나의 세션에 여러 토픽이 혼재할 때 무관한 내용이 다량 포함되어, 검색 정밀도 하락과 LLM의 긴 컨텍스트 이해 저하를 동시에 유발한다.
요약 기반 방법은 대화를 요약으로 변환하는 과정에서 QA에 필수적인 세부 수치/이유/근거가 손실되어 최적 이하의 응답을 생성한다.
LOCOMO 벤치마크는 평균 300턴, 세션당 약 9K 토큰에 달하는 매우 긴 대화를 포함하여 이러한 세분도 문제가 극적으로 드러나는 환경이다.
Full History 방식으로 모든 이력을 연결하면 LLM이 중간 구간의 정보를 놓치는 "lost in the middle" 현상과 함께 무관 정보로 인한 환각(hallucination)이 증가한다.
따라서 "어떤 세분도의 메모리 유닛이 검색 정확도와 응답 품질을 동시에 최적화하는가"는 장기 대화 에이전트의 핵심 미해결 병목이다.

## Motivation

자연 대화는 토픽별로 일관된 담화 단위(topically coherent discourse segments)로 본질적으로 구성된다는 관찰이 출발점이다.
턴 단위와 세션 단위 사이의 중간 세분도인 세그먼트(segment) 단위 메모리는, 관련 정보를 충분히 포함하면서도 무관한 내용을 배제하는 균형점을 제공할 수 있다는 가설을 세운다.
또한 자연어에는 본질적인 중복성(redundancy)이 존재한다는 Shannon(1951)의 고전적 통찰에 기반하여, 이러한 중복이 검색 시스템에 노이즈로 작용하여 핵심 정보 추출을 방해한다고 추정한다.
프롬프트 압축 기법 LLMLingua-2를 메모리 유닛에 적용하면 의미를 보존한 채 중복을 제거하는 디노이징(denoising) 효과를 얻을 수 있으며, 이는 별도 임베딩 파인튜닝 없이 플러그 앤 플레이로 적용 가능한 이점을 지닌다.
실제로 BM25와 MPNet 기반 검색기 모두에서 압축률(compression rate) 50% 이상일 때 검색 재현율(recall)이 K=1,2,3 모든 컨텍스트 예산에서 일관되게 개선된다.
디노이징 후 쿼리와 관련 세그먼트 간 유사도는 상승하고 무관 세그먼트와의 유사도는 하락하여, 검색 시그널 대 잡음 비율(SNR)이 본질적으로 개선됨을 실험적으로 확인하였다.
기존 요약 기반 디노이징은 LLM 호출 지연과 임베딩 모델 파인튜닝으로 인한 확장성·과적합 문제를 안고 있어, 압축 기반 접근이 구조적으로 우월하다.
이 두 가지 통찰—세그먼트 단위 메모리와 압축 기반 디노이징—을 결합하면 턴/세션/요약 방식의 한계를 동시에 극복할 수 있다는 것이 SeCom의 핵심 착상이다.

## Method

1. **연구 프레임워크 정의**: 대화 이력 H = {c_i}로부터 메모리 뱅크 M을 구성(Memory Construction)하고, 타깃 사용자 요청 u*와 컨텍스트 예산 N이 주어지면 검색기 f_R로 N개 메모리 유닛을 검색(Memory Retrieval)하며, 시간순 정렬 후 응답 생성기 f_LLM으로 최종 응답 r*를 생성(Response Generation)하는 3단계 파이프라인을 설정한다.

2. **대화 세그멘테이션 모델 f_I**: 세션 c_i = {t_j}를 토픽별로 일관된 세그먼트 {s_k}로 분할하며, 각 세그먼트는 시작 인덱스 p_k와 종료 인덱스 q_k로 표현되고 p_{k+1} = q_k + 1의 연속 조건을 만족한다.

3. **제로샷 세그멘테이션 (GPT-4-0125)**: 세션 데이터에 턴 인덱스와 역할 식별자(`Turn j:\n[user]: u_j\n[agent]: r_j`)를 부착하여 명확한 컨텍스트를 제공하고, 전체 세그먼트 경계를 한 번의 호출로 JSONL 형식으로 출력하게 하여 LumberChunker 방식의 반복적 세그멘테이션 지연을 회피한다.

4. **경량 세그멘테이션 모델 지원**: Mistral-7B-Instruct-v0.3와 SuperDialseg로 파인튜닝된 RoBERTa 기반 모델도 세그멘테이션 엔진으로 사용 가능하여 자원 제약 환경에서도 적용 가능하다.

5. **Reflection 기반 가이드라인 학습 (Few-shot)**: 소량의 주석 데이터가 있을 때, 제로샷 결과 중 WindowDiff 오류가 가장 큰 상위 K개 "hard examples"를 선별하고 LLM이 ground-truth와 비교하여 세그멘테이션 가이드라인 G를 반복적으로 개선한다.

6. **SGD 유비 업데이트 규칙**: 가이드라인 업데이트는 G_{m+1} = G_m − η∇L(G_m) 형태의 SGD 최적화로 해석되며, ∇L(G_m)은 LLM이 암묵적으로 추정하는 세그멘테이션 손실의 기울기로 간주된다.

7. **Rubric + 대표 예시 이원 구조**: 학습된 가이드라인은 (1) 10개의 Segmentation Rubric 항목과 (2) ground-truth/예측/반성(reflection)을 포함한 대표 예시로 구성되며, 상위 100개 poorly-segmented 샘플을 10개 미니배치로 분할하여 배치별로 rubric을 생성한다.

8. **세그먼트 단위 메모리 뱅크 구성**: 세그멘테이션으로 생성된 각 세그먼트 s가 하나의 메모리 유닛 m이 되어 |M| = Σ K_i 크기의 뱅크를 형성하며, 요약 변환 없이 원본 대화를 세그먼트 단위로 직접 저장하여 정보 손실을 원천 차단한다.

9. **압축 기반 메모리 디노이징 f_Comp**: 검색 전 LLMLingua-2(압축률 75%, xlm-roberta-large 백본)를 적용하여 자연어 중복을 제거하고, 검색은 f_R(u*, f_Comp(M), N) 형태로 수행된다.

10. **플러그 앤 플레이 호환성**: 디노이징은 검색기 임베딩을 파인튜닝하지 않으므로 BM25, MPNet 등 어떤 검색기와도 결합 가능하며 도메인 적응성과 확장성을 해치지 않는다.

11. **검색 및 응답 생성**: MPNet(multi-qa-mpnet-base-dot-v1) + FAISS 또는 BM25로 디노이징된 M에서 상위 N개 세그먼트를 검색한 뒤, 시간순으로 재정렬하여 GPT-3.5-Turbo 또는 Mistral-7B-Instruct-v0.3에 컨텍스트로 제공한다.

12. **컨텍스트 예산 설계**: LOCOMO는 4K 토큰 예산(약 5 세션, 10 세그먼트, 55 턴), Long-MT-Bench+는 1K 토큰 예산(약 1 세그먼트, 3 턴)으로 공정 비교를 위한 토큰 총량을 베이스라인과 맞춘다.

## Key Contribution

1. **메모리 세분도 문제의 체계적 규명**: 턴/세션/요약 기반 메모리 각각이 검색 정확도(DCG)와 의미 품질 측면에서 지니는 구조적 한계를 처음으로 정량적으로 분석하고, 세그먼트 단위가 최적 균형점임을 BLEU, ROUGE2, GPT4Score 다각 지표로 실증하였다.

2. **압축 = 디노이징 발견**: 프롬프트 압축 기법 LLMLingua-2가 메모리 검색 시스템의 효과적인 디노이징 메커니즘으로 작동함을 최초로 입증하여, 요약/파인튜닝 기반 디노이징보다 구조적으로 우월함을 보였다.

3. **SeCom 시스템 제안**: 대화 세그멘테이션과 압축 기반 디노이징을 결합한 통합 메모리 관리 시스템을 제안하여, LOCOMO와 Long-MT-Bench+ 두 장기 대화 벤치마크에서 모두 SOTA를 달성하였다.

4. **Reflection 기반 세그멘테이션 학습**: 단 100개의 hard example만으로 LLM이 세그멘테이션 가이드라인을 자기 반성 방식으로 학습하여, 전체 학습 데이터로 훈련한 지도 학습 RoBERTa 모델을 능가하는 전이 성능을 입증하였다.

5. **자원 제약 환경 호환성**: GPT-4뿐 아니라 Mistral-7B-Instruct-v0.3, SuperDialseg 파인튜닝 RoBERTa 등 경량 모델을 세그멘테이션 엔진으로 사용해도 대부분의 베이스라인을 능가하는 강건성을 확인하였다.

6. **세그멘테이션 벤치마크 SOTA**: DialSeg711, SuperDialSeg, TIAGE 세 개의 대화 세그멘테이션 벤치마크에서 비지도 및 전이 학습 설정 모두에서 최고 성능을 달성하여, 세그멘테이션 모듈 자체의 독립적 우수성을 검증하였다.

## Experiment

- **LOCOMO 벤치마크**: SeCom(BM25, GPT4-Seg)이 GPT4Score **71.57**을 기록하여 최선 베이스라인 ConditionMem(65.92) 대비 **+5.65점** 향상, Turn-Level(BM25) 65.58, Session-Level(BM25) 63.16, Full History 54.15, Zero History 24.86을 모두 상회.
- **Long-MT-Bench+ 벤치마크**: SeCom(MPNet, GPT4-Seg)이 GPT4Score **88.81**로 최고 성능을 달성하여 MemoChat(85.14), Turn-Level(MPNet, 84.91) 대비 각각 +3.67, +3.90점 우위.
- **디노이징 제거 Ablation**: LOCOMO에서 MPNet 기반 SeCom의 GPT4Score가 69.33 → **59.87로 9.46점 하락**, Long-MT-Bench+에서도 88.81 → 87.51로 1.30점 하락하여 압축 기반 디노이징의 결정적 기여를 확인.
- **Mistral-7B 응답 생성기 실험**: SeCom(MPNet)이 Long-MT-Bench+에서 GPT4Score **90.58**을 달성하여 Full History(78.73) 대비 **+11.85점**, Turn-Level(MPNet, 85.61) 대비 +4.97점 향상, 32K 컨텍스트 윈도우로 전체 이력 수용이 가능함에도 SeCom의 우위 유지.
- **GPT-4 Pairwise 비교 (LOCOMO, BM25)**: SeCom vs RecurSum 승률 **50.49%**(패 24.97%), vs MemoChat 승률 **44.21%**(패 31.94%), vs ConditionMem 승률 45.05%(패 29.43%), vs SumMem 승률 58.86%(패 22.18%).
- **세그멘테이션 성능 (비지도)**: DialSeg711에서 제로샷 F1 **0.888**, Score **0.895**로 기존 최선 비지도 모델 CSM(F1 0.610, Score 0.660) 대비 대폭 향상.
- **세그멘테이션 성능 (전이 학습)**: TIAGE→DialSeg711 전이에서 Reflection 적용 SeCom이 F1 **0.921**, Score **0.934**를 기록하여 RoBERTa 지도 학습(F1 0.650) 대비 큰 격차로 우위.
- **경량 세그멘테이션 모델 성능**: Long-MT-Bench+에서 Mistral-7B-Seg 사용 시 GPT4Score 86.32, RoBERTa-Seg 사용 시 81.52로 대부분 베이스라인 상회; LOCOMO에서도 RoBERTa-Seg가 GPT4Score 61.84를 유지.
- **효율성 비교 (Long-MT-Bench+)**: 입력 토큰 수 1,722(MemoChat 7,233 대비 4.2배 절감), 출력 토큰 135, 질문당 지연 2.61초로 MemoChat(5.60초)의 절반 수준이면서 GPT4Score는 88.81 vs 85.14로 우위.
- **컨텍스트 예산 민감도**: 검색 토큰 2K~8K 구간 전 범위에서 세그먼트 단위가 턴/세션 단위를 일관되게 능가(BM25 및 MPNet 검색기 모두).

## Limitation

세그멘테이션 모델의 성능에 전체 시스템이 민감하게 의존하며, GPT-4 기반 세그멘테이션에서 얻은 최고 성능이 RoBERTa-Seg로 대체될 경우 LOCOMO GPT4Score가 71.57에서 61.84로 9.73점 하락하여 자원 제약 환경에서 명확한 성능-비용 트레이드오프를 드러낸다.
LLMLingua-2의 압축률을 75%로 고정하였으며, 도메인이나 대화 특성에 따른 최적 압축률 탐색이 부재하여 압축률이 높을수록 정보 손실, 낮을수록 디노이징 효과 감소라는 이중 딜레마가 해소되지 않았다.
평가는 LOCOMO와 Long-MT-Bench+ 두 개의 영어 중심 벤치마크에서만 수행되어 다국어 대화, 의료/법률/금융 같은 도메인 특화 시나리오, 멀티모달 대화에서의 일반화 가능성은 검증되지 않았다.
대화가 스트리밍으로 진행될 때 새로운 세그먼트 경계를 실시간으로 결정해야 하는데, GPT-4 API 호출에 따른 지연과 비용이 실제 배포 시 병목이 될 수 있으며 논문은 오프라인 배치 세그멘테이션을 주로 가정한다.
세그먼트 단위 메모리와 요약 기반 접근을 상호 배타적으로 비교하였을 뿐, 세그먼트 내 핵심 정보를 선택적으로 요약하는 하이브리드 구조는 탐색되지 않아 잠재적 상승 여지가 남아 있다.
Reflection 학습은 100개의 주석 샘플을 전제로 하므로, 주석이 전무한 완전 제로샷 신규 도메인에서는 rubric 없이 기본 세그멘테이션 프롬프트에 의존해야 하고 최적 성능에 도달하기 어렵다.
검색된 세그먼트를 시간순으로 단순 연결하는 전략은 여러 세그먼트 간 상호 의존적 추론이 필요한 복합 질의(multi-hop reasoning)에서 한계를 보일 수 있으며, 그래프 구조나 순서 최적화는 다루지 않았다.
응답 생성기로 GPT-3.5-Turbo와 Mistral-7B만 평가하였고 GPT-4, Claude, Gemini 같은 최신 장문맥 모델에서의 성능 스케일링은 검증되지 않아, 메모리 관리 기법의 가치가 강력한 베이스 LLM에서도 유지되는지는 미지수이다.
