# SeCom: On Memory Construction and Retrieval for Personalized Conversational Agents

> **논문 정보**: Zhuoshi Pan, Qianhui Wu, Huiqiang Jiang, Xufang Luo, Hao Cheng, Dongsheng Li, Yuqing Yang, Chin-Yew Lin, H. Vicky Zhao, Lili Qiu, Jianfeng Gao (Tsinghua University, Microsoft)
> **arXiv**: 2502.05589 | **학회**: ICLR 2025
> **프로젝트**: https://aka.ms/secom

---

## Problem

장기 대화(long-term conversation)에서 개인화된 응답을 생성하려면 과거 대화 이력에서 관련 정보를 정확하게 검색하여 컨텍스트로 활용해야 한다. 기존 방법들은 대화 이력을 턴(turn) 단위, 세션(session) 단위, 또는 요약(summarization) 기반으로 메모리 뱅크를 구성하지만, 각각 고유한 한계를 지닌다. 턴 단위 메모리는 지나치게 세분화되어 검색 시 관련 정보가 여러 턴에 분산되면서 단편적이고 불완전한 컨텍스트를 제공한다. 세션 단위 메모리는 하나의 세션 내에 여러 토픽이 혼재할 경우 무관한 정보가 다량 포함되어 검색 정밀도가 떨어진다. 요약 기반 방법은 대화를 요약으로 변환하는 과정에서 QA에 필수적인 세부 정보가 손실되어 최적 이하의 응답을 생성한다. 이러한 메모리 단위(granularity) 선택 문제는 검색 정확도와 응답 품질 모두에 직접적인 영향을 미치며, 장기 대화 에이전트의 핵심 병목으로 작용한다.

---

## Motivation

대화는 본질적으로 토픽별로 일관된 담화 단위(topically coherent segments)로 구성된다는 관찰에서 출발한다. 턴 단위와 세션 단위 사이의 중간 세분도인 세그먼트(segment) 단위 메모리가, 관련 정보를 충분히 포함하면서도 무관한 내용을 배제하는 균형점을 제공할 수 있다. 또한 자연어에는 본질적인 중복성(redundancy)이 존재하며(Shannon, 1951), 이 중복성이 검색 시스템에 노이즈로 작용하여 핵심 정보 추출을 방해한다는 가설을 세운다. 프롬프트 압축 기법(LLMLingua-2)을 메모리 단위에 적용하면, 중복을 제거하는 디노이징(denoising) 효과를 통해 검색 정확도를 향상시킬 수 있다. 실제로 BM25와 MPNet 기반 검색기 모두에서 압축률 50% 이상일 때 검색 재현율이 일관되게 개선되며, 쿼리와 관련 세그먼트 간 유사도는 증가하고 무관 세그먼트와의 유사도는 감소하는 것을 확인하였다. 이 두 가지 통찰(세그먼트 단위 메모리 + 압축 기반 디노이징)을 결합하면, 기존 방법들의 한계를 동시에 극복할 수 있다.

## Method

1. **메모리 뱅크 구성 프레임워크**: 대화 이력 H = {c_i}를 메모리 뱅크 M으로 변환한 뒤, 검색 시스템 f_R로 관련 메모리 유닛을 검색하고, LLM f_LLM이 검색된 컨텍스트를 기반으로 응답을 생성하는 3단계 파이프라인(Memory Construction → Memory Retrieval → Response Generation)을 정의한다.

2. **대화 세그멘테이션 모델**: 각 세션 c_i를 토픽별로 일관된 세그먼트 {s_k}로 분할한다. 세그멘테이션 모델 f_I는 각 세그먼트의 시작/끝 턴 인덱스를 식별한다.

3. **제로샷 세그멘테이션**: GPT-4를 세그멘테이션 모델로 활용하며, 각 턴에 턴 인덱스와 역할 식별자를 추가하여 명확한 컨텍스트를 제공한다. 경량 모델(Mistral-7B, RoBERTa)로도 세그멘테이션이 가능함을 실험적으로 검증하였다.

4. **Reflection 기반 세그멘테이션 개선**: 소량의 주석 데이터가 있을 경우, 제로샷 결과에서 WindowDiff 기준 오류가 가장 큰 상위 K개 "hard examples"를 선별하고, LLM이 ground-truth와 비교하여 세그멘테이션 가이드라인을 반복적으로 개선한다.

5. **세그먼트 단위 메모리 뱅크 구성**: 세그멘테이션 결과로 생성된 세그먼트 각각이 하나의 메모리 유닛이 되어 메모리 뱅크를 구성한다. 요약 과정 없이 원본 대화를 세그먼트 단위로 직접 저장하여 정보 손실을 방지한다.

6. **압축 기반 메모리 디노이징**: 검색 전 LLMLingua-2(압축률 75%, xlm-roberta-large 기반)를 적용하여 메모리 유닛의 자연어 중복을 제거한다. 플러그 앤 플레이 방식으로 별도 파인튜닝 없이 어떤 검색기에도 적용 가능하다.

7. **메모리 검색 및 응답 생성**: MPNet(multi-qa-mpnet-base-dot-v1) + FAISS 또는 BM25로 디노이징된 메모리 뱅크에서 상위 N개 세그먼트를 검색한 뒤, 시간순으로 정렬하여 LLM에 컨텍스트로 제공하여 응답을 생성한다.

---

## Key Contribution

1. **메모리 세분도(granularity)의 체계적 분석**: 턴/세션/요약 기반 메모리 각각의 검색 정확도 및 응답 품질에 대한 한계를 최초로 체계적으로 규명하고, 세그먼트 단위가 최적의 균형점임을 실증.
2. **압축 기반 디노이징의 발견**: 프롬프트 압축 기법(LLMLingua-2)이 메모리 검색 시스템의 효과적인 디노이징 메커니즘으로 작동함을 최초로 입증.
3. **SeCom 시스템 제안**: 대화 세그멘테이션 + 압축 기반 디노이징을 결합한 메모리 관리 시스템으로, LOCOMO 및 Long-MT-Bench+에서 SOTA 달성.
4. **경량 모델 호환성**: GPT-4뿐 아니라 Mistral-7B, RoBERTa 규모 모델로도 세그멘테이션이 가능하여 자원 제약 환경에서도 적용 가능.
5. **Reflection 기반 세그멘테이션 학습**: 100개 예제만으로 LLM이 세그멘테이션 가이드라인을 자기 반성 방식으로 학습하여, 전체 학습 데이터로 훈련한 지도 학습 모델을 능가.

## Experiment & Results

- **LOCOMO 벤치마크**: SeCom(BM25, GPT4-Seg)이 GPT4Score **71.57**을 달성하여, 최선 베이스라인 ConditionMem(65.92) 대비 **+5.65점** 향상. Turn-Level(BM25) 65.58, Session-Level(BM25) 63.16, Full History 54.15를 모두 상회.
- **Long-MT-Bench+**: SeCom(MPNet, GPT4-Seg)이 GPT4Score **88.81**로 최고 성능 달성. MemoChat(85.14), Turn-Level(MPNet, 84.91) 대비 각각 +3.67, +3.90점 우위.
- **디노이징 효과**: MPNet 기반 SeCom에서 디노이징 제거 시 LOCOMO GPT4Score가 69.33 → **59.87로 9.46점 하락**, Long-MT-Bench+에서도 88.81 → 87.51로 1.30점 하락.
- **Mistral-7B 응답 생성기 사용 시**: SeCom(MPNet)이 GPT4Score **90.58**을 달성하여, Full History(78.73) 대비 **+11.85점**, Turn-Level(MPNet, 85.61) 대비 +4.97점 향상. 32K 컨텍스트 윈도우로 전체 이력을 수용 가능함에도 SeCom이 우세.
- **세그멘테이션 성능**: DialSeg711에서 제로샷 F1 **0.888**, Score **0.895**로 기존 최선 비지도 모델 CSM(F1 0.610, Score 0.660) 대비 대폭 향상. Reflection 적용 시 Score **0.936**까지 상승하여 지도 학습 모델도 능가.
- **GPT-4 Pairwise 비교**: LOCOMO에서 SeCom vs RecurSum 승률 **50.49%**(패 24.97%), vs MemoChat 승률 **44.21%**(패 31.94%).
- **경량 세그멘테이션 모델**: Mistral-7B-Seg 사용 시 Long-MT-Bench+ GPT4Score 86.32, RoBERTa-Seg 사용 시에도 81.52로 대부분의 베이스라인을 상회.

## Limitation

- **세그멘테이션 모델 의존성**: 최고 성능은 GPT-4 기반 세그멘테이션에서 달성되며, 경량 모델(RoBERTa)로 대체 시 성능 하락이 존재한다(LOCOMO GPT4Score: 71.57 → 61.84). 이는 자원 제약 환경에서의 성능-비용 트레이드오프를 발생시킨다.
- **압축률 민감성**: LLMLingua-2의 압축률을 75%로 고정하였으며, 도메인이나 대화 특성에 따른 최적 압축률 탐색이 부재하다. 압축률이 높을수록 정보 손실, 낮을수록 디노이징 효과 감소의 딜레마가 존재한다.
- **평가 범위**: LOCOMO와 Long-MT-Bench+ 두 개의 영어 벤치마크에서만 평가되었으며, 다국어 대화나 도메인 특화 시나리오(의료, 법률 등)에서의 일반화 가능성은 검증되지 않았다.
- **실시간 세그멘테이션 비용**: 대화가 진행됨에 따라 새로운 세그먼트 경계를 실시간으로 결정해야 하는데, GPT-4 호출에 따른 지연 시간과 API 비용이 실제 배포 시 병목이 될 수 있다.
- **요약과의 결합 미탐색**: 세그먼트 단위 메모리와 요약 기반 접근을 상호 배타적으로 비교하였으나, 세그먼트 내 핵심 정보를 선택적으로 요약하는 하이브리드 접근은 시도하지 않았다.
