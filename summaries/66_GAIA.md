# GAIA: A Benchmark for General AI Assistants

> **논문 정보**: Grégoire Mialon, Clémentine Fourrier, Craig Swift, Thomas Wolf, Yann LeCun, Thomas Scialom (FAIR Meta, HuggingFace, AutoGPT)
> **arXiv**: 2311.12983 (2023.11) | **학회**: ICLR 2024
> **코드**: https://huggingface.co/gaia-benchmark

---

## Overview

| 항목 | 내용 |
|------|------|
| **Problem** | 기존 벤치마크(MMLU, GSM8k 등)는 단일 기술에 집중하거나 폐쇄 환경으로 제한되어 범용 AI 어시스턴트의 실세계 복합 태스크 능력을 반영하지 못한다. 객관식 포맷은 데이터 오염에 취약하다. |
| **Motivation** | 웹 브라우징, 멀티모달 이해, 코드 실행, 다단계 추론을 동시에 요구하는 실세계 질문에서 인간 92% vs GPT-4+플러그인 15%의 격차는 AGI까지의 간극을 정량화하는 벤치마크의 필요성을 보여준다. |
| **Limitation** | 466개로 규모 제한. 영어 전용. 정답 추론 과정(trace) 미평가. 웹 링크 소멸로 질문 노후화 가능. 플러그인 재현성 문제. |

---

## Method

### 1. 설계 원칙
- **실세계성**: 사전학습 데이터에 평문 정답이 없도록 설계하여 메모리제이션 방지
- **해석 가능성**: 비전문가 성공률 92%의 개념적 단순 태스크
- **비-게임어빌리티**: 다단계 완료 필수, 개방형 단답으로 우연 정답 불가
- **사용 편의성**: 팩토이드 단답으로 자동 채점 가능

### 2. 질문 구성·검증
- 저자 설계 + 어노테이터 추가 생성, 독립 2명 교차 검증, 3명 일치 시 승인
- 68% 그대로 통과, 질문당 평균 ~2시간 작업

### 3. 3단계 난이도
- **Level 1**: 도구 0~1개, 5단계 이하
- **Level 2**: 여러 도구 결합, 5~10단계
- **Level 3**: 임의 길이 행동 연쇄, 임의 도구 조합

### 4. 요구 능력 분포
- 웹 브라우징 ~355개, 멀티모달 ~154개, 코딩 ~138개, 파일 처리 ~129개

### 5. 평가
- 정규화된 준정확 매칭으로 자동 채점, 3회 실행 평균
- Dev 166개(공개) + 리더보드 300개(비공개) 분리

---

## Key Contribution

1. **AGI 측정 기준점**: 인간 92% vs GPT-4+플러그인 15% — Level 3에서 GPT-4 ~0%
2. **비-게임어빌리티 방법론**: 데이터 오염·우연 정답 원천 차단
3. **후속 연구의 표준 벤치마크**: OWL, WORKFORCE 등에서 채택
4. **도구 증강 LLM의 잠재력 실증**: 플러그인이 핵심 방향임을 검증

---

## Experiment & Results

| 모델 | Level 1 | Level 2 | Level 3 | 전체 |
|------|---------|---------|---------|------|
| 인간 | ~92% | ~92% | ~92% | **92%** |
| GPT-4 + 플러그인 | ~30% | ~15% | ~0% | **~15%** |
| GPT-4 (플러그인 없음) | ~10% | ~5% | ~0% | - |
| AutoGPT | ~5% | 매우 낮음 | ~0% | **~5%** |

- **Level 3**: 모든 GPT-4 변형에서 ~0% — 10단계+ 복합 도구 사용이 절대적 한계
- **AutoGPT 역설**: 자동 도구 선택 능력에도 GPT-4 단독보다 낮은 성능 → 자율 오케스트레이션의 현 한계
- 인간 소요 시간: Level 1 ~6분, Level 3 ~17분
- 질문 검증: 68% 수정 없이 통과, 32% 수정/삭제
