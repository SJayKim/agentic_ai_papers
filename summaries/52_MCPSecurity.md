## Overview

| 항목 | 내용 |
|------|------|
| **Problem** | Anthropic이 주도한 MCP(Model Context Protocol)가 빠르게 확산되고 있지만, 보안 위협에 대한 체계적 분석이 부재하다. MCP 서버와 클라이언트 간의 통신, 도구 실행, 데이터 접근에서 다양한 공격 벡터가 존재. |
| **Motivation** | MCP가 AI 에이전트의 표준 프로토콜로 자리잡으면서, 프롬프트 인젝션, 도구 하이재킹, 데이터 유출 등의 보안 위협이 현실화. 4단계 라이프사이클별 위협 분류와 대응 전략이 필요. |
| **Limitation** | 저자 언급: 실제 공격 시연이 제한적이며 이론적 분석 위주. 독자 관점: MCP 사양이 빠르게 변화하므로 분석의 유효 기간이 짧을 수 있음. |

---

## Method

1. **4단계 MCP 라이프사이클 정의**: 설정(Configuration) → 연결(Connection) → 실행(Execution) → 종료(Termination)
2. **16개 활동 식별**: 라이프사이클 전반에 걸친 구체적 활동 분류
3. **4가지 공격자 유형**: 외부 공격자, 악의적 MCP 서버, 악의적 클라이언트, 내부 위협
4. **위협 분류 체계**: 라이프사이클 단계별 × 공격자 유형별 위협 매트릭스 구축

---

## Key Contribution

1. **MCP 최초의 포괄적 보안 분석**: 4단계 라이프사이클, 16개 활동, 4가지 공격자 유형의 체계적 분류
2. **실행 가능한 보안 대책**: 각 라이프사이클 단계별 구체적 방어 전략 제안

## Experiment & Results

- 서베이 논문으로, 위협 사례 분석과 방어 전략 제안에 초점
