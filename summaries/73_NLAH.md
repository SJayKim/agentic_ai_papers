# Natural-Language Agent Harnesses

> **논문 정보**: Linyue Pan, Lexiao Zou, Shuo Guo, Jingchen Ni, Hai-Tao Zheng (Tsinghua University Shenzhen International Graduate School, Harbin Institute of Technology Shenzhen)
> **arXiv**: 2603.25723 (2026.03)
> **코드**: N/A

---

## Problem

- 현대 에이전트 시스템의 성능은 모델 자체보다 주변의 **harness**(제어 스택)에 점점 더 의존하고 있으며, 하네스는 다단계 추론, 도구 사용, 메모리, 위임, 정지 조건 등을 제어하는 핵심 레이어이다.
- 기존 하네스 설계는 컨트롤러 코드, 프레임워크 기본 설정, 도구 어댑터, 검증 스크립트, 런타임별 가정 등에 **분산·은닉**되어 있어 하나의 일관된 이식 가능한 아티팩트로 노출되지 못한다.
- 이로 인해 하네스를 **다른 런타임으로 이식(transfer)하기 어렵고**, 특정 프레임워크와 강하게 결합되어 재사용이 불가능하다.
- 두 시스템 간 **공정한 비교가 불가능**하다: 명목상 하나의 설계 선택만 다른 두 시스템도 실제로는 프롬프트, 도구 중재, 아티팩트 규약, 검증 게이트, 상태 시맨틱스가 동시에 달라져 "컨트롤러 번들 비교"로 전락한다.
- **클린한 ablation을 수행할 수 없다**: 하네스의 개별 모듈이 성능에 미치는 영향을 독립적으로 측정할 방법이 없어, 어떤 설계 선택이 실제로 효과적인지 과학적으로 검증하기 어렵다.
- 장기 실행(long-horizon) 에이전트 환경에서 상태 관리, 컨텍스트 큐레이션 등의 제어 스택이 기본 모델이 고정되어 있어도 **성능 병목**이 되는 현상이 반복적으로 관찰된다.
- scaffold-aware 평가에서 동일한 기본 모델 하에서도 scaffold/하네스의 차이가 결과를 지배하는 현상이 보고되고 있다.
- 결과적으로 하네스 설계의 각 모듈이 독립적으로 연구·비교 가능한 **과학적 대상(scientific object)**으로 취급되지 못하고 있다는 것이 핵심 문제이다.

## Motivation

- AGENTS.md, AgentSkills 등 자연어 아티팩트가 저장소 로컬 규약과 재사용 가능한 절차를 **이식 가능한 텍스트로 패키징**할 수 있음을 이미 보여주었다.
- 최근 연구들은 경험 기반 스킬 생성, 컨텍스트 엔지니어링 스킬 진화, 재사용 가능한 절차적 메모리, 크로스 태스크 스킬 평가 등을 다루며 자연어 제어 지식의 범위를 확장하고 있다.
- **그러나 기존 접근은 한계가 명확하다**: 이들은 재사용 가능한 제어 지식의 실현 가능성을 보여줄 뿐, **하네스 전체의 계약(contract), 역할 경계, 상태 시맨틱스, 실패 처리, 런타임 어댑터**를 명시적·통합적으로 실행 가능한 표현으로 만들지는 못한다.
- 기존 방식은 주로 로컬 명령어나 재사용 루틴을 첨부하는 수준이며, 하네스 수준의 계약을 공유 런타임 아래에서 jointly executable하게 만드는 것과는 근본적으로 다르다.
- 저자들은 자연어를 단순한 재사용 절차의 운반체에서 **명시적이고 실행 가능한 하네스 객체(executable harness object)**로 격상시키는 것이 가능하다는 관찰에 기반한다.
- "프롬프트 엔지니어링"을 넘어 **컨텍스트 엔지니어링**의 중요성이 강조되는 현 시점에서, 각 단계마다 어떤 명령어·증거·중간 아티팩트·상태를 제공할지를 체계적으로 설계하는 것이 성능의 핵심이 되고 있다.
- 실무 경험들도 장기 실행 태스크에서 일회성 프롬프트 문구보다 **내구적 상태 표면(durable state surfaces), 검증 게이트, 명확한 책임 경계**가 더 중요하다는 점을 반복 강조하고 있다.
- 따라서 하네스 패턴 로직을 공유 런타임 하에서 실행 가능한 자연어 객체로 외재화하는 것은, 이미 증명된 이식 가능한 스킬 개념의 자연스러운 확장이다.

## Method

본 논문은 두 가지 핵심 구성요소를 제안한다:

1. **Natural-Language Agent Harness (NLAH)**: 하네스 제어 로직을 구조화된 자연어로 표현한 **이식 가능한 실행 아티팩트**
   - **Contracts**: 필수 입출력, 포맷 제약, 검증 게이트, 권한 경계, 재시도 및 중지 규칙
   - **Roles**: solver, verifier, researcher, orchestrator 등 비중복 책임을 가진 역할 프롬프트
   - **Stage structure**: 명시적 작업 토폴로지 (예: plan → execute → verify → repair)
   - **Adapters & scripts**: 결정론적 동작(테스트, 검증기, 파싱)을 위한 명명된 훅
   - **State semantics**: 단계 간 지속되는 아티팩트, 원장(ledger), 자식 워크스페이스 및 재개 방식
   - **Failure taxonomy**: 복구를 유도하는 명명된 실패 모드 (아티팩트 누락, 검증 실패, 도구 오류, 타임아웃)

2. **Intelligent Harness Runtime (IHR)**: NLAH를 직접 해석·실행하는 공유 런타임
   - **(a) In-loop LLM**: 각 단계에서 하네스, 현재 상태, 런타임 차터를 읽고 계약·예산에 부합하는 다음 행동을 선택
   - **(b) Backend**: 터미널 도구와 first-class 멀티에이전트 인터페이스 제공 (spawn_agent, wait_agent 등으로 자식 에이전트 생성·감독)
   - **(c) Runtime charter**: 계약, 상태, 오케스트레이션, 자식 생명주기의 시맨틱스를 정의하는 공유 정책 문서

3. **File-backed state 모듈**: 장기 자율성을 위한 선택적 모듈
   - **Externalized**: 상태를 일시적 컨텍스트가 아닌 아티팩트로 기록
   - **Path-addressable**: 후속 단계가 경로로 정확한 객체를 재개
   - **Compaction-stable**: 컨텍스트 절단, 재시작, 위임 시에도 상태 유지

4. **Agent call 개념**: 단일 completion을 명시적 실행 계약(필수 출력, 예산, 권한 범위, 완료 조건, 지정 출력 경로)으로 바운딩한 **에이전트 콜**로 격상

실험에서 IHR의 백엔드는 Codex CLI(v0.114.0) + GPT-5.4로 구현되었으며, 런타임 차터는 고정 runtime skill, 벤치마크별 하네스 로직은 harness skill로 분리하여 제어된 ablation을 가능하게 했다.

## Key Contribution

1. **Formulation**: 하네스 설계 패턴 레이어를 런타임 정책 및 저수준 실행 훅과 구별되는 **명시적 표현 객체**로 형식화
2. **Representation ingredients**: 실행 가능한 자연어 하네스가 노출해야 하는 구성요소(계약, 역할, 단계 구조, 어댑터, 상태 시맨틱스, 실패 분류)를 체계적으로 명세
3. **Intelligent Harness Runtime (IHR)**: 런타임 차터와 하네스 로직을 깨끗하게 분리하면서 하네스 로직을 직접 해석하는 **인-루프 LLM 런타임** 도입
4. **Controlled evidence**: 공유 런타임 행동 효과(RQ1), 모듈 구성·ablation(RQ2), 코드→텍스트 하네스 마이그레이션 충실도(RQ3)에 대한 **제어된 실험적 증거** 제공
5. **Harness representation science의 방향 제시**: 하네스 모듈을 모델 주변의 부수적 접착제가 아닌 **일급 연구 아티팩트**로 만드는 경로 제안

## Experiment & Results

**벤치마크**: SWE-bench Verified (코딩, 125개 샘플) 및 OSWorld (컴퓨터 사용, 36개 샘플).
모든 실험은 Codex CLI v0.114.0, GPT-5.4, reasoning effort xhigh, Ubuntu 24.04 (64 CPU, 251 GiB RAM) Docker 환경에서 수행.

**RQ1 (Behavioral Effect)**:
- TRAE 하네스: Full IHR **74.4%**, w/o Runtime Skill **76.0%**, w/o Harness Skill **75.2%** — 성능 대역은 좁지만, **프로세스 지표가 극적으로 변화**
- Full IHR: 프롬프트 토큰 16.3M, 도구 호출 642.6회, 런타임 32.5분 vs. w/o HS: 1.2M 토큰, 51.1회, 6.7분
- TRAE Full IHR에서 프롬프트 토큰의 **91.5%**, 도구 호출의 **90.2%**가 위임된 자식 에이전트에서 발생
- 125개 SWE 샘플 중 **110개 이상**이 Full IHR과 ablation 간 동일한 결과 — 의미 있는 차이는 소수의 경계 사례에 집중
- Full IHR은 **solved-set replacer**로 작동: 일부 Full-only 성공을 만들지만 가벼운 설정이 유지하는 직접 경로 수리도 일부 상실

**RQ2 (Module Ablation)**:
- SWE-bench: Basic **75.2%** → File-backed state **76.8%**(+1.6) → Self-evolution **80.0%**(+4.8, 최고) → Multi-candidate search **72.8%**(-2.4) → Verifier **74.4%**(-0.8)
- OSWorld: Basic **41.7%** → File-backed state **47.2%**(+5.5, 최대 개선) → Verifier **33.3%**(-8.4, 최대 하락) → Self-evolution **44.4%**(+2.7)
- Self-evolution의 주요 효과는 **수용 기준 게이트 기반의 규율 있는 시도 루프**로 탐색을 좁히는 것
- **"더 많은 구조가 항상 더 나은 것은 아님"**: 중간 행동과 평가자 수용 조건 사이의 경로를 좁히는 모듈만 효과적

**RQ3 (Code-to-Text Migration)**:
- OSWorld OS-Symphony: 네이티브 코드 **30.4%** vs. 마이그레이션된 NLAH **47.2%** — NLAH가 **+16.8pp** 우위
- 네이티브 하네스는 스크린샷 기반 GUI 수리 루프에 갇히는 반면, NLAH 실현은 파일/셸/패키지 수준 작업으로 전환하여 더 강한 완료 증명서를 제공
- 네이티브 메인 트레이스 평균 **18.1 스텝** vs. 마이그레이션 트레이스 평균 **18.2 고유 명령 + 58.5 총 로그 이벤트**

## Limitation

- **자연어의 정밀성 한계**: 자연어는 코드보다 정밀성이 떨어지며, 에이전트 제어 로직의 엣지 케이스를 완전히 명세하기 어렵다.
- **은닉 상태 복원 불가**: 숨겨진 서비스 측 상태, 독점 스케줄러, 학습 유도 행동에 의존하는 하네스 메커니즘은 텍스트에서 충실하게 복원할 수 없다.
- **런타임 오염(Runtime contamination) 위험**: 강력한 공유 런타임 차터가 하네스 텍스트에 귀속될 행동의 일부를 흡수할 수 있어, 하네스 자체의 기여를 과대·과소 평가할 수 있다.
- **인과적 식별의 한계**: 모듈 수준 ablation은 엄격한 인과 식별이 아니며, 텍스트 표현이 명령어 현저성(instruction salience)이나 프롬프트 길이 등의 교란 변수를 도입할 수 있다.
- **제한된 실험 규모**: 현재 결과는 예산 제약으로 인해 SWE-bench 125개, OSWorld 36개의 **부분 집합**에서만 보고되었다.
- **위험한 워크플로우 확산**: 이식 가능한 하네스 로직과 스크립트가 위험한 워크플로우의 진입 장벽을 낮출 수 있다.
- **새로운 공격 면(attack surface)**: 프롬프트 인젝션, 악의적 도구 이식, 공급망 오염 등 하네스가 도구 사용·아티팩트 처리·위임을 중재하기 때문에 발생하는 새로운 보안 위협이 존재한다.
