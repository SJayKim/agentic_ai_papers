# ALMA: Learning to Continually Learn via Meta-learning Agentic Memory Designs

## 필수 요소

| 항목 | 내용 |
|------|------|
| **Problem** | Foundation Model(FM) 기반 에이전트 시스템은 추론 시 상태가 없어(stateless) 과거 경험을 축적하고 지속적으로 학습하기 어렵다. 기존 해결책인 메모리 모듈은 사람이 직접 설계(hand-crafted)하기 때문에 도메인마다 별도의 설계가 필요하고, 태스크의 다양성과 비정상성(non-stationarity)에 적응하지 못한다. 예를 들어 대화 에이전트는 사용자 선호도를 저장하는 메모리가 필요하고, 전략 게임은 추상적 전략을 저장하는 메모리가 필요한데, 이를 각각 수동으로 설계하는 것은 노동 집약적이고 확장성이 없다. |
| **Motivation** | 머신러닝 역사에서 수작업 컴포넌트는 결국 학습된 컴포넌트로 대체되어 왔다(예: 수작업 피처 → 딥러닝 표현, NAS를 통한 신경망 구조 자동 탐색). 메모리 설계도 이 패러다임을 따라야 한다는 것이 동기다. 실제로 G-Memory, Reasoning Bank, Dynamic Cheatsheet 등 다양한 수작업 메모리 설계들이 존재하지만, 이들은 특정 도메인에 최적화되어 있어 다른 도메인에서 성능이 저하되거나 비효율적이다. |
| **Method** | **ALMA(Automated meta-Learning of Memory designs for Agentic systems)**는 Meta Agent가 코드 공간에서 메모리 설계를 탐색하는 오픈엔드 탐색(open-ended exploration) 프레임워크다. 핵심 파이프라인: (1) **Memory Design Archive**: 탐색된 모든 메모리 설계와 평가 로그를 저장하는 아카이브. (2) **Ideate & Plan**: Meta Agent가 아카이브에서 최대 5개의 메모리 설계를 샘플링하고, 이들의 코드와 평가 로그를 반영하여 새로운 메모리 설계 아이디어와 계획을 제안한다. (3) **Implement**: Meta Agent가 계획을 Python 코드로 구현한다. 검색 공간은 Python Abstract Class 기반이며, 서브모듈(`Sub_memo_layer`)과 전체 메모리 구조(`MemoStructure`)를 구현한다. (4) **Evaluate**: 새로운 메모리 설계를 샌드박스 환경에서 실행하여 성능을 평가하고 결과를 아카이브에 추가한다. 평가는 Memory Collection Phase(과거 궤적으로 메모리 구축)와 Deployment Phase(메모리를 활용해 태스크 수행)의 2단계로 진행한다. 11번의 학습 스텝 동안 총 43개의 메모리 설계를 탐색한다. Meta Agent는 GPT-5를 사용하고, 에이전트 시스템 평가에는 GPT-5-nano를 사용한다. 메모리 설계 내부에서 사용 가능한 도구로 GPT-4o-mini, GPT-4.1, text-embedding-3-small이 제공된다. |
| **Key Contribution** | (1) **메모리 설계의 자동화**: 수작업 메모리 설계를 Meta Agent가 자동으로 탐색하는 최초의 프레임워크로, 이론적으로 임의의 메모리 설계(데이터베이스 스키마, 검색/업데이트 메커니즘 포함)를 발견할 수 있다. (2) **코드 공간 탐색**: 메모리 설계를 실행 가능한 코드로 표현하여 표현력을 극대화한다. (3) **오픈엔드 탐색**: 탐욕적 탐색(greedy search) 대신 오픈엔드 탐색을 채택하여 중간 성능의 설계를 발판 삼아 더 나은 설계로 진화할 수 있도록 한다. 기존 유사 연구(Zhang et al., 2025d)는 다수의 기존 수작업 메모리 설계로 초기화하고 탐욕적 선택에 의존하는 반면, ALMA는 처음부터(from scratch) 오픈엔드 탐색을 수행한다. (4) **지속적 학습 최적화**: 기존 자동 에이전트 설계 연구(ADAS, AgentSquare 등)가 일회성(one-shot) 성능만 평가하는 반면, ALMA는 과거 경험으로부터의 지속적 학습 능력을 명시적으로 최적화한다. |
| **Experiment/Results** | 4개의 순차적 의사결정 벤치마크에서 평가: ALFWorld(텍스트 기반 가정 환경 태스크), TextWorld(텍스트 어드벤처 게임), Baba Is AI(규칙 조작 전략 퍼즐), MiniHack(절차적으로 생성된 던전 환경). **GPT-5-nano 기준(Table 1 상단)**: ALMA는 전체 평균 성공률 12.3%로, 메모리 없음(6.1%) 대비 +6.2%p 향상. 모든 수작업 기준선(Trajectory Retrieval: 8.6%, Reasoning Bank: 7.5%, Dynamic Cheatsheet: 7.2%, G-Memory: 7.7%)을 상회. 세부적으로 ALFWorld: 12.4%(기준선 최고 7.6% 대비 +4.8%p), TextWorld: 6.2%, Baba Is AI: 19.0%, MiniHack: 11.7%. **GPT-5-mini 전이(Table 1 하단)**: 더 강력한 FM으로 전이 시 전체 평균 53.9%로 메모리 없음(41.1%) 대비 +12.8%p 향상, 모든 수작업 기준선 상회(Trajectory Retrieval: 48.6%, Reasoning Bank: 40.1%, Dynamic Cheatsheet: 46.5%, G-Memory: 46.0%). ALFWorld 성공률 87.1%(기준선 최고 80.0% 대비 +7.1%p). **비용 효율**: ALMA의 end-to-end 메모리 비용은 $0.09로 대부분의 수작업 기준선보다 낮으며, 검색된 컨텍스트 토큰 수도 1,319 토큰으로 Trajectory Retrieval(9,149)나 G-Memory(6,095)보다 훨씬 적다. **스케일링**: Memory Collection Phase의 태스크 수가 증가할수록 ALMA는 더 빠르게 성능이 향상되고 더 높은 성능에 도달한다. **분포 변화 적응**: 학습에서 보지 못한 태스크 분포에서도 ALMA가 가장 효과적으로 적응한다(ALFWorld 동적 모드 결과). |
| **Limitation** | (1) **FM 의존성**: 메모리 설계가 코드 공간에서 학습되므로, 메모리 설계 내 기능이 기반 FM의 능력에 제한된다. 미래 연구에서는 네이티브 메모리 지원이 있는 FM 아키텍처의 자동 설계와 학습을 탐색할 수 있다. (2) **메모리와 에이전트 시스템의 공동 학습 불가**: 현재는 메모리 설계만 학습하며, 에이전트 시스템과 메모리를 동시에 학습하는 시스템을 시연하지 못한다. (3) **안전 우려**: AI-generating 알고리즘으로서 학습된 컴포넌트가 사람의 의도에서 벗어난 행동을 의도치 않게 도입할 수 있다. 현재는 실험 설계에서 샌드박스 환경을 통해 메모리 설계 실행을 격리하는 방식으로 완화하고 있다. (4) **비용 최적화 미흡**: 현재 알고리즘은 비용 효율성을 명시적으로 최적화하지 않아, 다중 목적 최적화 등의 기법 적용이 향후 과제로 남는다. |

## 선택 요소 (해당되는 것만)

| 항목 | 내용 |
|------|------|
| **Baseline 비교** | 4개의 수작업 메모리 기준선과 비교: (1) **Trajectory Retrieval** (Park et al., 2023; Xu et al., 2024): 유사 궤적을 직접 검색하는 단순하지만 강력한 방법. GPT-5-nano 기준 전체 평균 8.6%. (2) **Reasoning Bank** (Ouyang et al., 2025): 태스크 단위 경험 생성 방식. 전체 평균 7.5%. (3) **Dynamic Cheatsheet** (Suzgun et al., 2025): 적응형 메모리를 사용하는 테스트 시간 학습 방법. 전체 평균 7.2%. (4) **G-Memory** (Zhang et al., 2025a): 그래프 데이터베이스에 인사이트를 저장하고 그래프 순회로 검색하는 방법. 전체 평균 7.7%. ALMA(전체 평균 12.3%)는 GPT-5-nano 기준 최고 기준선(Trajectory Retrieval: 8.6%)보다 +3.7%p, GPT-5-mini 전이 기준 최고 기준선(Trajectory Retrieval: 48.6%)보다 +5.3%p 높은 성능을 달성. |
| **Ablation Study** | **오픈엔드 탐색 vs. 탐욕적 탐색(Greedy Search)**: ALFWorld에서 비교 실험 수행. 동일한 탐색 예산(43개 메모리 설계 탐색) 하에서, ALMA(오픈엔드 탐색)는 GPT-5-nano 기준 12.4±0.5%, GPT-5-mini 기준 87.1±1.4%를 달성한 반면, Greedy Search는 GPT-5-nano 기준 11.9±0.5%, GPT-5-mini 기준 77.1±0.8%에 그쳤다. 특히 GPT-5-mini 기준 성능 차이가 10%p로 크게 벌어져, 오픈엔드 탐색이 더 일반화 가능한 메모리 설계를 발견함을 보여준다. 탐욕적 탐색은 단일 계보(lineage)로 수렴하는 반면, 오픈엔드 탐색은 중간 성능의 설계들을 발판 삼아 다양한 방향을 탐색하여 궁극적으로 더 나은 최적 설계에 도달한다. |
