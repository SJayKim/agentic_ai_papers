# LightRAG: Simple and Fast Retrieval-Augmented Generation

> **논문 정보**: Zirui Guo, Lianghao Xia, Yanhua Yu, Tu Ao, Chao Huang (BUPT, University of Hong Kong)
> **arXiv**: 2410.05779 (2024.10)
> **코드**: https://github.com/HKUDS/LightRAG

---

## Overview

| 항목 | 내용 |
|------|------|
| **Problem** | 기존 RAG 시스템은 (1) 평면적(flat) 데이터 표현에 의존하여 엔티티 간 복잡한 상호 의존성을 이해하지 못하고, (2) 문맥 인식 부족으로 여러 엔티티와 관계에 걸친 일관된 응답을 생성하지 못하여 단편적 답변을 제공한다. GraphRAG는 이를 해결하지만 토큰 비용이 높고 증분 업데이트가 불가능. |
| **Motivation** | "전기차 보급이 도시 대기질과 대중교통 인프라에 미치는 영향은?" 같은 질문에 기존 RAG는 각 주제를 별도로 검색하여 통합 응답을 만들지 못한다. 그래프 구조로 엔티티 간 관계를 모델링하면 이를 해결할 수 있지만, 빠르고 확장 가능한 구현이 필요. |
| **Limitation** | 저자 언급: LLM 기반 엔티티 추출의 품질에 의존. 독자 관점: 이중 레벨 검색의 "high-level"이 추상적 키워드 매칭에 의존하여, 세밀한 의미 추론이 필요한 질문에서 검색 정확도 한계. 커뮤니티 요약 없이 엔티티/관계 수준에서만 검색하므로 진정한 "글로벌 센스메이킹"은 GraphRAG 대비 약할 수 있음. |

## Method

1. **그래프 기반 텍스트 인덱싱**
   - 문서를 청크로 분할 후 LLM으로 엔티티와 관계 추출
   - 추출된 엔티티를 LLM 프로파일링으로 중복 제거(deduplication)
   - 결과: 지식 그래프 D̂ = (V̂, Ê) 구축

2. **이중 레벨 검색 패러다임 (Dual-Level Retrieval)**
   - **Low-level 검색**: 특정 엔티티와 그 직접적 관계에 초점. 구체적 사실(인물, 날짜, 위치 등) 질문에 적합
   - **High-level 검색**: 추상적 개념과 넓은 주제에 초점. 키워드를 통해 관련 엔티티 그룹과 관계를 검색. 종합적 분석 질문에 적합
   - 그래프 구조와 벡터 표현을 결합하여 관련 엔티티·관계를 효율적으로 검색

3. **증분 업데이트 알고리즘**
   - 새 문서 D'에 대해 동일한 인덱싱 파이프라인 적용: D̂' = (V̂', Ê')
   - 기존 그래프와 노드·엣지 집합의 합집합으로 통합: V̂ ∪ V̂', Ê ∪ Ê'
   - **전체 재구축 불필요** — GraphRAG의 핵심 비용 문제 해결

4. **키-값 구조 기반 빠른 검색**: 엔티티와 관계를 키-값 쌍으로 저장하여 임베딩 매칭 없이 빠른 검색 가능

## Key Contribution

1. **경량화된 Graph RAG**: GraphRAG의 커뮤니티 탐지·계층적 요약 대신, 엔티티/관계 수준의 이중 레벨 검색으로 비용을 대폭 절감하면서 관계 인식 검색 유지
2. **증분 업데이트**: 새 데이터 추가 시 전체 인덱스 재구축 없이 그래프를 확장하는 알고리즘으로, 동적 환경에서의 실용성 확보
3. **이중 레벨 검색**: 구체적(low-level) + 추상적(high-level) 질문을 모두 커버하는 유연한 검색 전략

## Experiment & Results

- **데이터셋**: 4개 도메인의 텍스트 코퍼스 (Agriculture, CS, Legal, Mixed)
- **비교**: NaiveRAG, RQ-RAG, GraphRAG, HyDE 등
- **평가**: LLM-as-a-judge (포괄성, 다양성, 권한 부여도, 전반적 품질)

**주요 결과**:
- LightRAG가 4개 데이터셋에서 NaiveRAG, RQ-RAG 대비 전 지표에서 우위
- GraphRAG와 비교 시: 포괄성과 다양성에서 경쟁적이며, 검색 효율성에서 크게 우수
- 토큰 비용: GraphRAG 대비 대폭 절감 (커뮤니티 요약 생성 불필요)

**증분 업데이트**: 새 문서 추가 시 기존 그래프 구조를 보존하면서 즉시 통합, 전체 재인덱싱 대비 처리 시간 90%+ 절감

**Ablation**: Low-level만 사용 시 추상적 질문 성능 하락, High-level만 사용 시 구체적 사실 질문 성능 하락 → 이중 레벨 모두 필요
