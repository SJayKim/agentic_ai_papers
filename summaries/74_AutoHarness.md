# AutoHarness: Improving LLM Agents by Automatically Synthesizing a Code Harness

> **논문 정보**: Xinghua Lou, Miguel Lázaro-Gredilla, Antoine Dedieu, Carter Wendelken, Wolfgang Lehrach, Kevin P. Murphy (Google DeepMind)
> **arXiv**: 2603.03329 (2026.03)
> **코드**: N/A

---

## Problem

- LLM을 에이전트로 사용할 때, 모델이 외부 환경에서 엄격히 금지된 행동(illegal action)을 빈번하게 시도하는 문제가 심각하다.
- Kaggle GameArena 체스 대회에서 Gemini-2.5-Flash의 패배 중 **78%**가 전략적 실수가 아닌 **단순 불법 수(illegal move)** 때문이었다.
- 체스에서는 잘못된 기물 이동(예: 비숍을 직선으로 움직이거나, 킹을 체크 상태에 노출), 존재하지 않는 칸 지정 등이 전형적인 불법 수 사례이다.
- 체커, 오셀로 등 보드 게임에서도 점유된 칸에 기물을 배치하거나, 뒤집기 규칙을 위반하는 행동이 반복적으로 발생한다.
- Sudoku에서 같은 행/열/블록에 중복 숫자를 배치하거나, Minesweeper에서 이미 공개된 칸을 다시 선택하는 등 퍼즐 게임에서도 규칙 위반이 빈번하다.
- 이는 모델이 게임 규칙을 "이해"하는 것처럼 보이면서도 실제로 규칙을 준수하는 행동을 생성하지 못하는 근본적인 괴리를 드러낸다.
- 기존 대안 1: 게임 궤적 데이터로 LLM을 **파인튜닝**하는 방법은 대규모 모델에서 비용이 높고, instruction following 등 다른 태스크의 성능을 저하시킬 수 있다.
- 기존 대안 2: 수동으로 **harness**(행동 검증 래퍼)를 코딩하는 방법은 각 게임마다 별도 작업이 필요하여 확장성이 낮고, 코드가 취약(brittle)하며 유지보수 비용이 높다.
- 따라서 다양한 환경에 자동으로 적응하면서 불법 행동을 원천 차단하는 **확장 가능한 솔루션**이 필요하다.

## Motivation

- 핵심 통찰은 LLM 자체의 코드 생성 능력을 활용하여 **자기 자신을 위한 harness를 자동 합성**할 수 있다는 것이다.
- LLM은 이미 코드 합성(AlphaCode)과 수학 문제 풀이(올림피아드 수준)에서 뛰어난 능력을 보여주었으므로, 이 능력을 "action applicability" 문제 해결에 전용하는 것이 자연스럽다.
- harness 생성을 **프로그램 공간에서의 탐색 문제**로 정식화하여, 체계적 접근을 가능하게 한다.
- 단순 반복 프롬프팅 대신, **Thompson sampling 기반 트리 탐색**을 사용하여 탐험(exploration)과 활용(exploitation)의 균형을 맞추며 효율적으로 harness 코드를 탐색한다.
- LLM이 **돌연변이 연산자(mutation operator)** 역할을 하여, 환경의 피드백을 기반으로 코드를 점진적으로 개선하는 진화적 접근을 취한다.
- 기존의 내부 world model에 의존하는 접근(Tree of Thoughts 등)은 LLM의 내부 시뮬레이션이 환각(hallucination)에 취약하다는 한계가 있다.
- AutoHarness는 행동 유효성 검증을 **외부의 검증 가능한 프로그램**으로 위임하여 이러한 환각 문제를 구조적으로 회피한다.
- 궁극적으로, 에이전트를 "LLM + harness"의 조합으로 정의하고, LLM이 자기 자신의 harness를 코딩하여 에이전트를 **자동 완성**하는 패러다임을 제시한다.

## Method

AutoHarness의 핵심 아키텍처는 트리 탐색 기반 코드 합성 프레임워크로, 다음과 같은 구성 요소와 과정으로 이루어진다.

**1. 트리 구조 코드 가설 관리**:
- 여러 개의 코드 가설(harness 후보)을 트리 구조로 유지한다.
- 각 노드는 하나의 harness 코드 버전이며, 해당 코드의 합법적 행동 정확도(legal move accuracy)의 평균값이 휴리스틱 값으로 사용된다.
- 트리는 빈 코드 템플릿을 루트로 시작하여, 각 반복에서 자식 노드가 추가되며 확장된다.

**2. Thompson Sampling 기반 노드 선택**:
- 매 반복(iteration)마다 Thompson sampling을 사용하여 다음에 개선할 노드를 선택한다.
- 이를 통해 탐험(새로운 로직 구조 시도)과 활용(부분적으로 작동하는 harness 개선)의 균형을 맞춘다.
- 합법 행동 성공률이 1.0에 도달하면 학습이 종료된다.

**3. Rollout 메커니즘**:
- 선택된 노드의 코드로 **10개 병렬 환경**에서 최대 **1000 스텝**까지 롤아웃을 수행한다.
- 환경은 자동 리셋(auto environment resetting)을 지원하여, 에피소드가 끝나면 새 게임이 시작된다.
- 불법 수가 발생하거나 코드 실행이 실패하면 롤아웃이 즉시 종료된다.

**4. Critic (오류 분석)**:
- 롤아웃에서 수집된 최대 **5개의 실패 스텝**이 Critic에 전달된다.
- 다양한 오류 유형(잘못된 포맷, 규칙 위반, 런타임 에러 등)을 통합 분석하여 구조화된 피드백을 생성한다.

**5. Refiner (코드 개선)**:
- 오류 메시지와 기존 코드를 받아 LLM(Gemini-2.5-Flash)이 개선된 코드를 생성한다.
- 개선 전략은 오류 유형에 따라 조건부로 결정된다:
  - `is_legal_action()`이 True를 반환했는데 실제로 불법 행동인 경우 → **두 함수 모두** 개선
  - `is_legal_action()`이 False를 반환했는데 실제로 불법 행동인 경우 → `propose_action()`**만** 개선
- Refiner 프롬프트는 게임 규칙 설명, 보드 상태, 오류 피드백을 모두 포함하여 맥락 있는 코드 수정을 유도한다.

**6. 세 가지 harness 유형**:
- **Harness-as-Action-Filter**: `propose_action()`으로 합법적 행동 **집합**을 생성하고, LLM이 chain-of-thought 추론을 통해 그 중 최선을 선택
- **Harness-as-Action-Verifier** (주력 방식): LLM이 먼저 행동을 제안하고, `is_legal_action()`으로 검증. 불법이면 재시도
- **Harness-as-Policy**: 코드가 직접 행동을 선택하여 테스트 시 LLM 호출이 불필요. 순수 Python과 numpy 등 표준 라이브러리만 사용

**7. Harness-as-Policy 학습 전략**:
- 휴리스틱을 `H = 0`(불법 행동 시) 또는 `H = 0.5 + 0.5r`(합법 행동 시, r은 환경 보상)로 수정하여, 합법성뿐 아니라 **보상도 최적화**
- 최대 **256 반복**까지 학습하며, 평균 89.4 반복에서 휴리스틱 값 0.939에 도달

## Key Contribution

1. **"Code as Harness" 프레임워크 제안**: LLM이 자기 자신을 위한 harness를 자동으로 코드 합성하여, 불법 행동을 원천 차단하는 새로운 패러다임을 제시했다.
2. **Thompson Sampling 기반 트리 탐색**: 프로그램 공간의 효율적 탐색을 위해 Thompson sampling을 적용한 구조화된 코드 진화 방법을 도입했다.
3. **145개 TextArena 게임에서 100% 합법 행동률 달성**: 1인용 및 2인용 게임 모두에서 학습된 harness가 완벽한 합법성을 보장함을 실증했다.
4. **소형 모델이 대형 모델을 능가**: Gemini-2.5-Flash + Harness가 Gemini-2.5-Pro를 2인용 게임에서 56.3% 대 38.2%로 이기는 결과를 보여, 자동 harness가 모델 크기를 보상할 수 있음을 입증했다.
5. **Harness-as-Policy로 LLM 없이 최고 성능**: 순수 코드 정책이 GPT-5.2-High(0.844)를 능가하는 평균 보상 0.870을 달성하면서 테스트 시 비용이 거의 0에 가까운 효율성을 보였다.

## Experiment & Results

**실험 환경**:
- TextArena의 **145개 게임**을 대상으로 harness를 학습했다.
- 평가는 **16개 1인용(1P)** + **16개 2인용(2P)** 게임에서 수행했다.
- 자유 형식 텍스트/대화 행동 공간을 가진 9개 게임은 제외되었다.
- 난이도를 높이기 위해 관찰(observation)에서 "Available Moves" 힌트를 수동 제거했다.
- 1P 게임은 20회 매치, 2P 게임은 40회 매치(선수/후수 20회씩)로 평가했다.

**학습 효율성**:
- 평균 **14.5회** 트리 탐색 반복으로 학습 완료
- **19/32개** 게임이 10회 미만의 반복으로 완료
- 가장 어려운 게임: GermanWhist-v0(43회), Cryptarithm-v0(45회), Othello-v0(62회), Chess-v0(64회)
- 전 145개 게임에서 **100% 합법 행동 성공률** 달성

**2인용 게임 결과 (Harness-as-Action-Verifier)**:
- Gemini-2.5-Flash+Harness vs Gemini-2.5-Pro: **9/16개** 게임 승리, 전체 승률 **56.3%** (Pro의 승률 38.2%)
- Gemini-2.5-Flash+Harness vs vanilla Gemini-2.5-Flash: **12/16개** 게임 승리, 전체 승률 **64.8%**

**1인용 게임 결과 (Harness-as-Action-Verifier)**:
- 평균 보상: Flash+Harness **0.745** vs Pro **0.707** vs vanilla Flash **0.673**
- 16개 게임 중 8개에서 Pro를 초과, 5개에서 동점

**Harness-as-Policy 결과 (1P 16개 게임)**:
- 평균 보상: Harness-as-Policy **0.870** > GPT-5.2-High **0.844** > Gemini-2.5-Pro **0.707** > Gemini-2.5-Flash **0.673** > GPT-5.2 **0.635**
- 평균 학습 반복: 89.4회, 평균 휴리스틱 값 0.939 도달
- GPT-5.2 및 GPT-5.2-High 실험 비용이 약 **$640**인 반면, Harness-as-Policy는 테스트 시 비용이 거의 **$0**

## Limitation

- **환경별 개별 학습**: 현재 각 게임(환경)마다 별도의 harness를 생성해야 하므로, 새로운 환경이 추가될 때마다 학습 과정을 반복해야 한다. 재사용 가능한 harness 라이브러리 구축은 향후 과제이다.
- **자유 형식 텍스트 환경 제외**: 행동 공간이 자유 형식 텍스트/대화인 9개 게임은 실험에서 제외되었으며, 합법/불법의 경계가 모호한 이러한 환경에서의 적용 가능성은 미검증 상태이다.
- **2인용 게임에 대한 Harness-as-Policy 미적용**: 상대방의 정책에 대한 전략적 추론이 필요한 2인용 게임에서는 MCTS와 같은 런타임 탐색이 필요하여 순수 코드 정책 학습이 매우 어렵다.
- **텍스트 기반 게임에 한정**: 멀티모달 게임(Craftax, Terra Nova 등)으로의 확장은 아직 이루어지지 않았으며, 시각적 관찰을 포함하는 환경에서의 harness 생성은 추가 연구가 필요하다.
- **일반화 부재**: 학습된 harness가 유사 게임 간 전이(transfer)되는지에 대한 분석이 없다. 예를 들어, Chess-v0에서 학습된 harness가 Checkers-v0에 부분적으로 활용될 수 있는지 검증되지 않았다.
- **재귀적 자기 개선 미검증**: 도메인 전문가 에이전트를 기반 LLM에 다시 증류(distill)하는 재귀적 자기 개선 파이프라인은 구상만 제시되고 실험으로 검증되지 않았다.
