# AgentSquare: Automatic LLM Agent Search in Modular Design Space

> **논문 정보**: Yu Shang, Yu Li, Keyu Zhao, Likai Ma, Jiahe Liu, Fengli Xu, Yong Li (Tsinghua University)
> **arXiv**: 2410.06153v3 (2025.02.27) | **학회**: -
> **코드**: https://github.com/tsinghua-fib-lab/AgentSquare

---

## Problem

현재 LLM 에이전트 시스템 설계는 대부분 수작업(hand-crafted)에 의존하며, 특정 태스크에 맞춰 전문가가 직접 아키텍처를 설계해야 한다. 이러한 태스크 특화 설계는 새로운 태스크에 대한 적응력이 크게 떨어지며, 설계 비용과 전문 지식 의존도가 높다. 기존 자동 최적화 연구(OPRO, Promptbreeder 등)는 프롬프트 수준의 rewriting에 국한되어 탐색 공간이 제한적이다. ADAS와 같은 코드 공간 전체 검색 방식은 유연하지만, 기존 연구에서 검증된 모듈들을 명시적으로 재조합하는 능력이 부족하다. 서로 다른 코드베이스에 분산된 에이전트 모듈들의 강점을 체계적으로 결합할 수 있는 표준화된 프레임워크가 부재하다. 또한 에이전트 평가 비용이 매우 높아(예: GPT-4o 기반 ALFWorld 평가 1회에 약 $60), 대규모 검색이 경제적으로 지속 불가능하다.

---

## Motivation

LLM 에이전트 연구 커뮤니티에서 다양한 성공적 모듈(CoT, ToT, Voyager, Generative Agents 등)이 개별적으로 개발되었으나, 이들을 표준화된 인터페이스로 통합하여 재조합할 수 있는 플랫폼이 없다. 기존 에이전트 아키텍처를 Planning, Reasoning, Tool Use, Memory 네 가지 기본 모듈로 추상화하면, 모듈 간 조합을 통해 방대한 설계 공간(1,050개 이상의 조합)을 체계적으로 탐색할 수 있다. 모듈 수준의 표준화된 IO 인터페이스를 정의하면, 서로 다른 연구에서 나온 모듈들을 플러그인 방식으로 쉽게 통합할 수 있다. 프롬프트 수준 최적화만으로는 지역 최적해에 갇히기 쉬우므로, 모듈 진화와 재조합이라는 두 가지 메커니즘을 결합하면 더 넓은 공간을 효율적으로 탐색할 수 있다. AutoML의 Neural Architecture Search에서 영감을 받아, 에이전트 아키텍처에도 유사한 자동 검색 패러다임을 적용한다.

## Method

1. **MoLAS 문제 정의**: LLM 에이전트 A = (P, R, T, M)으로 정의하며, P(Planning), R(Reasoning), T(Tool Use), M(Memory) 네 모듈의 최적 조합을 찾는 최적화 문제로 공식화한다.

2. **모듈형 설계 공간 구축**: NeurIPS, ICML, ICLR 최근 3년 논문을 체계적으로 리뷰하여 16개의 인기 LLM 에이전트를 4개 모듈 카테고리로 분류하고, 각 모듈에 표준화된 IO 인터페이스를 정의. 1,050개 이상의 가능한 조합 공간을 구성한다.

3. **에이전트 워크플로우**: 태스크 수신 → Planning이 서브태스크로 분할 → Reasoning이 순차 해결 → 필요시 Tool Use 호출 → Memory가 과거 경험 저장/검색 → 환경 상호작용 → 피드백 기반 재계획의 반복 루프.

4. **모듈 재조합(Module Recombination)**: LLM을 self-adaptive proposer로 활용하여, 태스크 설명, 모듈 풀, 경험 풀을 참조해 유망한 모듈 조합 N개를 제안. 성능 예측기로 평가하고 최고 성능 후보를 선정.

5. **모듈 진화(Module Evolution)**: FunSearch에서 영감을 받은 진화적 메타 프롬프트로 프로그램 수준에서 새로운 모듈을 생성. 새 모듈은 풀에 추가되고, 자식 에이전트를 생성하여 실제 환경에서 테스트.

6. **성능 예측기**: in-context surrogate model로 태스크 설명, 모듈 프로필, 이전 성능 데이터 기반으로 새 에이전트 성능을 예측. 실제 평가 비용의 약 0.025%만으로 신뢰할 수 있는 추정이 가능.

7. **검색 종료**: 5회 연속 성능 개선이 없으면 종료. 진화와 재조합이 에피소드 단위로 교대 실행되는 순환 구조.

---

## Key Contribution

1. **MoLAS 문제 정의**: LLM 에이전트를 4개 모듈의 조합으로 추상화하고 표준화된 IO 인터페이스 기반의 모듈형 설계 공간을 최초로 제안.
2. **AgentSquare 프레임워크**: 모듈 진화(코드 수준 새 모듈 생성)와 모듈 재조합(기존 모듈 간 전략적 결합)을 결합한 이중 메커니즘.
3. **In-context 성능 예측기**: 실제 평가 비용의 0.025% 수준으로 후보 에이전트의 성능을 정확히 예측.
4. **6개 벤치마크에서 인간 설계 에이전트 대비 평균 17.2% 성능 향상** 달성.
5. **해석 가능한 설계 인사이트 자동 생성**: 검색된 에이전트가 왜 효과적인지 모듈 설명과 설계 원칙을 자동 도출.

## Experiment & Results

- **벤치마크**: Webshop(웹), ALFWorld(체화), SciWorld(체화), M3Tool(도구), TravelPlanner(도구), PDDL(게임) — 4개 도메인 6개 태스크.
- **기반 모델**: GPT-4o 및 GPT-3.5-turbo.
- **GPT-4o 기준 성능**: Webshop 0.607, ALFWorld 0.695, SciWorld 0.781, M3Tool 0.524, TravelPlanner 0.583, PDDL 0.669.
- **인간 설계 대비 개선율**: Webshop +14.1%, ALFWorld +26.1%, SciWorld +20.5%, M3Tool +30.6%, TravelPlanner +6.0%, PDDL +6.0%, 전체 평균 **17.2%** 향상.
- **ADAS 대비**: 동일 토큰 예산 하에 6개 중 5개 태스크에서 우위 (ALFWorld 0.695 vs 0.543, Webshop 0.607 vs 0.521).
- **Ablation**: 모듈 진화 제거 시 ALFWorld 0.695→0.649, 모듈 재조합 제거 시 ALFWorld 0.695→0.616, TravelPlanner 0.583→0.280. 두 메커니즘 모두 필수적이며 재조합이 더 큰 기여.
- **성능 예측기**: 예측-실제 성능 간 높은 상관관계. ALFWorld에서 실제 비용의 0.025%만으로 예측 가능.
- **검색 궤적**: 15회 반복 내 안정적 수렴. Random/Bayesian은 방향성 없이 정체.

## Limitation

- 설계 공간이 4개 모듈로 고정되어 멀티모달 인식, 안전성 모듈 등은 다루지 못한다.
- 단일 에이전트 최적화에 초점으로, 다중 에이전트 시스템의 역할 배분이나 상호작용 패턴 최적화와는 직교적 접근이다.
- 검색 과정의 LLM API 호출 비용이 발생하며, 특히 모듈 진화 단계의 실제 환경 평가가 비용 부담이 된다.
- 성능 예측기가 모듈 재조합에서만 사용되고 모듈 진화에는 적용할 수 없어 진화 단계의 비용 절감에 한계가 있다.
- 초기 모듈 풀의 다양성과 품질이 검색 결과의 상한선에 영향을 준다.
- GPT-4o와 GPT-3.5에서만 실험하여, 오픈소스 LLM이나 다른 상용 모델에서의 일반화 가능성은 미검증이다.
