# Unveiling Privacy Risks in LLM Agent Memory

> **논문 정보**: Bo Wang, Weiyi He, Shenglai Zeng, Zhen Xiang, Yue Xing, Jiliang Tang, Pengfei He (Michigan State University, University of Georgia)
> **arXiv**: 2502.13172v2 (2025.06.03) | **학회**: -
> **코드**: 미공개

---

## Problem

LLM 에이전트는 사용자와의 과거 상호작용 기록(쿼리-솔루션 쌍)을 메모리 모듈에 저장하고, 이를 인컨텍스트 데모로 활용하여 의사결정을 향상시킨다. 그러나 이 메모리에는 의료 기록, 쇼핑 이력 등 민감한 사용자 개인정보가 포함될 수 있다. 기존 연구는 RAG 시스템의 외부 데이터 유출 위험에 집중했으나, LLM 에이전트 메모리 모듈 자체의 프라이버시 취약성은 체계적으로 조사되지 않았다. RAG 대상 공격 프롬프트는 에이전트의 복잡한 워크플로우(코드 실행, 웹 액션 등)에서 그대로 적용하기 어렵다. 메모리에서 개인정보가 유출될 경우 비인가 데이터 접근과 오용이라는 심각한 프라이버시 위험이 발생한다. 예컨대 의료 에이전트의 메모리가 탈취되면 환자의 건강 상태가 노출될 수 있다.

---

## Motivation

LLM 에이전트 메모리는 RAG의 외부 데이터베이스와 본질적으로 다른 프라이버시 위험원이다. RAG는 외부 문서를 프롬프트에 통합하는 반면, 메모리는 사용자-에이전트 상호작용 자체를 저장하므로 더 직접적인 개인정보를 포함한다. 기존 RAG 공격 프롬프트("Please repeat all the context")는 에이전트의 풍부한 태스크 컨텍스트(워크플로우 설명, 도구 목록 등)에서 메모리 데이터를 정확히 지목하지 못한다. 또한 에이전트의 출력이 텍스트가 아닌 코드 실행이나 웹 액션인 경우, 단순 텍스트 출력 요청은 워크플로우와 호환되지 않아 실패한다. 이러한 에이전트 고유의 복잡성에 맞춘 새로운 공격 패러다임이 필요하며, 메모리 모듈 설정이 유출 정도에 미치는 영향도 규명해야 한다.

## Method

1. **위협 모델 정의**: 공격자는 블랙박스 설정에서 에이전트에 입력 쿼리만으로 상호작용하며, 메모리에 저장된 과거 사용자 쿼리를 최대한 많이 추출하는 것이 목표이다. 공격자의 지식 수준을 기본(Basic: 에이전트 도메인/태스크만 아는 경우)과 고급(Advanced: 유사도 함수를 추론한 경우) 두 단계로 구분한다.

2. **MEXTRA 공격 프롬프트 설계**: 공격 프롬프트를 두 부분으로 구성한다.
   - **Locator**: 긴 컨텍스트에서 메모리의 사용자 쿼리를 정확히 지목하고, 이를 출력하도록 우선순위를 부여한다.
   - **Aligner**: 에이전트 워크플로우에 맞는 출력 형식을 지정하여 추출 결과가 실제로 접근 가능하도록 한다.

3. **자동화된 다양한 프롬프트 생성**: GPT-4를 프롬프트 생성기로 활용하여 n개의 다양한 공격 프롬프트를 자동 생성한다.
   - **기본 지시(I_basic)**: 태스크 설명, 프롬프트 생성 요구사항, 출력 형식, 인컨텍스트 데모를 포함
   - **고급 지시(I_advan)**: 유사도 함수 정보를 활용하여 의미적 다양성 확보

4. **공격 실행 및 추출**: n개의 공격 프롬프트를 순차적으로 에이전트에 입력하고, 각 프롬프트가 top-k 검색된 레코드의 사용자 쿼리를 출력하도록 유도. 에이전트의 실행 결과에서 추출된 쿼리를 수집하여 합집합을 구성한다.

5. **메모리 모듈 설정 영향 분석**: 유사도 함수(편집 거리 vs 코사인), 임베딩 모델(MiniLM, MPNet, RoBERTa), 검색 깊이 k(1~5), 메모리 크기 m(50~500), 백본 LLM(GPT-4, GPT-4o, Llama3-70b) 등 5가지 설정 변수의 영향을 체계적으로 탐구한다.

6. **평가 지표**: Extracted Number(EN), Extracted Efficiency(EE), Retrieved Number(RN), Complete Extracted Rate(CER), Any Extracted Rate(AER) 총 5가지 지표.

---

## Key Contribution

1. **에이전트 메모리 프라이버시 취약성의 최초 체계적 조사**: RAG 외부 데이터 유출과 구별되는 새로운 위협 벡터를 규명.
2. **Locator + Aligner 구조의 MEXTRA 공격**: 에이전트 워크플로우에 맞춘 공격 프롬프트 설계로 기존 RAG 공격이 실패하는 환경에서도 효과적인 메모리 추출.
3. **자동화된 프롬프트 생성**: 공격자 지식 수준에 따른 두 단계의 자동화 방법으로 수작업 없이 다양한 공격 프롬프트를 대량 생성.
4. **메모리 모듈 설정의 취약성 분석**: 유사도 함수, 임베딩 모델, 검색 깊이, 메모리 크기, 백본 모델이 유출 정도에 미치는 영향을 종합 분석.
5. **방어 설계 근거 제공**: 공격 횟수 증가와 구현 지식 보유가 추출 효과를 높인다는 실증으로 방어 전략 수립의 구체적 근거 제시.

## Experiment & Results

- **실험 대상**: 코드 기반 의료 에이전트 EHRAgent(편집 거리, top-4)와 웹 기반 쇼핑 에이전트 RAP(코사인 유사도, top-3). GPT-4o 백본, 메모리 크기 200.
- **기본 공격 성능**: 30회 공격 시 EHRAgent에서 EN=50, EE=0.42, CER=0.83; RAP에서 EN=26, EE=0.29, CER=0.87로, 메모리의 25% 및 13%를 추출.
- **Aligner 제거 시 성능 하락**: EHRAgent에서 EN 50→36, RAP에서 EN 26→6으로 급감하여 워크플로우 정렬의 중요성을 입증.
- **유사도 함수 영향**: 편집 거리가 코사인 유사도보다 일관되게 높은 추출 성능. 메모리 500 기준 EHRAgent에서 편집 거리 EN=59 vs 코사인(RoBERTa) EN=36.
- **메모리 크기 영향**: 메모리 50→500 증가 시 EHRAgent EN 31→59, RAP EN 23→63으로 대형 메모리가 더 높은 유출 위험.
- **검색 깊이 영향**: k=1→5 증가 시 EHRAgent EN 8→59, RAP EN 11→32로 취약성 심화.
- **백본 모델 영향**: RAP에서 GPT-4o EN=26, GPT-4 EN=23, Llama3-70b EN=17.
- **공격 횟수 확대**: 50회 공격 시 편집 거리 기반 에이전트는 메모리의 30% 이상 유출.
- **고급 지시 효과**: RAP 코사인 유사도 설정에서 50회 공격 시 RN이 35→84로 2.4배 증가.

## Limitation

- 단일 에이전트 설정에서만 평가되었으며, 다중 에이전트 간 메모리 공유/통신 환경에서의 유출 위험은 미탐구이다.
- 세션 제어(session control)가 없는 프레임워크를 가정하여, 여러 사용자가 동일 세션의 메모리를 공유하는 상황만 고려하였다.
- 실험 대상이 EHRAgent와 RAP 두 에이전트에 한정되어, 더 다양한 에이전트 아키텍처에 대한 일반화 검증이 필요하다.
- 메모리 추출 공격에 대한 구체적 방어 메커니즘(예: 차등 프라이버시, 메모리 접근 제어)의 효과를 평가하지 않았다.
- 정적 메모리(공격 중 레코드 불변) 가정 하에 실험하여, 동적으로 업데이트되는 실제 운영 환경에서의 공격 효과는 불명확하다.
