# Large Language Models Cannot Self-Correct Reasoning Yet

> **논문 정보**: Jie Huang, Xinyun Chen, Swaroop Mishra, Huaixiu Steven Zheng, Adams Wei Yu, Xinying Song, Denny Zhou (Google DeepMind, UIUC)
> **arXiv**: 2310.01798 (2023.10)
> **코드**: N/A

---

## Overview

| 항목 | 내용 |
|------|------|
| **Problem** | "Self-correction"이 LLM의 정확성을 개선하는 유망한 기법으로 제시되어 왔으나, 외부 피드백(오라클 레이블, 도구 결과 등) 없이 모델 자체 능력만으로 추론을 교정하는 "내재적 자기 교정(intrinsic self-correction)"의 실제 효과는 미검증 상태이다. |
| **Motivation** | 근본적 역설: LLM이 스스로 교정할 수 있다면, 왜 처음부터 올바른 답을 내지 않는가? RCI, Reflexion 등 기존 연구의 성능 향상이 실제로는 오라클 레이블에 의존하거나 불공정한 비교에 기인한 것인지 비판적 검토가 필요하다. |
| **Limitation** | 저자 언급: 추론 태스크에 한정된 분석이며, 안전성이나 스타일 개선 등 다른 유형의 자기 교정은 효과적일 수 있음. 독자 관점: 2023년 10월 GPT-4 기준 실험으로, 이후 o1 등 추론 특화 모델에서의 유효성 재검증 필요. 외부 피드백의 범위(코드 실행기, 검증 모델 등)에 대한 체계적 분류 부족. |

---

## Method

1. **Intrinsic Self-Correction 정의**
   - 외부 피드백(오라클 레이블, 도구, 다른 모델) 없이 LLM이 자신의 초기 응답을 자체적으로 검토·수정하는 것
   - 3단계 프롬프팅: (1) 초기 응답 생성 → (2) 자기 검토 및 피드백 생성 → (3) 피드백 기반 재응답

2. **기존 연구의 문제점 분석**
   - **오라클 레이블 의존 (RCI, Reflexion)**: 정답 여부를 확인한 후 틀린 경우만 교정 → 실세계에서 불가능한 설정. 오라클 없이 실행하면 성능 향상 소멸
   - **불공정 비교 (Multi-Agent Debate)**: 다중 에이전트 토론이 self-consistency (동일 호출 수에서 다수결)와 비교 시 우위 없음
   - **비최적 프롬프트 (Self-Refine)**: 초기 응답 프롬프트가 의도적으로 약하고, 교정 프롬프트에 더 많은 정보가 포함. 교정 프롬프트의 정보를 처음부터 제공하면 교정 없이도 더 좋은 결과

3. **체계적 실험**: GSM8K, CommonSenseQA, HotpotQA에서 GPT-3.5, GPT-4, GPT-4-Turbo, LLaMA-2-70B로 최대 2라운드 교정 수행

---

## Key Contribution

1. **내재적 자기 교정의 한계 실증**: 외부 피드백 없이 프롬프트만으로 자기 교정을 시도하면 모든 모델·벤치마크에서 성능이 하락함을 체계적으로 입증
2. **기존 연구의 방법론적 문제 식별**: 오라클 의존, 불공정 비교, 비최적 프롬프트의 세 가지 문제점을 구체적으로 지적하여 자기 교정 연구의 평가 기준 재설정
3. **후속 연구 방향 제시**: 진정한 자기 교정을 위해서는 외부 피드백(코드 실행, 검증 모델 등) 통합이 필수적임을 시사

## Experiment & Results

- **데이터셋**: GSM8K (수학), CommonSenseQA (상식 추론), HotpotQA (다중 홉 QA)
- **모델**: GPT-3.5-Turbo, GPT-4, GPT-4-Turbo, LLaMA-2-70B

**오라클 레이블 사용 시 (기존 연구 재현)**:
- GPT-3.5 GSM8K: 75.9% → 84.3% (+8.4%) — 하지만 오라클이 "틀렸을 때만" 교정을 수행하므로 비현실적

**내재적 자기 교정 (오라클 없음)**:
- GPT-3.5 GSM8K: 75.9% → 74.7% (-1.2%, 2라운드 후)
- GPT-3.5 CommonSenseQA: 75.8% → 41.8% (-34.0%) — 극적인 성능 하락
- GPT-4 GSM8K: 95.5% → 89.0% (-6.5%, 2라운드 후)
- GPT-4 CommonSenseQA: 82.0% → 80.0% (-2.0%)
- GPT-4 HotpotQA: 49.0% → 43.0% (-6.0%)

**원인 분석**: LLM이 자신의 추론 정확성을 판단하는 능력이 부족. 교정 시도 시 정답→오답 변경이 오답→정답 변경보다 더 빈번. 특히 CommonSenseQA에서 교정 프롬프트가 모델을 오답 선택지로 유도하는 편향 발생

**Multi-Agent Debate**: 동일 호출 수의 self-consistency 대비 우위 없음 → 토론의 "개선"은 추가 샘플링에 의한 것
